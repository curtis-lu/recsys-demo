# 二元分類 GBDT：類別不平衡的數學影響與處理

這份手冊接續《[二元分類 GBDT 數學推導手冊](gbdt_binary_classification.md)》（下稱**主手冊**）。主手冊回答「給定一個樣本 $x$，訓練好的 GBDT 怎麼算出預測機率 $p(x)$」；本文站在那套機制之上，只追問一件事：

> **當訓練資料的正類比例 $\bar y$ 很小（類別不平衡）時，這件事在 GBDT 的數學運算裡到底改變了什麼？又該如何有效處理？**

先講一個容易誤解、但貫穿全文的關鍵結論：**類別不平衡不必然讓模型的機率「算錯」**。logistic loss 是 proper scoring rule，而主手冊 Ch 3 證過初始值 $F_0 = \log\frac{\bar y}{1-\bar y}$ 已經把資料的 base rate 吸收進起點 —— 資料夠多、模型夠有彈性時，機率估計不會因為不平衡而系統性偏掉。不平衡真正咬人的，是幾個**具體位置**：起點 $F_0$ 的高度、梯度的拔河、切點 Gain 的「稀有區飢餓」、葉輸出的收縮、以及 0.5 門檻的錯位。

本文分兩部：**第一部「診斷」**沿著主手冊的章節骨架，逐一把不平衡定位到這些數學量上；**第二部「處方」**逐項檢視處理手段，每一招都接回第一部它實際在動的那個量。

### 符號接續（沿用主手冊）

本文不重新定義主手冊已建立的記號，僅在此列出對照，方便獨立閱讀：

| 記號 | 意義 |
|---|---|
| $F(x)$ | raw score（log-odds），$p = \sigma(F)$ |
| $p$ | 預測機率 $\sigma(F) = 1/(1+e^{-F})$ |
| $\bar y$ | 訓練集正類比例（base rate） |
| $F_0$ | 初始常數 $\log\frac{\bar y}{1-\bar y}$（主手冊 Ch 3） |
| $\nu$ | learning rate |
| $g_i, h_i$ | 樣本一階、二階導數：$g_i = p_i - y_i$、$h_i = p_i(1-p_i)$ |
| $G, H$ | 節點層級總和 $\sum g_i$、$\sum h_i$ |
| $G_L, H_L, G_R, H_R$ | 切一刀後左右子節點的總和 |
| $\lambda$ | L2 正則化參數 |
| $\gamma$ | 葉節點輸出 $-G/(H+\lambda)$（主手冊 Ch 6） |
| $\text{Gain}$ | 切點增益 $\frac12\bigl[\frac{G_L^2}{H_L+\lambda}+\frac{G_R^2}{H_R+\lambda}-\frac{G^2}{H+\lambda}\bigr]$（主手冊 Ch 5.3） |

## Ch 1. 起點 $F_0$：prior 就是 base rate

主手冊 Ch 3 推過，第一棵樹還沒蓋之前，模型對所有樣本給出的常數值是

$$F_0 = \log \frac{\bar y}{1 - \bar y}$$

也就是「訓練集整體正類比例的 log-odds」，對應初始機率 $p_0 = \sigma(F_0) = \bar y$。這個式子本身沒有「平衡 / 不平衡」的概念 —— 它只是忠實反映資料的 base rate。不平衡的影響，是從這裡的**數值大小**開始的。

把幾個正類比例代進去看：

| $\bar y$ | $F_0 = \log\frac{\bar y}{1-\bar y}$ |
|---|---|
| $50\%$ | $\phantom{-}0$ |
| $10\%$ | $-2.197$ |
| $\phantom{0}1\%$ | $-4.595$ |
| $0.1\%$ | $-6.907$ |

$\bar y$ 越小，起點 $F_0$ 越往負的方向跑。對 $\bar y = 1\%$ 的資料，每個樣本的旅程都從 $F = -4.595$（$p = 0.01$）出發。

### 起點低 → 稀有正類要爬一段很長的路

主手冊 Ch 2 說過 $F(x) = F_0 + \nu f_1(x) + \nu f_2(x) + \cdots$ —— 各棵樹的輸出乘上 learning rate $\nu$ 後累加。一個樣本要被判成正類（$p \ge 0.5$，即 $F \ge 0$），所有樹合計必須把它的 $F$ 從 $F_0$ 一路抬到 $0$，淨抬升 $|F_0|$。

$\bar y = 1\%$ 時這段距離是 $4.595$；在 $\nu = 0.1$、每棵樹葉輸出量級 $O(1)$ 的常見設定下，光是抵銷起點就要相當數量的樹朝同方向用力。樹數有限、$\nu$ 又小時，**一個排序上明明很像正類的樣本，$F$ 也可能還沒爬過 $0$**。

### 「模型把所有人都預測成負類」常常是門檻假象

這帶出最常見的誤解。在 $\bar y = 1\%$ 的資料上訓練完，用 0.5 當門檻一看 —— 幾乎沒人被判成正類，於是下結論「模型壞了 / 不平衡毀了模型」。

但拆開看：一個**真正例**最後拿到 $p = 0.4$，它沒過 0.5 沒錯，可是 $0.4$ 是 base rate $0.01$ 的 **40 倍**。模型其實把它從「平均 1%」抬到了「40%」，這是高度有資訊的輸出 —— 排序上它早就遠遠排在多數負類前面。沒過 0.5，是因為 **0.5 這個門檻對 1% 的資料根本不合理**（Ch 5 細談），不是因為機率算錯。

換句話說：Ch 1 的起點效應主要製造的是「**門檻錯位**」這個症狀，而不是「機率系統性偏掉」。記住這個區分，第二部選招時才不會用錯藥。

## Ch 2. 梯度與海森的階級失衡

主手冊 Ch 4、Ch 5.3 給過每個樣本帶進樹的兩個量：

$$g_i = p_i - y_i, \qquad h_i = p_i(1 - p_i)$$

樹要學什麼、切點好不好，全靠這兩個數的加總（$G = \sum g_i$、$H = \sum h_i$）。要看不平衡怎麼作用，就把這兩個量在「起點」（所有人 $p_i = \bar y$）的樣子寫出來：

| | $g_i = p_i - y_i$ | $h_i = p_i(1-p_i)$ |
|---|---|---|
| 負類（$y=0$，多數） | $\bar y$（小正值） | $\bar y(1-\bar y)$ |
| 正類（$y=1$，少數） | $\bar y - 1$（接近 $-1$ 的大負值） | $\bar y(1-\bar y)$ |

$\bar y = 1\%$ 時：負類 $g = +0.01$、正類 $g = -0.99$。兩類的 $h$ 在起點剛好一樣（都等於 $\bar y(1-\bar y) = 0.0099$，因為 $h$ 只看 $p$、不看 $y$）。

### $G$ 是一場人數懸殊的拔河

$G = \sum g_i$ 把所有樣本的 $g$ 加起來。負類「人多、每個出小正值」，正類「人少、每個出大負值」。在**全體**上這場拔河是平手的 —— 主手冊 Ch 3 的一階條件正是 $\sum_i g_i = n\sigma(F_0) - \sum_i y_i = 0$，這也是 $F_0$ 被選成那個值的理由。

問題出在**局部**。樹不是看全體，是看一個個節點、一刀刀切。在任何一塊區域裡，少數正類那幾個大負梯度，很容易被周圍一大群負類的小正梯度稀釋、淹沒。要靠某個特徵把「正類的負梯度」從「負類的正梯度」中分離出來，才談得上 Gain —— 而正類越稀少，能被乾淨分離的訊號就越微弱、越零星。

### $\bar y$ 小，連 $H$ 都跟著小

順帶記下一件 Ch 3 馬上要用的事：起點處每個 $h_i = \bar y(1-\bar y)$，$\bar y$ 越小這個值越小（$1\% \Rightarrow 0.0099$，$0.1\% \Rightarrow 0.000999$）。於是整個節點的 $H = \sum h_i$ 也偏小。$H$ 出現在 Gain 與葉輸出的**分母**，分母一小，正則化參數 $\lambda$ 的相對份量就變重 —— 這正是下一章「稀有區飢餓」的引信。

## Ch 3. Gain 與「稀有區飢餓」

這是不平衡在訓練數學裡咬得最深的一章。主手冊 Ch 5.3 的 Gain 公式：

$$\text{Gain} = \frac12\left[\frac{G_L^2}{H_L+\lambda} + \frac{G_R^2}{H_R+\lambda} - \frac{G^2}{H+\lambda}\right]$$

樹挑 Gain 最大的切點。我們用一個能手算的小節點，看不平衡 + 正則化怎麼一起把「切出稀有正類」這件事壓下去。

### 範例設定

某節點有 20 個樣本：**18 個負類、2 個正類**，當前所有人 $p_i = 0.01$（剛離開起點不久）。由 Ch 2 的式子：

- 負類：$g = 0.01$、$h = 0.01 \times 0.99 = 0.0099$
- 正類：$g = 0.01 - 1 = -0.99$、$h = 0.0099$

節點層級：

$$G = 18(0.01) + 2(-0.99) = 0.18 - 1.98 = -1.80, \qquad H = 20 \times 0.0099 = 0.198$$

假設某個特徵剛好能把那 **2 個正類乾淨切到左葉**、18 個負類全到右葉：

- 左葉（2 正）：$G_L = 2(-0.99) = -1.98$，$H_L = 2(0.0099) = 0.0198$
- 右葉（18 負）：$G_R = 18(0.01) = 0.18$，$H_R = 18(0.0099) = 0.1782$

這是一刀**完美**的切 —— 把所有「想往正類推」（$g<0$）和「想往負類推」（$g>0$）的樣本分得乾乾淨淨。直覺上 Gain 應該很大。

### 沒有正則化（$\lambda = 0$）時，Gain 確實很大

$$\text{Gain} = \frac12\left[\frac{(-1.98)^2}{0.0198} + \frac{0.18^2}{0.1782} - \frac{(-1.80)^2}{0.198}\right] = \frac12[198.0 + 0.182 - 16.36] \approx 90.9$$

漂亮。但那個 $198.0$ 是怎麼來的？是 $\frac{G_L^2}{H_L} = \frac{3.92}{0.0198}$ —— **分母 $H_L = 0.0198$ 極小**把它撐爆的。而 $H_L$ 小，正是因為左葉只有 2 個樣本、每個 $h$ 又因 $p$ 接近 0 而很小（Ch 2）。換句話說，這個巨大 Gain 有很大一部分是「資訊量極少的小葉子」造出來的數值假象。

### 加上正則化（$\lambda = 1$），Gain 幾乎全沒了

$$\text{Gain} = \frac12\left[\frac{3.92}{0.0198 + 1} + \frac{0.0324}{0.1782 + 1} - \frac{3.24}{0.198 + 1}\right] = \frac12[3.844 + 0.0275 - 2.705] \approx 0.583$$

從 $90.9$ 掉到 $0.583$ —— **縮了大約 156 倍**。原因就是 $H_L = 0.0198$ 跟 $\lambda = 1$ 比起來微不足道：分母從 $0.0198$ 變成 $1.0198$，那一項從 $198$ 直接塌到 $3.84$。$\lambda$ 對「只有 2 筆證據的葉子」狠狠打了折扣。

不只 $\lambda$。實作上還有 `min_child_weight` / `min_sum_hessian_in_leaf`（葉內 $H$ 的下限）和 `min_data_in_leaf`（葉內樣本數下限）。左葉 $H_L = 0.0198$、只有 2 筆，很容易直接低於這些門檻，於是這一刀**根本不被允許切**。

### 這就是「稀有區飢餓」

把上面兩段合起來：一刀**完美分離稀有正類**的切點，在正則化與葉大小限制下，可能拿不到足夠 Gain、甚至被禁止。樹於是轉去切那些「服務多數類」、樣本多 $H$ 大、Gain 看起來更穩的切點。稀有正類的專屬區域，就這樣被「餓」著、長不出來。

要強調的是：**這不是 bug。** 對只有 2 個樣本撐起來的葉子保持懷疑，正是 $\lambda$ 與 min-child 限制存在的理由 —— 它們擋掉的是過擬合訓練雜訊。問題在於，當正類本來就稀少，「謹慎」與「把少數類學起來」之間的天平，會系統性地倒向前者。

這也預告了第二部的解法：如果能**人為放大那 2 個正類的 $g$ 和 $h$**，左葉的 $G_L$、$H_L$ 就會變大、$\lambda$ 的相對份量變小，這刀的 Gain 就能救回來。那正是 Ch 11 類別權重要做的事 —— 我們會把這個一模一樣的範例再算一次。

## Ch 4. 葉輸出 $\gamma$ 的收縮

切點定了之後，主手冊 Ch 6 給每片葉子的最佳輸出：

$$\gamma = -\frac{G}{H + \lambda}$$

這是 Newton step，**公式本身對不平衡是中性、無偏的** —— 它沒有偏袒任何一類。但分母 $H + \lambda$ 又一次讓稀有正類吃虧。

沿用 Ch 3 那個乾淨左葉（2 個正類）：$G_L = -1.98$、$H_L = 0.0198$。

- $\lambda = 0$：$\gamma = -\dfrac{-1.98}{0.0198} = 100$
- $\lambda = 1$：$\gamma = -\dfrac{-1.98}{0.0198 + 1} = \dfrac{1.98}{1.0198} \approx 1.94$

$\lambda = 0$ 時 $\gamma = 100$ 是個荒謬的大跳（把 2 個樣本的訊號放大成爆衝，這正是主手冊 Ch 6 要 $\lambda$ 的原因）；加了 $\lambda = 1$ 後 $\gamma$ 被拉到 $1.94$。$1.94$ 不算小，但別忘了 Ch 1：在 $\bar y = 1\%$ 下，一個正類要被推過門檻得淨爬升 $4.595$，而這片葉子每次的貢獻還要再乘上 learning rate $\nu$（例如 $\times 0.1$ 變成 $0.194$）。

於是稀有正類面對的是雙重壓制：**起點低（要爬 $|F_0|$）＋ 每次能爬的步伐被 $\lambda$ 收縮**。葉輸出公式無偏，但「小樣本葉被正則化拉小修正幅度」這件事，落在稀有正類身上的次數遠多於多數類。

## Ch 5. 決策門檻 0.5 的錯位

前四章都在講「$F$ 怎麼被算出來、又怎麼被壓低」。這章講最後一步：把 $p$ 變成「正 / 負」決策時，那條線該畫在哪。

主手冊 Ch 1.4 用的是預設門檻 0.5：$p \ge 0.5$ 判正類。展開它的意思：

$$p \ge 0.5 \iff F \ge 0 \iff \frac{p}{1-p} \ge 1 \;\;(\text{odds} \ge 1{:}1)$$

也就是說，0.5 這條線默認「正類和負類一樣可能、而且判錯的代價對稱」。可是不平衡資料的 base odds 是 $\dfrac{\bar y}{1-\bar y} \ll 1$（$\bar y = 1\%$ 時約 $1{:}99$）。拿一條「假設五五波」的線，去切一個「實際一比九十九」的世界，當然幾乎切不出正類 —— 這就是 Ch 1 講的「全被預測成負類」的門檻面真相。

### 門檻是可以挪的，而且該挪

對機率設任意門檻 $\tau$，等價於對 raw score $F$ 設門檻：

$$p \ge \tau \iff F \ge \log\frac{\tau}{1-\tau} = \operatorname{logit}(\tau)$$

所以「調門檻」從頭到尾沒有動模型、沒有重訓練，只是把那條線從 $F = 0$ 平移到別處。把 $\tau$ 往下調（例如調到接近 $\bar y$、或更精細地按成本決定），稀有正類就抓得回來。**怎麼挑這個 $\tau$ 是第二部 Ch 10 的主題**，這裡先確立一件事：

> 在不平衡資料上，0.5 不是中性預設，而是一個幾乎注定錯的選擇。把它當成可調的旋鈕，是處理不平衡最便宜的一步。

## Ch 6. 不平衡程度光譜：$1\% \to 0.1\% \to 0.01\%$

前五章每一個效應，嚴重程度都隨 $\bar y$ 變化。這章把它們攤平成 $\bar y$ 的函數，掃過三個級距，給一個「程度感」。假設總樣本固定 100 萬筆：

| | $\bar y = 1\%$ | $\bar y = 0.1\%$ | $\bar y = 0.01\%$ |
|---|---|---|---|
| $F_0 = \log\frac{\bar y}{1-\bar y}$ | $-4.595$ | $-6.907$ | $-9.210$ |
| 推過 $p=0.5$ 要爬升的 $F$（$=\lvert F_0\rvert$） | $\approx 4.6$ | $\approx 6.9$ | $\approx 9.2$ |
| 100 萬筆中的正類數 | $10{,}000$ | $1{,}000$ | $100$ |
| $\text{scale\_pos\_weight} \approx n_{neg}/n_{pos}$ | $\approx 99$ | $\approx 999$ | $\approx 9{,}999$ |
| 主要後果 | 門檻偏移、要多爬一點 | 稀有區開始飢餓 | 加權方差爆炸、極少正類噪音主導 |

三件事隨 $\bar y$ 變小而惡化：

1. **起點越來越深（$\lvert F_0\rvert$ 對數成長）。** $1\% \to 0.01\%$，要爬的距離從 $4.6$ 增到 $9.2$、翻倍。Ch 1 的「爬不完」在極端區更明顯。
2. **正類的絕對數量驟減。** 同樣 100 萬筆，正類從 1 萬掉到 100 個。Ch 3 的「稀有區飢餓」不只是比例問題，更是**絕對證據量**問題 —— 100 個正類散落在高維特徵空間裡，能乾淨切出來的純正類區域屈指可數，每個切點背後的 $H$ 都小到被 $\lambda$ 主宰。
3. **想用權重補償，補償量本身變危險。** 第二部會用 $w \approx n_{neg}/n_{pos}$ 來平衡梯度（Ch 11）。但 $\bar y = 0.01\%$ 時 $w \approx 10000$ —— 把那 100 個正類每個乘上一萬倍，等於整個目標函數被「100 個樣本」主宰；其中任一筆的標註雜訊或離群，都被放大一萬倍灌進梯度。**補償不平衡的代價，是把方差換上來**：模型在極少數正類上劇烈擺動、極易過擬合。

這條光譜也說明為什麼「處理不平衡」沒有萬靈丹：$1\%$ 時調個門檻可能就夠；到 $0.01\%$，問題從「門檻」升級成「資料根本不夠撐起稀有結構」，再多的重採樣 / 加權也只是搬動方差，無法無中生有更多正類訊號。這時往往得回頭找「更多正類標註 / 更好的特徵 / 換問題定義」，而非在演算法層硬調。

## Ch 7. 評估與早停的陷阱（簡述）

不平衡也會騙過你看模型的眼睛。這節點到為止，三個常見坑：

- **Accuracy 被多數類灌水。** $\bar y = 1\%$ 時，一個「全部猜負類」的廢模型 accuracy 就有 99%。準確率在不平衡下幾乎不帶資訊，別拿它當主指標。
- **ROC-AUC 對 base rate 不敏感。** ROC-AUC 只看正、負類各自內部的排序，與正類比例無關 —— 它可以很漂亮（0.9+），卻沒告訴你「在你實際要用的操作點上，抓到的正類裡有多少是真的」。不平衡情境更該看 **PR-AUC（precision-recall 曲線下面積）/ average precision**，它對正類比例敏感，直接反映稀有正類的抓取品質。
- **早停看錯指標會提早收手。** 用驗證集 log-loss 早停時，loss 被一大群多數類主導，可能在模型還沒學出稀有正類結構前，整體 log-loss 就停止改善而觸發早停。早停的監看指標要對齊你真正在意的東西（見 Ch 15）。

「到底該用什麼指標、怎麼挑」屬於處方範圍，留待第二部一併談（Ch 9 的「要機率還是要決策」會牽動指標選擇）。

## Ch 8. 第一部小結：一張對照表

把第一部的診斷收成一張表 —— 不平衡咬在哪、對應主手冊的哪個數學量、表現出什麼症狀、以及第二部對應的處方：

| 咬在哪 | 數學量（主手冊章） | 症狀 | 處方 |
|---|---|---|---|
| 起點太低 | $F_0 = \log\frac{\bar y}{1-\bar y}$（Ch 3） | 稀有正類要爬 $\lvert F_0\rvert$、常爬不過 0.5 | 調門檻（[Ch 10](#ch10)）、權重（[Ch 11](#ch11)） |

| 葉輸出收縮 | $\gamma = -G/(H+\lambda)$（Ch 6） | 稀有正類每步修正幅度被拉小 | 權重（[Ch 11](#ch11)）、調 $\lambda$ |
| 門檻錯位 | $p \ge 0.5 \iff F \ge 0$（Ch 1.4） | 用 0.5 幾乎切不出正類 | 調門檻（[Ch 10](#ch10)） |

一個橫貫全表的觀察：**多數症狀的根源不是「機率算錯」，而是「起點低 ＋ 正則化對小樣本葉的合理懷疑」這兩件事，剛好都對稀有正類不利。** 這決定了第二部的基調 —— 先分清楚你要的是「可信的機率值」還是「操作點上的好決策」，再對症下藥。

## Ch 9. 先問對問題：要機率值，還是要操作點上的好決策？

第一部反覆出現一個訊息：不平衡的多數症狀不是「機率算錯」，而是「門檻錯位」「稀有區被正則化壓抑」。所以開藥之前，先確定你真正需要模型輸出什麼。單一二元分類的用法分兩種：

- **(a) 需要機率數值本身可信。** 你要拿 $p$ 去做後續計算 —— 期望損益、風險加權、跟其他數字相乘比較。這時 $p = 0.4$ 必須真的代表「40% 會發生」，絕對值有意義。
- **(b) 只需要在某個操作點做對二元決策。** 你只要選一個門檻 $\tau$，對 $p \ge \tau$ 的樣本採取行動。這時重要的只有**排序**（誰比誰更該被選）和**門檻切在哪**，$p$ 的絕對校準無所謂 —— 不論它是 $0.4$、還是某個被整體拉偏後的數值，都不影響「它排在多數負類前面、且高於門檻」這件事。

**為什麼這個區分是選招的總開關？**

很多處理手段（權重、重採樣）會把輸出機率弄偏 —— 這是 Ch 11、Ch 12 會證明的數學後果。但這個「副作用」傷不傷得到你，完全取決於你在 (a) 還是 (b)：

- 落在 **(b)**：機率被弄偏**無所謂**。你本來就會在驗證集上重挑門檻 $\tau$，偏移只是把「好門檻」挪了位置，你照樣挑得到。所以 (b) 的人可以放心用權重 / 重採樣，不必管校準。
- 落在 **(a)**：機率被弄偏**有所謂**。這時要嘛避免會扭曲機率的手段、要嘛事後把機率校正回來（Ch 11、Ch 12 各給一行校正式；完整的機率校準是另一個獨立主題，超出本文範圍）。

換句話說：**本文預設大多數讀者落在 (b)**（操作點決策是不平衡情境最常見的需求），所以全文把「校準」壓到最小 —— 只在權重 / 重採樣處留一句警語。如果你確定自己在 (a)，讀到那兩處警語時請特別留意。

## Ch 10. 調門檻：最便宜、不扭曲模型的解

Ch 5 已確立：0.5 對不平衡資料是個錯門檻，而門檻可以自由平移、完全不動模型。這章講怎麼挑那個 $\tau$。

### 有明確成本時：算出最佳門檻

假設判錯有明確代價：把一個負類誤判為正類（false positive）成本 $C_{FP}$、把一個正類漏判為負類（false negative）成本 $C_{FN}$。對一個預測機率為 $p$ 的樣本，兩種決策的期望成本：

- 判為正類：只有當它其實是負類（機率 $1-p$）才付出 $C_{FP}$ → 期望成本 $(1-p)\,C_{FP}$
- 判為負類：只有當它其實是正類（機率 $p$）才付出 $C_{FN}$ → 期望成本 $p\,C_{FN}$

「判正類」划算的條件是它的期望成本較低：

$$(1-p)\,C_{FP} < p\,C_{FN} \iff p > \frac{C_{FP}}{C_{FP} + C_{FN}}$$

所以最佳門檻

$$\boxed{\;\tau^* = \frac{C_{FP}}{C_{FP} + C_{FN}}\;}$$

注意這條式子裡**沒有 $\bar y$** —— 它只看成本比。不平衡的影響已經透過模型的 $p$（其分布因 $F_0$ 偏低）反映進來了；門檻只負責「按成本決定切在哪」。漏掉一個正類很貴（$C_{FN} \gg C_{FP}$）時，$\tau^*$ 會遠低於 0.5，於是抓回更多稀有正類 —— 這正是不平衡情境通常想要的方向。

### 沒有明確成本時：在驗證集上掃

成本說不清楚時，就在**驗證集**上掃一圈 $\tau$，挑最大化你在意指標的那個：

- 最大化 $F_\beta = (1+\beta^2)\dfrac{\text{precision}\cdot\text{recall}}{\beta^2\,\text{precision} + \text{recall}}$（$\beta > 1$ 偏重 recall、$\beta < 1$ 偏重 precision）
- 或：在「recall 不低於某目標」的限制下挑 precision 最高的 $\tau$（反之亦然）

關鍵是 $\tau$ 要在**獨立的驗證 / 校準集**上挑，不能用訓練集（會高估），也別直接用測試集（會洩漏）。

### 為什麼這招該最先試

調門檻**不重訓練、不動模型、不扭曲機率**（它只決定怎麼讀模型的輸出）。對落在 Ch 9 (b) 的人，很多時候光調門檻就解掉了「全被預測成負類」的問題，根本不必動到後面那些帶副作用的手段。**先試最便宜、無副作用的，再考慮要不要加重藥。**

## Ch 11. 類別權重 / `scale_pos_weight`

調門檻（Ch 10）不改模型。如果問題不只是門檻、而是 Ch 3 的「稀有區飢餓」—— 樹根本切不出稀有正類的專屬區域 —— 那就得在訓練時動梯度。類別權重是最直接的做法。

### 機制：把正類的 $g_i, h_i$ 乘上權重 $w$

給正類一個權重 $w > 1$（$w$ 即「每個正類算幾個」）。主手冊的 $g_i, h_i$ 是 loss 對 $F$ 的一、二階導數，乘 $w$ 後：

$$g_i \to w\,g_i, \qquad h_i \to w\,h_i \quad (\text{正類；負類不變})$$

這放大了正類在 $G = \sum g_i$、$H = \sum h_i$ 裡的份量 —— 於是能分離正類的切點 $G_L$ 變大、Gain 變大；含正類的葉子 $H$ 變大、$\lambda$ 的相對壓制變小。

### 用 Ch 3 的範例再算一次

回到 Ch 3：20 樣本（18 負 + 2 正），$p = 0.01$。取 $w = n_{neg}/n_{pos} = 18/2 = 9$。正類加權後：

$$g_{\text{正}} = 9 \times (-0.99) = -8.91, \qquad h_{\text{正}} = 9 \times 0.0099 = 0.0891$$

節點層級（負類不變）：

$$G = 18(0.01) + 2(-8.91) = 0.18 - 17.82 = -17.64, \qquad H = 18(0.0099) + 2(0.0891) = 0.1782 + 0.1782 = 0.3564$$

那刀乾淨分離 2 正類的切點：

- 左葉（2 正）：$G_L = 2(-8.91) = -17.82$，$H_L = 2(0.0891) = 0.1782$
- 右葉（18 負）：$G_R = 0.18$，$H_R = 0.1782$

代入 Gain（$\lambda = 1$，與 Ch 3 同樣有正則化）：

$$\text{Gain} = \frac12\left[\frac{(-17.82)^2}{0.1782 + 1} + \frac{0.18^2}{0.1782 + 1} - \frac{(-17.64)^2}{0.3564 + 1}\right] = \frac12[269.5 + 0.0275 - 229.4] \approx 20.1$$

對比 Ch 3 同一刀、同樣 $\lambda = 1$、**沒加權**時的 Gain $= 0.583$。加權把它從 $0.583$ 拉到 $\approx 20.1$ —— 這刀重新變成全節點最值得切的選擇。**這就是權重救回「稀有區飢餓」的數學**：放大正類的 $g, h$ 後，純正類切點的訊號終於蓋過 $\lambda$ 的壓制。

### 怎麼選 $w$

最常見的起點是 $w = n_{neg}/n_{pos}$（讓兩類的總梯度質量大致相等）。但這不是鐵則 —— Ch 6 提醒過，$\bar y$ 極端時 $w$ 會衝到上萬，反而把方差換上來。實務上 $w$ 是個可調超參數：從 $n_{neg}/n_{pos}$ 出發，依驗證集表現往下調（常常不必調到全平衡就夠）。

### LightGBM 的三個入口

- **`scale_pos_weight`**（純量）：把所有正類的 $g, h$ 乘上這個數，就是本章的 $w$。
- **`is_unbalance=True`**：要 LightGBM 自動把 `scale_pos_weight` 設成 $\approx n_{neg}/n_{pos}$。**與 `scale_pos_weight` 互斥**，別同時開（會重複作用）。
- **`sample_weight`**（逐列權重）：最一般，可每筆樣本給不同權重（不限於「按類別」），適合成本隨樣本而異的情況。

三者本質都是在改 $g_i, h_i$ 的加權，差別只在「權重怎麼指定」。

### 與 $\lambda$ / `min_sum_hessian` 的交互

加權同時放大了葉內 $H$（上例 $H_L$ 從 $0.0198$ 變 $0.1782$，正好 $\times 9$）。所以它有個附帶好處：原本因 $H$ 太小而卡在 `min_sum_hessian_in_leaf` 下限、不被允許的純正類切點，加權後 $H$ 按 $w$ 倍放大，較容易跨過門檻。換句話說，權重不只放大 Gain，也鬆綁了葉大小限制對稀有正類的封鎖。

### 機率警語（落在 Ch 9 (a) 的人才需要看）

加權改變了被最小化的 loss —— 模型不再估計真實的 $P(y=1\mid x)$，而是估計「正類被放大 $w$ 倍後的世界」的機率。結果是**輸出 $p$ 系統性偏高、不再校準**。

- 落在 **Ch 9 (b)**（只要操作點決策）：無所謂，在驗證集上重挑門檻即可。
- 落在 **Ch 9 (a)**（要可信機率）：可用這條 prior-shift 把機率還原回真實世界：

$$p_{\text{true}} = \frac{p_w}{p_w + (1 - p_w)\,w}$$

（$p_w$ 是加權模型的輸出。直覺：加權把正類的 odds 放大了 $w$ 倍，除回去即還原。）完整的機率校準（例如再用一個獨立集做等滲迴歸）超出本文範圍。

## Ch 12. 重採樣：欠採、過採、SMOTE

重採樣不改梯度公式，改的是「丟進去訓練的資料分布」。三種做法，各自動到第一部不同的量。

### 欠採樣（undersample 多數類）

隨機丟掉一部分負類，讓正類比例升高。直接的數學後果是 **base rate 變了**：訓練集的 $\bar y$ 從真實值升到 $\bar y_s$，於是 Ch 1 的起點

$$F_0 = \log\frac{\bar y_s}{1 - \bar y_s}$$

跟著上移，稀有正類不必再從那麼深的地方起爬。好處：類別變平衡、資料量變小、訓練更快。代價：**丟掉的負類就是丟掉的資訊**，多數類結構學得更糙、估計方差上升。

如果你落在 Ch 9 (a) 要還原真實機率：欠採只是平移了 prior，可用已知量校正。訓練在 $\bar y_s$、真實是 $\bar y_t$ 時，

$$F_{\text{true}} = F_{\text{sampled}} + \log\frac{\bar y_t /(1-\bar y_t)}{\bar y_s /(1-\bar y_s)}$$

特例：欠採到完全平衡（$\bar y_s = 0.5$）時，後項 $\log\frac{0.5/0.5}{\,\cdot\,}$ 中的 $\log\frac{0.5}{0.5} = 0$，平移量恰好等於真實的 $F_0 = \log\frac{\bar y_t}{1-\bar y_t}$ —— 等於「把訓練時人為歸零的起點，加回它本來該有的深度」。

### 過採樣（oversample 少數類，複製）

把正類複製數份，讓比例升高。這裡有個值得記住的等價關係：

> **複製一個樣本 $k$ 份，在 GBDT 的 Gain / $\gamma$ 計算上，完全等同於把它加權 $k$ 倍。**

證明很短：任何包含樣本 $i$ 的節點，$G = \sum g$ 與 $H = \sum h$ 現在要把 $i$ 的 $g_i, h_i$ 各加 $k$ 次，貢獻變成 $k\,g_i$、$k\,h_i$ —— 跟 Ch 11 給 $i$ 權重 $k$ 後的 $w\,g_i, w\,h_i$ 一模一樣。而 Gain、$\gamma$ 全由 $G, H$ 決定，所以兩者算出的切點與葉輸出完全相同。也因此 **整數倍過採 = 整數權重**，Ch 11 的機率警語對它一樣適用。

但兩者有個微妙差異藏在「葉大小限制」裡：

- `min_data_in_leaf`（葉內**樣本數**下限）：複製把 $k$ 份算成 $k$ **筆**，較容易湊過門檻；加權仍是 **1 筆**。
- `min_sum_hessian_in_leaf`（葉內 **$H$** 下限）：兩者都看 $k\,h_i$，沒有差異。

還有複製獨有的風險：**完全相同的列永遠落在同一片葉子**，樹可以靠這幾個被反覆複製的點「背答案」，過擬合風險比加權更高。

### SMOTE（合成少數類）

SMOTE 不複製，而是在特徵空間中對鄰近正類做**插值**，造出「新的」合成正類。它在影像 / 連續特徵上有時有效，但對 GBDT + 表格資料有幾個侷限：

- 樹靠閾值切分，對「兩個真實正類中間插出來的點」未必買單；插值點可能落在現實不存在的特徵組合上（例如插出「年齡 35、年資 40 年」這種矛盾樣本）。
- 類別型特徵無法線性插值（需 SMOTE-NC 等變體，效果參差）。
- 計算成本：要算最近鄰，高維大資料下不便宜。

對表格 + 樹模型，SMOTE 通常不是首選；先試權重 / 欠採往往更穩。

### 三法對照

| 手段 | 對 $g/h$ 總和的效果 | 對 base rate / $F_0$ | 變異數 / 過擬合 | 計算成本 | 何時用 |
|---|---|---|---|---|---|
| 欠採多數類 | 少了負類項 | $\bar y_s$ 升、$F_0$ 上移（可校正） | 丟資料 → 方差上升 | 變低（資料變少） | 資料量大、負類冗餘多時 |
| 過採少數類（複製） | 正類項 $\times k$（= 加權 $k$） | 同加權，prior 偏移 | 複製列易背答案 → 高 | 變高（資料變多） | 想湊 `min_data_in_leaf` 時 |
| 類別權重（Ch 11） | 正類 $g,h \times w$ | 同上，prior 偏移 | 不增資料、較可控 | 幾乎不變 | 一般首選的「動梯度」手段 |
| SMOTE | 加入合成正類項 | 偏移且引入合成雜訊 | 合成點可能不真實 → 風險 | 高（算最近鄰） | 表格 + 樹模型通常不優先 |

一句話：**欠採動 base rate、過採與權重動梯度權重（兩者數學等價）、SMOTE 動資料本身。** 多數情況下類別權重（Ch 11）是 CP 值最高的起點 —— 不丟資料、不造假資料、成本幾乎為零，且和 Ch 3 範例展示的「救回 Gain」直接對應。

## Ch 13. 自訂 loss：focal / 加權 log-loss

權重（Ch 11）對「所有正類」一視同仁地乘 $w$。更精細的做法是換掉 loss 本身，讓它**按樣本難易**自動調整梯度。GBDT 支援自訂 loss —— 你只要提供它對 $F$ 的一階、二階導數（$g_i, h_i$），主手冊推導的 Gain、$\gamma$ 機制照常運作。

**加權 log-loss** 是最簡單的一種：把 Ch 11 的逐類權重寫進 loss，效果等價。

**Focal loss** 更進一步，乘上一個調變因子 $(1 - p_t)^\gamma$（$p_t$ 是模型對「正確類別」給的機率，$\gamma \ge 0$ 是聚焦強度）：

- 對已分對的「容易樣本」（$p_t \to 1$），$(1-p_t)^\gamma \to 0$，梯度被大幅調降；
- 對還沒分對的「困難樣本」（$p_t$ 小），調變因子接近 1，梯度幾乎不打折。

在不平衡情境，大量多數類往往是「容易樣本」。focal loss 把它們的梯度壓下去，等於讓 $G = \sum g_i$ 不再被多數類灌爆，學習力自動集中到困難的（常是稀有正類附近的）樣本 —— 作用點和 Ch 2 的「梯度拔河」是同一個，只是改用「難易」而非「類別」來分配權重。

**實作注意**：

- 你得自己算 focal loss 對 $F$ 的一、二階導數餵給 GBDT，公式比 log-loss 繁瑣。
- focal loss 的**二階導數可能變號（出現負值）**，而 GBDT 的 $\gamma = -G/(H+\lambda)$ 假設 $H > 0$；負 Hessian 會讓葉輸出失去意義，實作上常要把 $h_i$ 截斷在一個小正數下限。
- 多了個超參數 $\gamma$ 要調，且和權重一樣會扭曲機率（落在 Ch 9 (a) 要留意）。

focal loss 是進階選項：當調門檻、權重、重採樣都試過仍不夠，再考慮。多數不平衡問題用不到這一步。

## Ch 14. 選招決策流程

把第二部收成一個「先試什麼、再試什麼」的順序。兩個問題決定路線：**(一)** 你要可信機率值，還是操作點上的好決策（Ch 9）？**(二)** 判錯的成本說得清楚嗎？

```
要操作點上的好決策（Ch 9 (b)）
├─ 成本說得清楚 → 用 τ* = C_FP/(C_FP+C_FN) 設門檻（Ch 10）。先試這個。
└─ 成本說不清楚 → 驗證集上掃 τ、最大化 F_β 或達標 recall/precision（Ch 10）。
   └─ 調門檻後稀有區仍學不出來（Ch 3 飢餓）
      → 加類別權重 w（Ch 11，首選）；資料太大可改欠採（Ch 12）。
      → 仍不足 → focal loss（Ch 13）。

要可信機率值（Ch 9 (a)）
├─ 盡量只調門檻、不動訓練（門檻不扭曲機率）。
└─ 非動梯度不可（飢餓嚴重）→ 用權重 / 欠採，但事後務必用對應的
   prior-shift 把機率還原（Ch 11、Ch 12 的校正式），或另做完整校準。

任何路線都別忘：用對指標看（PR-AUC / F_β，別只看 accuracy / ROC-AUC；Ch 7、Ch 15），
早停也監看同一個指標。
```

順序的原則：**從最便宜、副作用最小的開始**（調門檻 → 權重 / 重採樣 → 自訂 loss），每加一手都先確認上一手不夠用。

## Ch 15. 一個常被忽略的事實

繞了一圈，回到開頭那個框架收尾。

第一部證明了：不平衡的多數症狀 —— 「全被預測成負類」「抓不到稀有正類」—— 根源是**起點低 ＋ 正則化對小樣本葉的合理懷疑**，而不是「機率算錯」。第二部則顯示：最便宜的調門檻（Ch 10）不動模型，而會「修好」模型的權重 / 重採樣（Ch 11、Ch 12）其實是拿**機率校準**換來的。

把這兩件事擺在一起，得到一個常被忽略的結論：

> **如果你只需要操作點上的好決策（Ch 9 (b)）—— 而這是不平衡情境最常見的需求 —— 那麼「調門檻 ＋ 用對指標」往往就夠了，原生 GBDT 不需要任何重採樣 / 加權，而且它的機率還是校準的。**

很多人一遇到不平衡就反射性地上 SMOTE、開 `is_unbalance`，卻沒先問自己 Ch 9 那個問題。結果常是：用一個會扭曲機率的手段，去解一個本來只要平移門檻就能解的問題 —— 換來的「平衡」是假的（機率被弄偏了），代價（方差、過擬合，尤其 Ch 6 的極端區）卻是真的。

當然，重採樣 / 加權 / 自訂 loss 都有真正派上用場的時候（Ch 3 的飢餓嚴重到調門檻救不回來、或你確實落在 Ch 9 (a) 且能配合校準）。重點不是「不要用」，而是**先診斷、分清需求，再對症下藥** —— 這正是本文兩部式結構想傳達的事。

### 一句話總結

> 類別不平衡很少把 GBDT 的機率「算錯」；它把起點壓低、把稀有區的切點餓住、把 0.5 門檻變得不合理。先看清自己要機率還是要決策，多數時候從調門檻下手、需要時再用權重 —— 並且永遠用對的指標檢查。
