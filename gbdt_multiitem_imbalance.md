# 二元分類 GBDT：多 item 共享模型下的冷熱門不平衡

這份手冊接續《[二元分類 GBDT 數學推導手冊](gbdt_binary_classification.md)》（下稱**主手冊**）與《[二元分類 GBDT：類別不平衡的數學影響與處理](gbdt_class_imbalance.md)》（下稱**手冊2**）。主手冊回答「一個訓練好的 GBDT 怎麼算出 $p(x)$」；手冊2 回答「當訓練集正類比例 $\bar y$ 很小時，單一目標的不平衡改變了什麼」。本文把場景再推一步：

> **當同一個 GBDT 二元模型同時服務多個 item，而這些 item 的冷熱門差距懸殊時（換匯每月 4 萬人申購，理財型保單卻只有 100 人），冷熱門差異在訓練的數學運算裡到底改變了什麼？在「一個共享模型」的框架內，又能處理到什麼程度？**

手冊2 把不平衡定位成兩種傷害：便宜的**判讀問題**（門檻錯位）與棘手的**學習問題**（稀有正類在梯度與 Gain 的拔河裡被淹沒）。本文要說的是：當「單一稀有類」變成「多個冷熱門懸殊的 item 共處一張訓練表」，學習問題會**多長出一個維度**——冷門 item 不只在自己內部稀少，還要在同一棵樹裡**和熱門 item 爭奪切點與葉子**。

**這份手冊的範疇（刻意收斂）：**

- 只談**單一共享模型**：一個 LightGBM 二元模型吃下所有 $(\text{客戶}, \text{item})$ 列，item 當成一個類別特徵。冷熱門差異，在數學上就化為「同一訓練集裡各 item 的正類率懸殊」。
- 聚焦**訓練動態**：冷熱門 item 在訓練過程中的行為，以及在共享模型框架內能動的處方。

**明確不談（留待後續手冊）：** 換成 per-item 獨立模型（每個 item 各一個）、換成 learning-to-rank（lambdarank，以 NDCG/mAP 當訓練目標 vs `binary_logloss` 的梯度差異）、完整的排序／top-K 評估、以及機率校準的絕對值問題（沿手冊2 立場，排序／挑正類用途下不展開）。這些都是「換模型形態」或「換目標函數」的故事，與本文「在同一個共享模型內怎麼辦」是不同層次的問題。

### 建模設定（本文的「基準佈局」）

本文假設的是最具代表性的**單一共享 pointwise 模型**（pointwise＝把每一列獨立當一道二元分類題來算 loss，相對於把同一位客戶名下多個 item 成對／成列一起比的排序式損失）：

- 訓練資料是 $(\text{客戶}, \text{item})$ 粒度的列，每列一個二元標籤（這位客戶這個月有沒有買這個 item）；**item 本身是模型的一個類別特徵**。
- 採「完整候選」佈局：每位客戶對每個 item 都產生一列（對應每週對 ~10M 客戶 × 22 產品打分的情境）。於是各 item 的**列數 $n_j$ 大致相等**（≈ 客戶數），冷熱門的差異**主要落在正類率 $\bar y_j$**（購買率），而不是列數。換匯 vs 保單的「4 萬 vs 100」，在這個佈局下是**正類筆數差約 400 倍**，而列數相近。

（實務上常對熱門 item 的負類做欠採，使 $n_j$ 不再相等——那會改變下面的分析，我們留到 Ch 10 再處理。在此之前，請先以「等列數、冷熱門差在 $\bar y_j$」為基準。）

### 符號接續（沿用主手冊與手冊2）

本文不重新定義主手冊／手冊2 已建立的記號（$F, p, \bar y, F_0, g_i, h_i, G, H, G_L/H_L, \gamma, \text{Gain}, \nu, \lambda$），只新增「以 item 分組」的記號：

| 記號 | 意義 |
|---|---|
| $j$ | item 索引，$j = 1, \dots, M$ |
| $n_j$ | item $j$ 在訓練集的列數 |
| $P_j$ | item $j$ 的正類（購買）筆數 |
| $\bar y_j = P_j / n_j$ | item $j$ 的正類率（base rate） |
| $\bar y_{\text{global}}$ | 全訓練集正類率 $= \sum_j P_j \big/ \sum_j n_j$ |
| $F_{0,j} = \log\frac{\bar y_j}{1-\bar y_j}$ | item $j$ 各自的最佳常數起點（對照全域 $F_0$） |
| $G_j, H_j$ | 限制在 item $j$ 的列上的梯度 / 海森總和 |

「熱門 item」指 $P_j$ 大者；「冷門 item」指 $P_j$ 小者。

## Ch 1. 從 1 個 item 到 M 個：訓練集變成「混合體」

手冊2 從頭到尾只有一個正類率 $\bar y$。多 item 一進場，第一件變的事是：**全域的 $\bar y$ 不再是一個單純的數，而是各 item 各自正類率的混合**。

把全訓練集的正類率寫開：

$$\bar y_{\text{global}} = \frac{\sum_j P_j}{\sum_j n_j} = \frac{\sum_j n_j \bar y_j}{\sum_j n_j}$$

這是各 item 正類率 $\bar y_j$ 以**列數 $n_j$ 為權重**的加權平均。在我們的基準佈局裡 $n_j$ 大致相等，於是分母被均攤、**分子 $\sum_j P_j$ 由熱門 item 的正類筆數主導**。

舉本文要一路用到的玩具例子：兩個 item，等列數各 80 列——

- 熱門 H：$P_H = 16$，$\bar y_H = 16/80 = 0.20$
- 冷門 C：$P_C = 2$，$\bar y_C = 2/80 = 0.025$

$$\bar y_{\text{global}} = \frac{16 + 2}{80 + 80} = \frac{18}{160} = 0.1125$$

注意 $0.1125$ 離冷門 C 的真實率 $0.025$ 差了 4.5 倍——而它離熱門 H 的 $0.20$ 反而近得多。**全域 base rate 反映的是熱門 item 的購買傾向，冷門 item 的訊號在全域統計裡幾乎是隱形的。** 真實世界把 $16{:}2$ 換成 $40000{:}100$，這個傾斜只會更極端。

這個現象在推薦系統文獻裡有個名字：**流行度偏差（popularity bias）／長尾（long-tail）**（Abdollahpouri et al., 2017）。熱門 item 佔據了大部分的正向訊號，冷門（長尾）item 的偏好難以被學到。本文要做的，就是把這個偏差**追到 GBDT 的每一個數學量上**，看它具體咬在哪。

## Ch 2. 全域 $F_0$ vs per-item $F_{0,j}$：起點錯位，但這段便宜

主手冊 Ch 3 推過，第一棵樹還沒蓋之前，模型給所有列一個常數起點 $F_0 = \log\frac{\bar y}{1-\bar y}$。在多 item 的共享模型裡，這個 $F_0$ 是**單一的全域常數**——它用的是 $\bar y_{\text{global}}$，所有 item 共用同一個起點：

$$F_0 = \log\frac{\bar y_{\text{global}}}{1 - \bar y_{\text{global}}} = \log\frac{0.1125}{0.8875} = -2.065$$

但每個 item 各自的「最佳起點」並不相同。如果可以替每個 item 設一個專屬常數，它應該是 $F_{0,j} = \log\frac{\bar y_j}{1-\bar y_j}$：

$$F_{0,H} = \log\frac{0.20}{0.80} = -1.386, \qquad F_{0,C} = \log\frac{0.025}{0.975} = -3.664$$

全域起點 $-2.065$ 對兩個 item 都不對：對冷門 C 而言**太高**了（起點機率 $p = 0.1125$，是 C 真實率 $0.025$ 的 4.5 倍——模型一開始就系統性高估冷門 item），對熱門 H 而言又太低。

### 但這段錯位很便宜

起點錯位聽起來嚴重，其實是整個故事裡**最便宜**的一段。

在我們的兩-item 例子裡，樹切一刀「以 item 分組」，就能讓兩個 item 各自落到自己的 base rate：左葉裝冷門 C、輸出一個負的 $\gamma$ 把它往下壓到 $\bar y_C$；右葉裝熱門 H、輸出一個正的 $\gamma$ 把它往上抬到 $\bar y_H$。這一刀**同時修好兩個 item 的先驗**，因為它一致地幫到兩邊各 80 列，Gain 不小（Ch 7 會算出 $\text{Gain}(\lambda{=}1) \approx 5.45$）。

「便宜」要講準一點：$5.45$ 這個值大不大，得看跟誰比。它**遠高於**去切冷門個人化的那一刀（$2.97$）——意思是**樹會優先修先驗、而不是去個人化冷門**；至於它比熱門個人化的那刀（$39.6$）小，是另一回事（熱門個人化更誘人，正是 Ch 3–Ch 4 要講的）。

**那如果有 20、30 個 item，不就要切很多刀？** 對——兩個 item 一刀，$M$ 個 item 要全部各自歸位，大約需要 $M-1$ 刀（一棵 $M$ 葉的樹；LightGBM 的類別切分一刀只把 item 分成兩組，要遞迴切到每組剩一個），對應樹深約 $\log_2 M$（$M = 22$、$30$ 都只要深度 ~5）。但這仍然**便宜**，有三個理由：

1. **維度低**：per-item 先驗總共只有 $M$ 個數字（每個 item 一個 base rate）。$M$ 很小，$M$ 刀就是它的全部自由度。
2. **每刀都高 Gain、又廣泛共享**：每一刀都一致地修好它兩邊**所有列**的先驗（不分客戶），Gain 大、boosting 樂意花預算；何況 base rate 相近的 item 會被歸成一組、共用一個修正，實際需要的刀數往往比 $M-1$ 還少。
3. **可攤在多棵樹上**：boosting 有很多棵樹，這 $M$ 刀不必擠進一棵樹的 `num_leaves`，前幾棵樹就把先驗吸收掉了。

換句話說：**per-item 先驗不是深層問題**——它是一筆 $O(M)$、低維、GBDT 付得起的小帳。$M$ 變大，先驗也只是多 $M$ 刀的小帳。

這個區分不是本文獨有。電腦視覺裡處理極端不平衡的 focal loss（Lin et al., 2017）做的正是同一件事：它也用一個**稀有類先驗初始化** $\pi$（角色等同這裡的 $F_0 / F_{0,j}$）來穩定訓練，但論文明白寫著——先驗初始化**不能解決**不平衡，它只是讓訓練別一開始就爆掉；真正的解，是想辦法讓**大量「易分樣本」不再主導梯度**。本文後面會反覆用到這個「易分樣本／梯度被主導」的框架。

那麼，多 item 共享模型真正難的是什麼？是**先驗修好之後**的事：在每個 item 內部，模型能不能分辨「誰會買」。這就是 Ch 3–Ch 5 的主題。

對照手冊2 Ch 1：手冊2 的痛點是「起點 $F_0$ 太低、稀有正類要爬很長的路」；這裡換成「全域起點對冷門 item 太高、但一刀就修掉」——同樣是起點，但多 item 的版本反而是便宜的那一段。

## Ch 3. 跨 item 的梯度質量：熱門 item 擁有大部分「正類訊號」

主手冊 Ch 4 給過每列帶進樹的一階導數 $g_i = p_i - y_i$。在起點處（所有列 $p_i = \bar y_{\text{global}} = 0.1125$）：

- 正類（$y=1$）：$g = \bar y_{\text{global}} - 1 = -0.8875$（大負值，「想被往上抬」）
- 負類（$y=0$）：$g = \bar y_{\text{global}} = 0.1125$（小正值，「想被往下壓」）

節點層級的 $G = \sum_i g_i$，現在可以按 item 拆成 $G = \sum_j G_j$。樹要學會「分辨誰會買」，靠的是把「想往上抬的正類」從「想往下壓的負類」中分離出來——而推動這件事的力量，主要來自正類那些大負梯度。

關鍵在於：**在起點處**（所有列 $p_i = \bar y_{\text{global}}$），**正類的梯度質量總量** $\approx \big(\sum_j P_j\big) \cdot |\bar y_{\text{global}} - 1|$，被熱門 item 的正類筆數 $P_j$ 主導。本文玩具例裡，熱門 H 有 16 個正類、冷門 C 只有 2 個——熱門 H 貢獻了 $16/18 \approx 89\%$ 的正類梯度質量。真實的 $40000{:}100$ 下，冷門 item 的正類梯度幾乎是捨入誤差。

### 後果（一）：item 切開「之前」，共享切點服務的是熱門

樹在找切點時會評估各種特徵。其中一類是**非 item-specific 的切點**——切在客戶屬性上（年齡、資產、月收入……），跨 item 共用。這種切點的 Gain 由它能分離多少正類梯度質量決定；而正類梯度質量被熱門主導，所以這種切點的**方向**是「服務熱門 item 的買家輪廓」。在 item 還沒被切開的節點（一棵樹的上層，或 item 根本不在特徵裡時），冷門 item 的列只能被動跟著這些熱門導向的切點走。

但這個效應別說過頭。LightGBM 是 leaf-wise 生長，而切出 item 的那一刀 Gain 很高（Ch 2、Ch 7 的 ①），所以樹**往往很早就把 item 切開**；一旦冷門 item 被隔進自己的子樹，子樹裡後續的切點就改用**冷門自己的梯度**評估，不再被熱門的買家輪廓綁架。換句話說，「被熱門服務」主要是 item 切開**之前**的事（這也是 Ch 6 負遷移的同一個源頭，而且同樣會被「早切 item」弱化）。

### 後果（二）：item 切開「之後」，冷門換一種方式繼續吃虧

那 item 切開之後，冷門就沒事了嗎？沒有——問題只是換了形式。冷門子樹雖然能用自己的梯度找切點，但那些切點的 Gain 太小，在**全域的葉預算競爭**裡贏不過熱門子樹的切點。這才是冷門 item 真正、且躲不掉的瓶頸（它不依賴「樹有沒有先切 item」這個前提，所以比後果（一）穩固得多）。下一章專講。

對照手冊2 Ch 2：手冊2 的拔河發生在「單一 item 內部，正 vs 負」；這裡多疊了一層「item 之間，熱 vs 冷」爭奪同一棵樹的切點與葉子。

## Ch 4. 葉預算的競爭：冷門 item 的個人化切點被餓死

先驗便宜地修好了（Ch 2）；item 也被早早切開、冷門進了自己的子樹（Ch 3 後果二）。那麼在冷門 item 自己的子樹裡，要怎麼做**個人化**——分辨出它那少數真的會買的客戶？

需要的是：在「已經被隔出來的冷門 item」這個子區域裡，用客戶特徵把冷門的少數正類切出來。這正是手冊2 Ch 3「稀有區飢餓」的場景——但在多 item 共享模型裡，它變成**雙重稀少**。

第一重，和手冊2 一樣：冷門正類的**絕對筆數**少。切出它們的左葉，$G_L$ 與 $H_L$ 都很小（$H_L$ 小是因為列數少、且每列 $h_i = p_i(1-p_i)$ 在 $p$ 接近 0 時本來就小）。$H_L$ 一小，正則化 $\lambda$ 的相對份量就重，這刀的 Gain 被狠狠打折；葉內樣本數下限 `min_data_in_leaf` 與葉內海森下限 `min_sum_hessian_in_leaf` 更可能直接**禁止**這刀。

第二重，是多 item 才有的。LightGBM 是 **leaf-wise（逐葉）生長**：每要長出一片新葉，它都在「當前所有葉子的所有候選切點」裡挑出**全域 Gain 最大**的那一刀，而總葉數受 `num_leaves` 上限。於是冷門 item 的個人化切點，不只要過 $\lambda$ / min-child 那關，還要在這個全域排行榜上**贏過熱門 item 的個人化切點**——而後者有多得多的正類撐腰。

本文玩具例的數字（Ch 7 完整算）已經很懸殊：

- 熱門 H 的個人化切點：$\text{Gain}(\lambda{=}1) \approx 39.6$
- 冷門 C 的個人化切點：$\text{Gain}(\lambda{=}1) \approx 2.97$

差約 **13 倍**。在 `num_leaves` 有限（樹的葉子是有預算的）的前提下，每多長一片葉子，樹都會挑當下 Gain 最大的那刀——於是葉預算被熱門 item 一路搶走，**冷門 item 的個人化根本排不到隊**。把比例拉到真實的 $400{:}1$，這個差距還會大幅張開。

**這是整本手冊的核心洞見：** 冷門 item 學不好，最深的原因不是它的先驗不對（那便宜），而是它**爭不到樹的葉預算**去做個人化。模型最後對整個冷門 item 給出的，幾乎是一個「人人都差不多等於 base rate」的平坦預測——對「該推薦給誰」這個排序問題毫無幫助。

## Ch 5. 雙重懲罰

把 Ch 3–Ch 4 收成一句話：冷門 item 同時挨兩記。

1. **跨 item 這一層**：它的個人化切點 Gain 輸給熱門 item，搶不到樹的葉預算（Ch 4 的第二重稀少）。
2. **item 內部這一層**：就算它幸運分到一點預算，冷門正類的絕對筆數本來就少，內部仍是手冊2 那套稀有區飢餓——$\lambda$ 與 min-child 把小葉子打折、禁切（Ch 4 的第一重稀少）。

兩記是**遞迴**的：先在跨 item 的競爭裡輸掉入場資格，少數僥倖進場的，又在 item 內部被原本那套不平衡機制吃掉。手冊2 的不平衡是一維的（正 vs 負）；多 item 的不平衡是二維的（熱 vs 冷，再 × 正 vs 負）。

一張對照表，把兩本手冊的傷害定位到同一組數學量上：

| 數學量 | 手冊2（單一目標不平衡） | 手冊3（多 item 冷熱門） |
|---|---|---|
| 起點 $F_0$ | 太低，稀有正類要爬長路（真問題） | 全域起點對冷門太高，但 $O(M)$ 刀 item split 就修好（**便宜**） |
| 梯度 $G$ | 少數正類的負梯度被多數負類稀釋 | 再多一層：正類梯度質量被**熱門 item** 主導 |
| Gain | 稀有正類切點被 $\lambda$/min-child 打折 | 同上 **＋** 還要在全域排行榜上輸給熱門 item 的切點 |
| 葉預算 | （單目標下不顯著） | **冷門 item 搶不到葉子做個人化（核心）** |

這也呼應 focal loss（Lin et al., 2017）的核心診斷：問題不在先驗，在於**大量、易分、佔多數的樣本主導了梯度**，把少數、難分、重要的樣本的訊號淹沒。多 item 共享模型只是把「多數樣本」換成了「熱門 item 的樣本」。

## Ch 6. 共享葉子的跨 item 負遷移

Ch 3 末尾提過：item 被早早切開後，跨 item 的污染就被弱化。但在 item 尚未切開的葉子裡，它仍會以一個有正式名字的形式短暫出現——**負遷移**。

機制是這樣：如果某棵樹先切的是**客戶特徵**（而非 item），那這棵樹的葉子裡會**同時裝著冷門和熱門的列**。葉輸出 $\gamma = -G_{\text{leaf}}/(H_{\text{leaf}}+\lambda)$ 由葉內多數方（熱門）的梯度主導，於是落在這片葉子裡的冷門列，被套上了一個「照熱門買家輪廓推導出來的」調整。在多任務學習裡，這叫**負遷移（negative transfer）**：共享參數被主導任務調校，反而傷到少數任務。它可正可負——冷熱買家若相似，冷門沾光；若不相似，冷門被帶偏。

### 為什麼它多半次要

關鍵在於上面那個「**如果**樹先切客戶特徵」的前提，在實務中經常不成立。LightGBM 處理類別特徵時，是把各類別依其累積的 $\sum g / \sum h$ 排序，再在排序後的序列上找一刀切點（Fisher 1958 證過：依這個比值排序後，最佳的「哪些類別歸一組」必定落在排序上的某一刀，不必窮舉所有分組）。這些是實作細節，不必深究——對我們的目的，只要知道 **item 就是一個照 Gain 跟其他特徵競爭的候選欄**，而且 22 個 item 遠低於 ~1000 的 cardinality 上限、落在原生類別處理的舒適區即可。而我們在 Ch 2 看到，切出 item 的那刀 Gain 很高（玩具例 $\approx 5.45$）。所以 leaf-wise 成長**常常很早就把主要的 item 切出來**了——一旦 item 被隔開，後面的葉子就不再混冷熱，跨 item 的負遷移自然被弱化。

所以負遷移雖然真實、有名字，但在「item 是高 Gain 特徵」的共享模型裡通常是二階的，不必把它說成「校準全面崩壞」。（至於跨 item 的分數可比性、機率絕對值是否準——那是校準問題，沿手冊2 立場本文不展開。）

## Ch 7. 貫穿範例：熱門 + 冷門兩 item 手算

前六章的每一個主張，都濃縮在這個能完全手算的小例子裡。把 Ch 1 的玩具設定攤開：同一張訓練表，兩個 item，**等列數各 80**。

| item | 列數 $n_j$ | 正類 $P_j$ | $\bar y_j$ |
|---|---|---|---|
| 熱門 H | 80 | 16 | 0.200 |
| 冷門 C | 80 | 2 | 0.025 |
| 全體 | 160 | 18 | 0.1125 |

### 起點上的各個量

所有列從同一個全域起點出發，$p = \bar y_{\text{global}} = 0.1125$，於是每列 $h_i = p(1-p) = 0.1125 \times 0.8875 = 0.09984$。各起點常數：

$$F_0 = \log\tfrac{0.1125}{0.8875} = -2.065, \quad F_{0,H} = \log\tfrac{0.2}{0.8} = -1.386, \quad F_{0,C} = \log\tfrac{0.025}{0.975} = -3.664$$

把梯度按 item 加總（正類 $g = -0.8875$、負類 $g = 0.1125$）：

$$G_H = 16(-0.8875) + 64(0.1125) = -7.0, \qquad H_H = 80(0.09984) = 7.987$$
$$G_C = \phantom{1}2(-0.8875) + 78(0.1125) = +7.0, \qquad H_C = 80(0.09984) = 7.987$$

為什麼 $G_H + G_C$ 剛好是 $0$？這不是巧合，是**必然**：$F_0$ 當初就是選來讓「全是常數」的模型最優的，那個最優條件正是整體一階導數為零，即 $\sum_i g_i = 0$（主手冊 Ch 3）。把所有列拆成 H、C 兩塊後，兩塊的梯度和自然必須等值反號，所以 $G_C = -G_H$ **一定**成立——至於它的值剛好是 $7$，才是這組數字決定的。（換成 3 個以上 item，各 $G_j$ 仍加總為 $0$，但不再兩兩反號。）

符號的意義很直白：$G_H = -7 < 0$ 表示熱門 H 整體「想被往上抬」（起點 $0.1125$ 低於它的真實率 $0.20$），$G_C = +7 > 0$ 表示冷門 C「想被往下壓」（起點高於它的 $0.025$）；全域 $F_0$ 卡在加權中點，兩邊剛好抵消。這就是 Ch 2 說的起點錯位。

### 三種切點的 Gain 對比

用主手冊 Ch 5.3 的 Gain 公式 $\text{Gain} = \frac12\big[\frac{G_L^2}{H_L+\lambda} + \frac{G_R^2}{H_R+\lambda} - \frac{G^2}{H+\lambda}\big]$，算三刀：

| 切點 | 左葉 $(G_L, H_L)$ | 右葉 $(G_R, H_R)$ | $\text{Gain}(\lambda{=}0)$ | $\text{Gain}(\lambda{=}1)$ |
|---|---|---|---|---|
| ① **item-isolation**（C \| H） | C: $(7.0,\ 7.987)$ | H: $(-7.0,\ 7.987)$ | $6.13$ | $\mathbf{5.45}$ |
| ② **熱門個人化**（H 內切出 16 正） | $(-14.2,\ 1.597)$ | $(7.2,\ 6.39)$ | $64.1$ | $\mathbf{39.6}$ |
| ③ **冷門個人化**（C 內切出 2 正） | $(-1.775,\ 0.1997)$ | $(8.775,\ 7.788)$ | $9.77$ | $\mathbf{2.97}$ |

先講清楚這三刀**在哪裡**，免得拿來亂比：① 是在 **160 列的根節點**上、把兩個 item 分開的那一刀；②、③ 則是各自在 **H、C 子樹（各 80 列）內**再切一刀做個人化。三者都用「所有列都還停在起點 $p = 0.1125$」這個**同一參考點**評估，所以可以擺在一起比「此刻各自值多少」（樹真正動手後梯度會更新，但這個快照足以看清相對大小）。

三段解讀，正好對應前面三章：

- **① 印證 Ch 2「先驗便宜」**：把兩個 item 隔開、各自修先驗的那刀，$\text{Gain}(\lambda{=}1) = 5.45$。它比熱門個人化 ② 小，但**遠大於**冷門個人化 ③——所以樹會優先修先驗。
- **②≫③ 印證 Ch 4「葉預算競爭」**：熱門個人化 $39.6$ 把冷門個人化 $2.97$ 壓在地上摩擦（差約 13 倍）。樹挑 Gain 最大的切點，葉預算一路流向熱門 item。
- **③ 印證雙重稀少**：冷門個人化的左葉只裝了 **2 列**——遠低於 `min_data_in_leaf` 的預設值（20），這刀**根本不被允許切**；就算放寬列數下限，$H_L = 0.1997$ 也很小，容易再撞上 `min_sum_hessian_in_leaf`。冷門 item 的個人化，連嘗試的資格都沒有。

### 放大到真實量級

這個玩具例的熱:冷正類是 $16{:}2 = 8{:}1$，差距已是 $13$ 倍。真實的換匯:保單約 $400{:}1$，差距會張到什麼程度？把列數一起放大到真實規模（每個 item $n = 10^6$ 列、熱門 $P_H = 40000$、冷門 $P_C = 100$，於是 $\bar y_{\text{global}} \approx 0.020$）實算這兩刀：

| 個人化切點 | $\text{Gain}(\lambda{=}1)$ | 左葉 $H_L$ |
|---|---|---|
| 熱門 | $\approx 976{,}000$ | $\approx 786$ |
| 冷門 | $\approx 1{,}720$ | $\approx 1.96$ |

熱/冷個人化的 Gain 比，從玩具的 $13$ 倍張到 $\approx \mathbf{567}$ 倍。注意一個轉折：冷門這裡有 $100$ 個正類，左葉 $100$ 列、$H_L \approx 1.96$，已經**不會**被預設的 `min_data_in_leaf`（20）或 `min_sum_hessian_in_leaf` 擋掉了——它不是被「禁切」，而是 Gain 太小、在全域排行榜上**徹底排不到隊**。換句話說，item 越冷，先是輸在 Gain 競爭（這裡），再冷到正類只剩個位數時，連 min-child 那關都過不去（玩具例的 ③）——兩道關卡接力把冷門擋在門外。

> **一個算術陷阱（給想自己驗算的讀者）：** 要示範 $400{:}1$，務必把列數 $n_j$ 一起放大，**不能**把 $n = 80$ 的玩具直接硬塞 $P_H = 400$——那會算出 $\bar y_{\text{global}} > 1$ 的荒謬值。

## Ch 8. item-aware sample weight

第一部診斷出冷門 item 真正的病根：它的個人化切點 Gain 太小（Ch 7 的 ③ $= 2.97$），在全域排行榜上贏不過熱門 item（② $= 39.6$），搶不到葉預算。最直接的處方，就是**人為把冷門 item 的梯度放大**，讓它的切點 Gain 重新有競爭力。

機制很乾淨。樣本權重 $w$ 進到 GBDT，就是把該列的 $g_i, h_i$ 同乘 $w$（主手冊的一階／二階量是線性疊加的，所以權重直接縮放它們）。**item-aware** 的意思是：權重隨 item 給，冷門 item 的列乘上一個較大的 $w_j$（取法可以隨 $\bar y_j$ 反向，或正比於 $1/P_j$）。

### 用 Ch 7 的同一組數字再算一次

沿用手冊2 Ch 11 的做法：固定在起點、只把冷門 C 的列乘上權重，重算它的個人化切點 ③。取 $w_C = 8$——這讓冷門 C 的正類質量變成 $2 \times 8 = 16$，與熱門 H 的 16 拉平。

冷門個人化切點 ③ 在 $w_C = 8$ 下（每個 $G, H$ 都乘 8）：

- 左葉（2 個冷門正類）：$G_L = 8 \times (-1.775) = -14.2$，$H_L = 8 \times 0.1997 = 1.598$
- 右葉（78 個冷門負類）：$G_R = 8 \times 8.775 = 70.2$，$H_R = 8 \times 7.788 = 62.30$
- 母節點（整個冷門 C）：$G = 56$，$H = 63.90$

$$\text{Gain}(\lambda{=}1) = \frac12\left[\frac{14.2^2}{1.598+1} + \frac{70.2^2}{62.30+1} - \frac{56^2}{63.90+1}\right] = \frac12[77.6 + 77.8 - 48.3] \approx 53.6$$

冷門個人化的 Gain 從 $2.97$ 跳到 $\mathbf{53.6}$——**反超熱門個人化的 $39.6$**。同樣重要的是，左葉的海森從 $H_L = 0.1997$ 抬到 $1.598$，不再一碰 `min_sum_hessian_in_leaf` 就被禁。冷門 item 終於搶得到下一刀葉預算去做個人化。

釐清一個容易混淆的點：這跟手冊2 Ch 11 的 `scale_pos_weight` 是**不同軸**的加權。`scale_pos_weight` 調的是**類別**權重（把正類相對負類放大，治單一目標**內部**的正負不平衡）；這裡調的是 **item** 權重（把整個冷門 item——它的正類和負類**一起**——相對熱門 item 放大，治的是**跨 item** 的話語權不均）。上面的 $w_C = 8$ 就是純 item 加權：冷門 C 的每一列（不分正負）都乘 $8$。兩軸可以**疊加**——你大可在冷門 item 內部再對它的正類多加一層 class weight——但那是另一個獨立的旋鈕。

### $w_j$ 大概怎麼設？

沒有萬用公式，但有起手式：

- **起點：拉平正類質量**，即 $w_j \propto 1/P_j$（或 $1/\bar y_j$）。上面 $w_C = 8 = 16/2$ 就是讓冷門的正類質量追平熱門。
- **但別用純 $1/P_j$，要阻尼。** 最冷的 item 正類可能只有個位數，純倒數會把它的權重開到天上，等於拿那幾筆（很可能是雜訊）去主導整棵樹。實務上改用**開根號**（$w_j \propto 1/\sqrt{P_j}$）或直接**設一個權重上限**來壓。一個有理論依據的阻尼是 Cui et al. (2019) 的「有效樣本數」：$w_j \propto (1-\beta)\big/(1-\beta^{\,n_j})$，用 $\beta \in [0,1)$ 控制阻尼強度（$\beta \to 0$ 退化成不加權，$\beta \to 1$ 趨近純 $1/n_j$）。
- **最後一定要在驗證集上調，而且看對的指標。** 看 per-item 或 macro 平均的 recall / AP（每個 item 等權），而不是被熱門 item 灌水的整體 accuracy。權重開多大，沒辦法純推導，要量。

### 兩條誠實提醒

- **放大訊號的同時也放大雜訊。** 冷門 C 只有 2 個正類，把它們的梯度乘 8，等於讓這 2 筆樣本（可能含標註雜訊、可能是巧合）對樹的影響力暴增。權重太猛，模型會去硬背冷門 item 的個別樣本（過擬合），而不是學到可推廣的規律。$w_j$ 是一個要小心調的旋鈕，不是越大越好。
- **加權會讓輸出機率系統性偏高。** 一旦對冷門 item 加權，模型在那些 item 上吐出的 $p$ 會偏離真實購買率（偏高）。在「排序、挑高分客戶」的用途下這無妨——模型仍然**保序**（每位客戶名下各 item 的高低分次序不變），排序不受影響。但如果你需要機率的絕對值，就得另做校準；沿手冊2 的立場，校準是獨立主題，本文不展開。

## Ch 9. 全域正則化旋鈕的多 item 困境

Ch 8 的權重能把冷門 item 的 Gain 抬起來，但還有一組旋鈕站在另一邊扯後腿：$\lambda$、`min_data_in_leaf`、`min_sum_hessian_in_leaf`。手冊2 Ch 3 已經說過它們怎麼餓死稀有正類。多 item 共享模型給這件事加了一個新的扭結：**這些旋鈕是全域的，沒辦法 per-item 設定。**

整棵樹只有一組 $\lambda$、一個 `min_data_in_leaf`。但冷門和熱門 item 要的設定**剛好相反**：

- 冷門 item 想要**寬鬆**的限制：它的個人化葉子本來就小（少數正類），$\lambda$ 小、min-child 低，這些小葉子才活得下來。
- 熱門 item 想要**嚴格**的限制：它的列多、訊號強，太寬鬆的限制會讓它過擬合一堆雜訊切點。

於是你被夾在中間：

- 把限制**放鬆**到足以讓冷門個人化切點存活 → 熱門 item 開始長出一堆過擬合的葉子。
- 把限制**收緊**到能壓住熱門 item 的過擬合 → 冷門 item 又被餓回去。

用 Ch 7 的數字看更具體：同樣把 $\lambda$ 從 $0$ 加到 $1$，熱門個人化切點 ② 的 Gain 從 $64.1$ 掉到 $39.6$（剩約 $62\%$），冷門個人化切點 ③ 卻從 $9.77$ 掉到 $2.97$（**只剩約 $30\%$**）。同一個 $\lambda$ 對冷門那刀的相對打擊重得多——因為它的葉子小、$H$ 小，$\lambda$ 在分母 $H+\lambda$ 裡的相對份量更大（手冊2 Ch 3 的老問題）。你沒辦法用單一全域 $\lambda$ 同時剛好服侍冷熱兩邊。

手冊2 裡這只是「一個」目標的取捨；多 item 把它變成**一個全域設定要同時服侍冷熱兩種互斥需求**的困境。這也是為什麼 Ch 8 的 item-aware 權重往往比硬調全域 $\lambda$ 更對症——權重能**分 item** 給，正則化旋鈕不能。

## Ch 10. per-item 負採樣 / 列比例再平衡

還有一條路是從**資料的列比例**下手，而不是動梯度權重。Ch 3 指出共享切點被熱門 item 綁架，根源是熱門 item 在 $\sum_j P_j$、$\sum_j n_j$ 裡的份量太大。那就**對熱門 item 的負類欠採**，把它的列數壓下來，讓各 item 在訓練表裡的份量更平均——共享切點就不再一面倒地服務熱門買家輪廓。

### 它確實降低熱門的主導——用數字看

這裡有個值得正面回答的直覺擔心：欠採熱門的負類，不是讓熱門的正類「更不被稀釋、更容易被切乾淨」嗎？那豈不是反而**幫了**熱門？

關鍵要分清「切得乾不乾淨」和「Gain 大不大」是兩回事——**Gain 衡量的是總 loss 下降量，會隨資料量縮水**。把熱門的負類從 $64$ 欠採到 $16$（玩具例、其他不動），重算熱門個人化切點：它的 $\text{Gain}(\lambda{=}1)$ 從 $39.6$ **掉到 $18.5$**，而冷門幾乎不動（$2.97 \to 2.93$）。於是熱對冷的主導比從 $13.3$ 倍縮到 $\mathbf{6.3}$ 倍——熱門的葉預算優勢被削掉一半。切得更乾淨是真的，但**能降的 loss 變少了**，所以 Gain 反而下降，這正是我們要的。

**注意這個槓桿是不對稱的：只欠採熱門，別動冷門。** 冷門 item 本來就 data-starved，對它的負類也欠採，只會讓它的列更少、Gain 更低，雪上加霜。要平衡跨 item 的份量，方向是**把過多的熱門壓下來**，而不是把稀少的冷門也一起砍。

### 代價：動了 base rate，要把它修回來

欠採是有副作用的：它**改變了各 item 的 $\bar y_j$ 與全域 base rate**。模型在被採樣過的資料上學到的 $F$，對應的是**採樣後**的機率，不是真實世界的購買率——輸出分數會系統性偏移。（也別誤會它能順手修好先驗：只欠採熱門負類其實會**抬高**全域 $\bar y$、把冷門的起點推得**更偏**；但先驗本來就便宜（Ch 2），加上下面的 logQ 兜底，這點不礙事。）

標準的修法叫 **logQ 修正**（或抽樣機率修正）：對每個被採樣的 item，在分數（log-odds）上加回一個位移——若負類保留比例為 $r$（$r<1$），位移量約為 $\log r$（負值，把因欠採而被抬高的分數壓回）——就能把機率拉回真實尺度。概念上，這就是手冊2 Ch 12「重採樣會搬動 base rate、要把 $F_0$ 補回來」的**跨 item 推廣**——只是現在每個 item 的採樣比例可能不同，補正也要分 item 做。

要分清楚這跟手冊2 重採樣的差別：手冊2 調的是**單一 item 內部的正負比**；這裡調的是**跨 item 的列比例**（哪個 item 在訓練表裡佔多少列）。前者治「正類太少」，後者治「熱門 item 佔太多」。

## Ch 11. 選招流程 + 收尾

把第二部的處方串成一條可操作的流程：

1. **先確認 item 是不是模型的特徵。** 如果 item 根本沒進特徵，模型連「以 item 分組修先驗」（Ch 2 那刀便宜的 ①）都做不到，後面全部免談。先把 item 放進去。
2. **先驗便宜，先讓樹自己修。** item 進了特徵，全域起點錯位（Ch 2）大致會被早期的 item-isolation 切點吸收掉，不必特別處理。
3. **量一下冷門 item 是不是被餓死了。** 看冷門 item 的預測是不是一片平坦（人人約等於 base rate）、或它的個人化切點 Gain 是不是遠輸熱門（Ch 4／Ch 7 的 ②≫③）。是，才需要動下面的處方。
4. **首選 item-aware 權重（Ch 8）。** 它能**分 item** 給力道，直接把冷門個人化的 Gain 抬到有競爭力，且不像全域正則化那樣牽一髮動全身。注意別把權重開太猛（過擬合冷門少數正類）。
5. **正則化旋鈕當輔助（Ch 9）。** 知道它是全域的、冷熱需求互斥；通常不靠它當主力，而是配合權重微調。
6. **資料份量太偏時，再考慮列比例再平衡（Ch 10）。** 對熱門 item 欠採負類，記得做 logQ 修正把分數拉回真實尺度。

### 加權與欠採會打架嗎？

Ch 8（對冷門加權）和 Ch 10（對熱門欠採）其實在做同一件事——抬高冷門相對熱門的 Gain——所以它們**部分冗餘**。事實上，把熱門負類的保留比例降到 $r$，效果**約等於**把那些被丟掉的熱門列加權成 $0$；反過來，對冷門加權 $w$ 也約等於把冷門「複製」$w$ 份。兩個一起開，很容易**過頭**：冷門的相對 Gain 被抬太高，樹開始硬背它那寥寥幾筆正類（Ch 8 的過擬合風險加倍），熱門則被削到學不好。

幾個務實原則：

- **擇一當主力。** 通常選 item 加權（Ch 8），因為它不丟資料、旋鈕連續好調；欠採（Ch 10）多半是為了**算力**（列數太多）順帶做的，不必兩個都開到滿。
- **若兩個都用**，欠採記得配 logQ 修正（Ch 10），而且兩者的力道要**一起**在 macro / per-item 驗證指標上調，不要各調各的——否則你會對同一個失衡重複補償。
- **盯著過擬合訊號**：冷門 item 的訓練指標漂亮、驗證指標卻崩，就是補過頭了，把力道收回來。

### 收尾：能緩解，但別期待根治

這些手段都在**同一個共享模型內**動手腳，它們能把冷門 item 的處境**緩解**到可用——但要老實說：當冷門 item 的**絕對正類筆數**低到一個程度（保單那 100 人放進 1000 萬列裡），單一共享模型有它的天花板。權重再大，也只是放大那寥寥幾筆樣本，變不出本來不存在的訊號；這時答案往往不在「再調一個旋鈕」，而在資料層（更長的觀察窗、更好的正類定義、外部訊號）或更上層的結構。

冷門 item 的偏好天生難學，這是 cold-start／popularity bias 的固有難處（Meehan & Goyal, 2025），不是調參能無中生有的。誠實地知道天花板在哪，比硬把一個指標調漂亮更重要。

### 後續跟進

本文刻意只談「單一共享模型內怎麼辦」。真正可能突破天花板的，是**換結構**或**換目標函數**，那是另一份手冊的事：

- **per-item 獨立模型**：每個 item 各訓一個模型，徹底消除跨 item 競爭——代價是冷門 item 的樣本更碎、失去跨 item 的訊號共享，以及 22 倍的訓練與維運成本。
- **Learning-to-rank（lambdarank）**：把目標從 pointwise 的 `binary_logloss` 換成**按客戶分組**的排序損失，並用 **NDCG / mAP 當訓練目標**。其梯度不再是 $g_i = p_i - y_i$，而是按「交換一對樣本造成的指標變化」加權——這個「按 query 正規化」本身就是一種冷熱門再平衡。它與 `binary_logloss` 的梯度差異，留待下一份手冊細推。
- **跨 item 排序與校準**：top-K 推薦要跨 item 比分數，分數可比性與校準會浮上檯面。
