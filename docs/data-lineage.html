recsys_tfb — Data Lineage

catalog.yaml 全部資料產物的「誰產生、誰消費、長什麼樣」。邊（資料流）逐一對照各 pipeline.py 的節點接線；schema 與型別對照 catalog.yaml / 節點程式碼 / 實際 parquet。範例列：來源三表取自實際合成資料，其餘為依 schema 構造的示意列。

recsys_tfb 是通用排序框架，商業銀行產品推薦只是本 repo 的示例。feature_table / sample_pool / label_table 是使用者自定義的來源表——框架只規範 schema 的角色欄（time / entity / item / label），其餘特徵 / 分群 / carry 欄由應用自定。因此下方表卡 只列角色欄，user 欄位一律以抽象 descriptor 與 placeholder（prod_A、f1…fN、 <enc>）示意，不列舉銀行專屬欄位。示例規模 ~10M 客戶 × 22 產品 × ~1500 特徵 （本 repo 合成版約 1.5 萬客戶月、8 個 item 值）。

Lineage 總覽

階段由上而下：source_etl → dataset → training →（分叉）inference / evaluation。 每個叢集列出它的 catalog 產物（底色 = 版本層）。叢集之間標註實際流動的資料表； 叢集內的逐表上下游見下方各表卡的「產生 / 下游」。

※ label_table 同時餵 dataset（組 model_input）與 evaluation（算指標）； feature_table 同時餵 dataset（前處理）與 inference（組評分母體）。為避免長跨線，這兩條共用流向見各表卡。

各表表卡

source_etl： feature_table label_table sample_pool dataset（base）： preprocessor category_mappings preprocessed_feature_table val_keys / test_keys val_model_input / test_model_input dataset（train-variant）： sample_keys train_keys / train_dev_keys train_model_input / train_dev_model_input dataset（calibration-variant）： calibration_keys calibration_model_input training： model best_params training_eval_predictions evaluation_results 診斷 JSON ×3 inference： score_table ranked_staging / ranked_predictions evaluation： evaluation_report enriched_eval_predictions evaluation_comparison_report

dataset — base_dataset_version 層

preprocessor JSONbase

產生：dataset · fit_preprocessor_metadata（在 train 期 feature_table 上 fit，與抽樣解耦）　| 下游：dataset（apply_preprocessor_to_features、build_model_input）、training（prepare_lgb_train_inputs、tune、finalize、calibrate、predict、診斷）、inference（apply_preprocessor）　| path：data/dataset/<base_dataset_version>/preprocessor.json

類別欄編碼字典與前處理 metadata（transform 時用）。是跨 dataset / training / inference 共用的前處理合約。

{ "categorical_columns": [<item>, <cat_col>, ...], "category_to_code": { "<cat_col>": {"<v1>":0,"<v2>":1}, ... }, "feature_columns": [...], ... }

category_mappings JSONbase

產生：dataset · fit_preprocessor_metadata（與 preprocessor 同節點同時產出）　| 下游：—（人類可讀的對照 / 檢視用；DAG 下游節點只吃 preprocessor）　| path：data/dataset/<base_dataset_version>/category_mappings.json
{ "<cat_col>": {"<v1>":0,"<v2>":1}, "prod_name": {"prod_A":0,"prod_B":1, ... } } ← item 值由 schema.categorical_values 宣告

train_keys · train_dev_keys Hivetrain-variant

產生：dataset · split_train_keys（把 sample_keys 依 cust_id hash 以 train_dev_ratio 切兩份）　| 下游：dataset（build_train_model_input / build_train_dev_model_input）　|　partition：snap_date

training — model_version 層（driver-local 產物 + 1 張 Hive）

model 模型model_version

產生：training · finalize_model（→ 校準時再經 calibrate_model 包裝）　| 下游：training（predict_and_write_test_predictions、compute_feature_importance、compute_shap_diagnostics、log_experiment）、inference（predict_scores）　| path：data/models/<model_version>/model.txt（driver-local，Python open() 寫，不認 hdfs://）

單一共用 LightGBM booster（pointwise binary 或 LTR lambdarank/rank_xendcg，由 parameters_training.yaml 決定）。校準開啟時 model 是校準後包裝（calibrate_model 的輸出仍命名 model）。

best_params JSONmodel_version

產生：training · tune_hyperparameters（Optuna 在 val 上挑出的最佳超參）　| 下游：training（finalize_model 用來訓最終模型、log_experiment 記錄）　| path：data/models/<model_version>/best_params.json
{ "num_leaves": 63, "learning_rate": 0.043, "min_child_samples": 120, "feature_fraction": 0.8, ... } （示意）

training_eval_predictions Hivemodel_version

產生：training · predict_and_write_test_predictions（對 test set 評分後 chunked 寫 Hive；節點宣告產出是 predict_manifest）　| 下游：training（compute_test_mAP_spark 讀回算 test mAP）、evaluation 情境 1（prepare_eval_data，--post-training）　| partition：snap_date, prod_name（外加 model_version）

只有 test 窗內至少 1 個正例的客戶會被 scoring。不存 rank——Spark mAP 端用 rank_within_query 重算。

evaluation_results JSONmodel_version

feature_statistics · feature_importance · shap_diagnostics JSONmodel_version

產生：training · compute_feature_statistics（讀 train 快取）/ compute_feature_importance（讀 model）/ compute_shap_diagnostics（讀 model + test 快取）　| 下游：training（log_experiment → MLflow）　| path：data/models/<model_version>/diagnostics/*.json

訓練診斷：特徵統計、LightGBM 原生 importance、SHAP。皆為 model_version 層的旁路產物，不影響下游 inference / evaluation。

ranked_predictions Hivemodel_version （rank 後的 production 表；發布前 staging = ranked_staging）

產生：inference · rank_predictions → ranked_staging → validate_predictions（sanity checks）→ publish_predictions　| 下游：evaluation 情境 2（prepare_eval_data，預設監控）　| partition：snap_date, prod_name, model_version

production 由 publish_predictions 寫入，且只在 validate_predictions 通過後才執行：rank_predictions 先寫 ranked_staging，validate 讀它跑 sanity check，失敗即中止、production 不被寫（失敗批次留在 staging 供排查）。ranked_predictions 顯式宣告讓 evaluation 不跑 inference 也能 standalone 讀，避免 MemoryDataset fallback 靜默壞掉。

欄位型別說明
cust_idstringentity
scoredoublescore
rankbigintrank（query group 內名次）
snap_date=2026-02-28 / prod_name=prod_A / model_version=<mv>（partition） cust_id score rank C000010 0.83 1 （示意）

evaluation — model_version 層

evaluation_report 報表model_version

產生：evaluation · generate_report（吃 eval_predictions + evaluation_metrics + baseline_metrics）　| 下游：—（終端產物，給人看）　| path：data/evaluation/<model_version>/<snap_date>/report.html

eval_predictions 由 prepare_eval_data 把預測（情境 1 = training_eval_predictions；情境 2 = ranked_predictions）⋈ label_table 而來。evaluation_metrics / baseline_metrics 是 in-memory 中間產物，不持久化於 catalog。

enriched_eval_predictions Hivemodel_version

產生：evaluation · persist_eval_predictions（把 label join + rank + segment enrichment 後的 eval_predictions 寫回 Hive）　| 下游：evaluation（--compare-only 由 catalog auto-load 讀回，B4 validator 檢查 partition）　| partition：snap_date（外加 model_version）；columns: auto（schema 從 DataFrame 推得）

若 segment_sources 在兩次 run 之間改變，需先 drop 此表再重跑（ALTER TABLE schema evolution 是待辦 follow-up）。

evaluation_comparison_report 報表model_version

產生：evaluation · generate_comparison_report（--compare / --compare-only 模式才有）　| 下游：—（終端產物）　| path：data/evaluation/<model_version>/<snap_date>/report_comparison.html

把本模型與 compare_sources（另一 model_version 或 external_hive）在共同客戶上對比。--compare-only 走短 pipeline：讀回 enriched_eval_predictions → restrict_to_common → 只產此報表。

未列為 catalog 產物的中間物（MemoryDataset / driver-local handle）：scoring_dataset · X_score · eval_predictions · evaluation_metrics · baseline_metrics · predict_manifest · *_parquet_handle · *_lgb_handle · sample_weight_report · *_model_input_unfiltered。這些只在單次 DAG 內銜接，不持久化。
