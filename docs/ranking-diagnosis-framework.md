> 2026-07-06。這份文件回答一個問題：**一個用 pointwise 二元分類 GBDT 做 per-query 排序的模型，當主指標（macro per-item mAP）不理想時，該量哪些東西、怎麼判讀、然後動哪個訓練槓桿？**
>
> 它是一套**診斷框架**（方法論），不是工具規格、也不綁定任何 pipeline 實作。內容整併自 `docs/handbooks/gbdt/` 四冊手冊的分析、以及一輪方向診斷（`docs/notes/2026-07-05-ranking-diagnosis-direction.md`）的結論。

## Ch 1. 問題設定與指標：先把「好」定義清楚

### 1.1 問題形狀

- 訓練資料是 (query, item) 粒度的列；query ＝ entity × time（例如客戶×月份）。
- **候選數少**：每個 query 內約 10–100 個 item（本文的示例情境是 22 個銀行產品）。這個「少」是框架的重要前提——它讓 Ch 3 好幾個診斷項目（逐 item 替換、全 item 配對）在計算上變得可行。
- 模型是**單一共享的 pointwise 二元 GBDT**（LightGBM，`binary_logloss`），item id 是一個類別特徵。推論時每個 query 內按分數排名。
- item 冷熱懸殊：熱門 item 正類數 ~40000、最冷 ~100（約 400:1），query 數每期 ~10M。

**「完美模型」的操作性定義**：對每位客戶，把他實際申辦的產品全部排到最前面——只申辦一個的排第 1；同時申辦三個的，那三個佔據前 3 名。下面的指標就是這句話的數學化。

### 1.2 主指標：macro per-item mAP

分三步：

1. 對某筆正例（客戶 $q$ 這期真的買了 item $j$），看它在**自己 query 內**的名次 $k$，取 precision@$k$ ＝「該 query 前 $k$ 名中正例的比例」。
2. item $j$ 的 AP ＝ 對它**所有正例列**把 precision 取平均。
3. macro mAP ＝ 對 $M$ 個 item 的 AP **等權**平均。

驗算「完美模型 ⇒ 指標＝1」：客戶買了 3 個產品、模型把它們排在前 3 名，三筆正例的 precision 分別是 1/1、2/2、3/3，全是 1 ✓。注意第 2、3 名的分子把上方的正例也算進去——**排在你上面的若是真正例，不扣你的分**。這個性質在 Ch 2 是關鍵。

指標定義的兩個直接推論，後面反覆使用：

- **沒有任何正例的 query 完全不影響指標。** 所以任何「某 item 在多少 query 裡如何」的論證，都必須在「相關 item 有正例的 query」上條件化，不能用全體 query 的邊際比例（Ch 4.3 的陷阱三）。
- **每筆正例在指標裡的權重是 $\frac{1}{M \cdot P_j}$。** 以 $M=22$：最冷 item（$P_j\approx100$）一筆正例的權重是最熱（$P_j\approx40000$）的 **400 倍**。這是「macro 等權」的具體意思，也是 Ch 6 整章要討論的東西。

### 1.3 診斷的輸入需求

| 輸入 | 用途 | 沒有它會怎樣 |
|---|---|---|
| model.txt | 結構統計（Ch 3 的診斷項目 8） | — |
| **同分布、有標籤的代表性樣本** | 幾乎所有診斷項目 | 兩個病灶軸都量不到——它們是「模型×母體×標籤」三者合起來的性質，model.txt 只有模型【`docs/notes/2026-07-05-ranking-diagnosis-direction.md` §5.2】 |
| 訓練配置（各 item 的負採樣保留率 $r_j$、sample_weight $w_j$） | 診斷項目 1（配置對帳） | 會把自己配置造成的偏移當成模型病灶研究 |

### 1.4 貫穿範例

熱門 item $H$（換匯型）、冷門 item $C$（保單型）、陪襯 $X$，角色沿用手冊3/4。

- **客戶 Alice**：買了 $C$，沒買 $H$、$X$。分數 $s_H=2.0$、$s_C=1.5$、$s_X=0.0$ → 排序 $H$@1、$C$@2。她那筆 $C$ 正例的 precision@2 ＝ 1/2 ＝ **0.5**。
- **客戶 Bob**：$H$、$C$ 都買了。同樣分數下 $H$@1（正例）、$C$@2（正例），$C$ 那筆的 precision@2 ＝ 2/2 ＝ **1.0**。

兩人只差一件事：壓在 $C$ 上方的 $H$ 是不是該 query 的正例。Alice 受害、Bob 無損。

## Ch 2. 框架骨幹：兩個病灶軸與 2×2 象限

整個框架建立在一個分解上：把 item $j$ 的分數粗分成

$$s(q,j) \;=\; \underbrace{c_q}_{\text{query 效應}} \;+\; \underbrace{\delta_j}_{\text{item 水準（level）}} \;+\; \underbrace{\varepsilon(q,j)}_{\text{條件部分（隨 context 變動）}}$$

query 內名次由分數**差**決定，所以 $c_q$ 對指標完全沒有影響（同 query 內全加同一個數，次序不變）；$\delta_j$ 與 $\varepsilon$ 全額影響。由此得到兩個互相獨立的病灶軸：

- **水準軸**：item $j$ 的平均高度 $\delta_j$ 相對「它該有的高度」偏了——偏高的 item 在每個 query 佔前排、壓別人（它自己反而得利）；偏低的 item 自己吃虧。
- **條件判別力軸**：$\varepsilon(q,j)$ 該在「真的會買的客戶」那裡升高，卻沒有——模型分不出這個 item 該推給誰。

**這個分解當「指標上的操作」是精確的**：對推論分數加 per-item 常數就是在動 $\delta_j$，零訓練成本、且完全不動 $\varepsilon$。**當「模型內部結構」則只是投影**——GBDT 學到的 $F$ 有 item × context 交互，$\delta_j$ 依賴取平均的母體【推論·信心高】。所以本框架用它組織診斷與量測，但不把兩個成分當成模型裡真實存在的零件去做特徵歸因。

### 2×2 象限：每個 item 落在哪

| | **條件判別力好** | **條件判別力差** |
|---|---|---|
| **水準對** | 健康 | **典型冷門受害者**：先驗有學到、個人化沒學到（手冊3 Ch4 葉預算餓死的預測輪廓）。自己的 AP 差。 |
| **水準偏**（偏高或偏低） | 偏高＝**加害者**：自己 AP 好、把別人壓下去；偏低＝自己的正例被整體壓後 | 最糟象限；先修水準（便宜），修完重量再判斷判別力 |

用貫穿範例讀這張表：一個分數**恆為常數 2.5** 的 item（條件判別力＝零的極端），在 Alice 那種 query（其他人最高 2.0）永遠排第 1——**它自己的 AP 接近滿分**，同時把所有人壓一名（右下往右上跑的加害者）；同樣「恆為常數」但常數是 0.5 的 item，正例永遠墊底、AP 爛（右下受害者）。**同樣的「不可分」，指標一好一壞，差別全在水準**——這就是為什麼兩軸必須一起看、且「分佈重疊」單獨不能當診斷結論（Ch 4 判讀準則的核心）。

兩個原始現象對應到軸上：「某些 item 分數整體偏高、壓低其他 item」＝水準軸的問題；「某 item 正負分數重疊、自己 AP 低」＝條件判別力軸的問題。**它們部分同根因**：共同上游都是冷熱懸殊（$P_j$ 量級差），但中介不同——水準走「base rate 先驗＋採樣/加權扭曲」這條**便宜、幾乎保證會被學到**的路【手冊3 Ch2】；條件判別力走「葉預算競爭」這條**貴、對冷門幾乎保證堵住**的路【手冊3 Ch4】。一個機制講不了兩邊，所以**處方必須分開開**。

## Ch 3. 關鍵診斷項目清單（量什麼）

十個診斷項目，按「便宜且該先做」到「貴且針對性強」排。每項給：量什麼、需要什麼、量出來接哪條判讀（Ch 4）。

### 對帳層（先做，成本最低、確定性最高）

**診斷項目 1：配置對帳。**
把現行訓練配置換算成**理論上必然造成的分數偏移**，跟實測比對。核心公式：對 item $j$ 的負類欠採、保留率 $r_j$，未修正時該 item 的 log-odds **必然**偏高 $-\log r_j$【手冊3 Ch10 的推導】。數字感：$r_j = 0.1$ → 偏高 $\ln 10 \approx 2.3$——比貫穿範例裡 $H$、$C$ 的分數差（0.5）大好幾倍，足以徹底重排每個 query。sample_weight $w_j$ 造成同方向偏移（手冊3 Ch8 的定性提醒；精確偏移量依權重作用面推算）。
*需要*：訓練配置。*接*：判讀 1。

**診斷項目 2：per-item 校準差距。**
在有標籤樣本上按 item 分組，比「模型平均預測機率」vs「實際購買率」，**在 log-odds 尺度上**看差（機率尺度在 0 附近被壓縮、差不好比；log-odds 上偏移是可加常數）。例：$H$ 實際購買率 0.4%、模型平均預測 2% → odds 差約 5 倍 → 偏高 $\ln 5 \approx 1.6$。按組別驗校準的一般化概念是 multicalibration【文獻：Hébert-Johnson et al., 2018】。
*需要*：有標籤樣本。*接*：判讀 1、2。

### 行為層（框架的主體）

**診斷項目 3：item 內 ROC-AUC。**
只取 item $j$ 自己的列（跨所有 query），算「隨機一正例、一負例，正例分數較高」的機率。它**從不跨 item 比較**，per-item 常數偏移被整個消掉——所以它是**條件判別力軸的專用儀表、對水準軸完全免疫**。跨 item 互比要用它而不是 AUC-PR（AUC-PR 的隨機基線 ≈ 正類率，冷熱 item 差 100 倍、天生不可比【文獻：Saito & Rehmsmeier, 2015；Davis & Goadrich, 2006】）。
*需要*：有標籤樣本。*接*：判讀 2、3。

**診斷項目 4：per-item AP 與信賴區間。**
主指標的分項，但**必須附信賴區間才可讀**：最冷 item 的 AP 由 ~100 筆正例估出，幾筆雜訊就能搬動它。區間用**以 entity 為群組單位**的 bootstrap——同一客戶跨期的列高度相關，按列重抽會嚴重低估變異【推論·信心高；群組化 bootstrap 是標準統計手法，「此處必須群組化」是本文判斷】。
*需要*：有標籤樣本。*接*：所有判讀的顯著性門檻。

**診斷項目 5：名次佔據統計。**
對每個 item 算：佔據 top-1（或 top-$k$）的 query 比例 vs 它的正類率；以及「以**負例**身分壓在其他 item 正例上方」的次數。這是水準軸傷害的**直接觀測**——比看邊際分數直方圖可靠（Ch 4 陷阱二會演示直方圖怎麼抓錯犯人）。
*需要*：有標籤樣本。*接*：判讀 1、4。

**診斷項目 6：offset sweep（per-item 常數偏移搜尋）。**
對推論分數加 per-item 常數 $\{\delta_1,\dots,\delta_M\}$，在驗證集上調它們讓 macro mAP 最大（$M$ 小，用座標下降——一次動一個 $\delta_j$、對它做一維掃描、輪流迭代——即可；注意各 $\delta_j$ 之間互相牽動名次，不能各自獨立調完就當最優）。讀法是本框架的樞紐：**offset 能收復的缺口＝水準問題（不必重訓就能修）；收不回的部分＝只能來自條件判別力（必須動訓練）**。理由：加常數只動 $\delta_j$、完全不動 $\varepsilon$（Ch 2）。同時它給出水準傷害的量：$\text{mAP}(\delta^*)-\text{mAP}(0)$。
注意過擬合：冷門 item 的 $\delta_j$ 只由 ~100 筆正例的訊號估出，要向 0 收縮（正則化）或用折外資料挑，並連同診斷項目 4 的區間一起讀【推論·信心高】。
*需要*：有標籤樣本。*接*：判讀 2（整個框架的分流閥）。

**診斷項目 7：替換實驗（ablation）與成對帳本。**
兩個互補的傷害歸因工具：
(a) **逐 item 替換**：把 item $j$ 的分數換成一個校準常數（它的 base rate 對應的 log-odds）、重算指標——變化量＝$j$ 的「個性化分數」對全局的淨貢獻；把「$j$ 以外所有人」換成基準——量 $j$ 自己被競爭壓掉多少。候選數少讓這件事只要 $O(M)$ 次指標評估，完全可行。
(b) **成對帳本**：對驗證集每一對「排錯的 $(i,j)$」記 $|\Delta\text{AP}_{ij}|$（交換這對名次會讓指標變多少）——這就是 lambdarank 的 λ 梯度定義【文獻：Burges, 2010】，這裡只做會計、不訓練。產出「誰壓了誰、代價多少」的帳。
*需要*：有標籤樣本。*接*：判讀 4。

**診斷項目 8（結構層）：Gain 帳本。**
從 model.txt 跨樹聚合、按 item 記帳：每個 item 被 item-id 切點隔出後，子樹內還有多少用 context 特徵的切點、累積多少 Gain。冷門 item 若隔出來之後**幾乎沒有後續切點**，就是「先驗有修、個人化沒學」（葉預算餓死）的結構鐵證【手冊3 Ch4】。注意：這是**唯一**只需 model.txt 的診斷項目；也注意它的反面——**逐棵讀樹不是有效診斷**，單棵樹只反映訓練路徑的偶然，同一個 $F$ 可由無數種樹序列組成【推論·信心高】。
*需要*：model.txt。*接*：判讀 3。

**診斷項目 9：條件化 SHAP。**
對「item ＝ $j$ 的子母體」做 TreeSHAP，背景樣本也取自該子母體（interventional 模式），必要時對 loss 而非分數歸因（`model_output='log_loss'`，需標籤）。回答：「在這個 item 內部，哪個 context 特徵有／沒有把正負分開」。兩個注意：(a) interventional 需要你提供背景樣本；`tree_path_dependent` 雖只需 model.txt，但背景固定是訓練分佈的路徑摘要、且對相關特徵會抹散貢獻【文獻：Lundberg et al., 2020；Chen et al., 2020】。(b) **不要用全域 importance 回答 per-item 問題**——gain importance 有不一致問題、split 次數偏向連續/高基數特徵【文獻：Lundberg et al., 2018；Strobl et al., 2007】。
*需要*：model.txt＋有標籤樣本。*接*：判讀 3。

**診斷項目 10：交叉購買率。**
對 item 對 $(j,k)$ 估 $P(\text{買 } k \mid \text{買 } j)$。用途：水準傷害的折扣因子——壓在 $j$ 正例上方的 $k$ 若在該 query 也是正例，不造成傷害（Bob 效應，Ch 1.4）。相關母體是「$j$ 的買家」，**不是全體客戶**——買家是活躍客群，交叉購買率幾乎必然高於 $k$ 的全體正類率（0.4% 之類的邊際數字在這裡是錯的條件化）【推論·信心高】。
*需要*：有標籤樣本。*接*：判讀 4 的傷害修正。

## Ch 4. 判讀準則（怎麼判斷觀察到的現象）

### 4.1 判讀流程（按順序走，每步有進出條件）

**第 0 步：建評估基座。** 有標籤代表樣本＋主指標計算＋診斷項目 4 的信賴區間。沒有區間就沒有「顯著」可言，後面全部免談。

**第 1 步：對帳（判讀 1）。** 診斷項目 1 的理論偏移 vs 診斷項目 2 的實測校準差距：

- 對得上（實測差 ≈ $-\log r_j$ 或權重效應）→ **這不是模型病灶，是配置的數學後果**。處方：推論期 logQ 修正（在 log-odds 上把偏移加回去）或修訓練配置（Ch 5 槓桿 1）。修完**回到第 0 步重量**。
- 對不上的殘餘差距 → 帶著進第 2 步。

**第 2 步：分流（判讀 2）。** 跑診斷項目 6（offset sweep）：

- **缺口大部分被 offset 收復** → 問題在水準軸。再細分：(a) 殘餘校準差距可歸因於配置沒對乾淨 → 回第 1 步；(b) 模型其實校準正確、但 macro 指標就是要把冷門往上搬 → 這是**指標的要求、不是模型缺陷**——去 Ch 6 做指標決策，處方是常態化的後處理 offset 或訓練端 item 權重（Ch 5 槓桿 2）。
- **缺口 offset 收不回** → 條件判別力問題，進第 3 步。

第 2 步 (b) 需要一句理論說明，因為反直覺：就算模型把每個機率都估準，按機率排序最優化的也是 per-query 的 precision 型期望指標（probability ranking principle【文獻：Robertson, 1977】；對 AP 的延伸在 query 內獨立假設下成立【推論·信心中高】），**不是** macro per-item mAP——macro 給冷門正例 400 倍權重，最優解必須把冷門相對機率故意上移。**「修好校準」與「優化 macro」是互斥方向，要選邊**。

**第 3 步：定位條件判別力病灶（判讀 3）。** 對 offset 救不回的 item，看診斷項目 3（item 內 AUC）＋診斷項目 8（Gain 帳本）＋診斷項目 9（條件化 SHAP），分兩型：

| 證據組合 | 判定 | 處方去向 |
|---|---|---|
| item 內 AUC 低 ＋ Gain 帳本顯示該 item 子樹**沒有**個人化切點 | **餓死型**：訊號可能存在，模型沒分到容量去學 | Ch 5 槓桿 3（加權/欠採/HPO 先驗） |
| item 內 AUC 低 ＋ **有**切點預算 ＋ 條件化 SHAP 顯示沒有任何 context 特徵在該子母體有區分力 | **特徵缺失型**：現有特徵裡沒有訊號 | Ch 5 槓桿 5（補特徵）——注意這是框架裡唯一「方法只能縮小搜索範圍、不能給答案」的一段 |

**第 4 步：傷害歸因（判讀 4，選做）。** 要回答「具體是誰壓了誰」時用診斷項目 5、7、10：名次佔據找嫌疑人 → 成對帳本算代價 → 交叉購買率打折。

**第 5 步：動槓桿後回第 0 步。** 特別注意：動條件判別力的槓桿（加權/欠採）**會同時搬動水準**【手冊3 Ch8：加權讓輸出機率系統性偏高】，所以每輪之後第 1、2 步要重走。

### 4.2 判讀的組合速查表

| 觀測組合 | 判定 |
|---|---|
| 校準差距大、item 內 AUC 好、AP 差 | 水準問題：先對帳（配置），再看指標錯位 |
| 校準 OK、item 內 AUC 好、AP 仍差 | 被別人壓：查加害者（名次佔據＋成對帳本），處方多半在**別人**的水準或指標再平衡 |
| 校準 OK、item 內 AUC 差、無切點預算 | 餓死型 |
| 校準 OK、item 內 AUC 差、有預算、SHAP 無區分特徵 | 特徵缺失型 |
| 校準差、AUC 也差 | 兩軸都有；**先修水準**（便宜、閉式），修完重量再判 AUC |
| item 內 AUC ≈ 0.5 但該 item AP 很好 | 高水準常數型「加害者」（Ch 2 的例子）：它自己沒事，去查它壓了誰 |

### 4.3 三個判讀陷阱（每個都有具體的錯法）

**陷阱一：拿 AP／AUC-PR 跨 item 直比「誰比較不可分」。** 這兩個量的隨機基線 ≈ 正類率；冷熱 item 正類率差 100 倍，AP 差可能全是基線差。跨 item 比判別力一律用 item 內 ROC-AUC（基線恆 0.5）。

**陷阱二：拿邊際分數直方圖找「偏高的加害者」。** 傷害發生在**具體 query 內**的比較，跟跨 query 混起來的直方圖可以脫鉤。反例（數字造的，機制是真的）：客群分年輕/退休兩半，$C$ 的買家全在退休群、正例分數 1.5。item $H$ 年輕群 3.0、退休群 1.0 → 邊際平均 2.0 很顯眼，但在退休群從沒壓過 $C$；item $Y$ 年輕群 1.2、退休群 1.6 → 邊際平均 1.4 不顯眼，但每筆 $C$ 正例都被它壓。看直方圖會查 $H$ 放過 $Y$。所以水準傷害的觀測單位是 per-query 名次事實（診斷項目 5），直方圖只當初篩。另外名次對分數是階梯狀的：壓人者 1.6 vs 正例 1.5，把它下修 0.05 一格都不動、修 0.2 才翻轉——**傷害不與偏移量成正比**，取決於多少分數差落在偏移量之內。

**陷阱三：機率條件化錯母體。** 指標只由「有正例的 query」貢獻（Ch 1.2），所以「$H$ 壓上來時是不是假正例」要在「$C$ 買家的 query」裡量（＝交叉購買率，診斷項目 10），不能用全體 query 的邊際正類率。

## Ch 5. 槓桿與處方（建模可以調什麼）

先給總表，再逐一講機制。「鏈條強度」＝從診斷結果到「動這個槓桿會改善」之間，推理有多少既有依據。

| # | 槓桿 | 對應診斷結果 | 鏈條強度 |
|---|---|---|---|
| 1 | 推論期 offset／logQ 修正；或修採樣配置 | 判讀 1：校準差距可由配置解釋 | **閉式**——有公式，無推論鴻溝【手冊3 Ch10；文獻：King & Zeng, 2001；Yi et al., 2019】 |
| 2 | 常態化的後處理 per-item offset；或 item-aware sample_weight | 判讀 2(b)：校準正確但指標要再平衡 | **強**——直接對指標優化 |
| 3 | item-aware weight／熱門負類欠採（配 logQ）／HPO 搜索範圍先驗 | 判讀 3：餓死型 | **方向可靠、幅度不可預測**——必須配快驗證迴路 |
| 4 | 分箱（`max_bin`／`min_data_in_bin`） | 僅當量到「冷門子母體的特徵變異被全域分箱壓進同一 bin」的證據 | **次要、證據前置**【推論·信心中】 |
| 5 | 補特徵 | 判讀 3：特徵缺失型 | **斷層最大**——方法只能告訴你缺在誰身上，補什麼是領域知識 |
| 6 | 換訓練目標（lambdarank） | 整體性選項，不對應單一診斷 | 方向有依據【手冊4 Ch6】，是否採用是顯式決策 |

### 槓桿 1：logQ／offset 修正（水準軸的閉式解）

機制：欠採把「負對正的 odds」乘上保留率 $r_j$，模型學到的 log-odds 比真實高 $-\log r_j$；推論期加回 $\log r_j$ 正好抵銷【手冊3 Ch10】。這是全框架**唯一有閉式公式的處方**——先做它，成本近零、期望值最高。

### 槓桿 2：item 權重／常態化 offset（指標要求的再平衡）

判讀 2(b) 成立時（模型校準正確、macro 指標仍要抬冷門），有兩條等效路線：**後處理**——把診斷項目 6 找到的 $\delta^*$ 直接常態化進推論（透明、可隨指標換、不動模型）；**訓練端**——item-aware sample_weight 把冷門的梯度質量放大（手冊3 Ch8 有完整機制與數字：$w_C = 8$ 能把冷門個人化切點的 Gain 從 2.97 抬到 53.6、反超熱門）。差別：後處理只動水準；訓練端加權**同時**動水準與條件判別力（它讓冷門搶得到葉預算），所以若判讀同時顯示兩軸的問題，訓練端一石二鳥、但副作用（下述）也一起來。

### 槓桿 3：加權／欠採／HPO 先驗（餓死型的容量再分配）

- **item-aware weight 是首選**（力道能分 item 給；手冊3 Ch11 的選招順序）。起手 $w_j \propto 1/P_j$ 但要阻尼（開根號或設上限）——最冷 item 純倒數會把個位數正例的雜訊放大成主導【手冊3 Ch8】。
- **熱門負類欠採**與加權部分冗餘（欠採到保留率 $r$ ≈ 把被丟的列加權成 0），兩個都開要一起調、且欠採必配 logQ【手冊3 Ch10/Ch11】。
- **HPO 的角色只能是搜索空間先驗**：正則化旋鈕是全域的、冷熱需求互斥（冷門要鬆、熱門要緊），per-item 診斷映射不到 per-item 的 HPO 動作【手冊3 Ch9】。可辯護的先驗：`min_data_in_leaf` 的搜索上限別開太高、`num_leaves` 下限別太小——都是「別讓 HPO 把冷門僅存的切點空間鎖死」的方向【推論·信心中】。
- 共同的**副作用警告**：這組槓桿全部會搬動水準（輸出機率偏高），每輪之後判讀第 1、2 步要重走。

### 槓桿 4：分箱（證據前置的次要旋鈕）

機制假說：直方圖分箱是在全域特徵分佈上建的，冷門 item 相關的特徵範圍可能被壓進少數幾個 bin、個人化切點無切可選。**先量再動**：在有標籤樣本上檢查「冷門買家的關鍵特徵值域 vs bin 邊界」，量到證據才調 `max_bin`／`min_data_in_bin`【推論·信心中——機制合理，但本 repo 未實證過它是主要瓶頸】。

### 槓桿 5：補特徵（框架的真空段，誠實條款）

特徵缺失型的處方是補特徵，但**沒有任何方法能從診斷直接推出「該造什麼特徵」**——診斷（條件化 SHAP、子母體分析）能做到的極限是縮小搜索範圍：「這個 item 的買家，現有特徵沒有一個分得開他們」＋這群人的側寫。從側寫到新特徵，是領域知識。把這段講成方法能閉合，是過度宣稱。

### 槓桿 6：換訓練目標（清單外、但該顯式決策的選項）

pointwise `binary_logloss` 的訓練目標與排序用途本來就有錯位（手冊4 Ch1 的脫鉤範例）；by-query 的 lambdarank 會中和「共池稀釋」（冷門正例被全域 loss 總和淹沒那條軸），且 λ 的 $|\Delta|$ 權重可換成 AP 型【手冊4 Ch6/Ch8；文獻：Yue et al., 2007】、甚至掛上 per-item 權重逼近 macro 指標【推論·信心中——機制成立，效果未驗】。代價：輸出脫離機率尺度（手冊4 Ch8）。**不換可以，但「不換」應該是一個寫下來的決策，不是清單漏了。**

## Ch 6. item 權重怎麼選：macro～micro 光譜（並列選項，不選邊）

macro（item 等權）不是中性的預設——它隱含「每個 item 的商業價值相等」這個判斷，且把最冷 item 的估計雜訊放大 400 倍進主指標。這裡把選項與代價攤開；**選哪個是價值判斷，本文不替你選**。所有選項都該搭配診斷項目 4 的信賴區間協定。

| 選項 | 定義 | 優點 | 代價 |
|---|---|---|---|
| **(a) 原樣 macro** | 每 item 等權 | 冷門 item 有完整話語權，最貼近「每個產品線都要顧」 | 最冷 item 100 筆正例的雜訊佔 $1/M$ 的指標權重；把資料層治不了的部分（Ch 8）放大進主指標 |
| **(b) 門檻版 macro** | $P_j$ 低於門檻（例如 500）的 item 移出平均、另列觀察名單 | 去掉統計上讀不動的部分 | 門檻武斷；被移出的 item 失去優化壓力 |
| **(c) 收縮版 macro** | per-item AP 先向全體平均收縮（如 $\tilde{AP}_j = \frac{P_j}{P_j+k}AP_j + \frac{k}{P_j+k}\overline{AP}$）再等權平均 | 連續、不丟 item、雜訊自動降權 | 多一個超參數 $k$；解讀變複雜【推論·信心中——收縮是標準統計手法，用在 per-item AP 上是本文的組合建議，無文獻直接背書】 |
| **(d) 冪次權重** | $w_j \propto P_j^{\alpha}$，$\alpha\in[0,1]$：$\alpha=0$ ＝ macro、$\alpha=1$ ＝ micro（每筆正例等權，item 依 $P_j$ 比例分到話語權） | 一個參數表達整條立場光譜 | $\alpha$ 本身就是那個價值判斷，沒有逃掉 |
| **(e) 業務價值權重** | $w_j$ ＝ 每個 item 的商業價值 | 最誠實：指標直接對齊目的 | 要業務給數，而且會隨策略變 |

正交的第二個決策：**要不要截斷（AP@K）**。不截斷的 AP 連排第 10 名的正例都計分，但業務通常只推前幾名——若使用面只看 top-3，AP@3 才對齊。截斷會讓深名次的改善「不被看見」，是刻意的取捨。

一個貫穿的警告：**槓桿實作的就是指標隱含的權重**（item-aware weight 就是在執行你選的 $w_j$）。指標沒定案就調槓桿，等於對著會漂移的靶射擊——指標一換，調參全部重來。所以 Ch 6 的決策順位在所有槓桿之前。

## Ch 7. 理論依據總表與關鍵推導

本框架每個關鍵主張的依據等級，整理成一張表（標【推論】的條目＝讀者要自行驗證後再依賴的部分）：

| 主張 | 依據 |
|---|---|
| 冷門 item 的先驗便宜、個人化被葉預算餓死 | 【手冊3 Ch2/Ch4，含手算 Gain 數字】 |
| 欠採未修正 → log-odds 偏高 $-\log r$ | 【手冊3 Ch10 推導；文獻：King & Zeng, 2001】 |
| 加權讓輸出機率偏高、但保序 | 【手冊3 Ch8】 |
| 正則化旋鈕全域、冷熱需求互斥 | 【手冊3 Ch9】 |
| lambdarank 中和共池稀釋、但變不出票數 | 【手冊4 Ch6/Ch7】 |
| per-query 常數平移不影響指標、per-item 偏移全額影響 | 本文 Ch 2 的分解（初等，見下方推導一） |
| 「上方是真正例則不扣分」 | 指標定義的直接後果（Ch 1.2 驗算） |
| item 內 AUC 對水準免疫、與 per-item AP 互不保證（一個好推不出另一個好） | 推導二＋Ch 2 的常數分數反例【推論·信心高】 |
| 校準排序 ≠ macro 最優 | PRP【文獻：Robertson, 1977】＋ 400 倍權重論證【推論·信心中高】 |
| rank 指標不可加 → SHAP 不能直接分解它 | 推導三【推論·信心高，屬數學性質的直接檢查】 |
| 交叉購買率才是傷害折扣的正確條件化 | 指標定義（無正例 query 不計）＋【推論·信心高】 |

### 推導一：為什麼 per-item 偏移恰好是「平移不變性豁免不掉」的分量

query $q$ 內 item $j$、$k$ 的相對名次由 $s(q,j)-s(q,k)$ 的正負決定。把 $s(q,j) = c_q + \delta_j + \varepsilon(q,j)$ 代入：

$$s(q,j)-s(q,k) = (\delta_j-\delta_k) + \big(\varepsilon(q,j)-\varepsilon(q,k)\big)$$

$c_q$ 消掉了——所以「對同一 query 全體加常數、排序不變」成立；但 $\delta_j-\delta_k$ 留著——**per-item 偏移直接改變每一個 query 裡的每一對比較**。用貫穿範例驗算：Alice 三個分數同加 10 → 次序不變；只把 $s_H$ 從 2.0 降到 1.0（動 $\delta_H$）→ $C$ 從第 2 升到第 1、precision 從 0.5 變 1.0 ✓。

### 推導二：item 內 AUC 與 per-item AP 為什麼一個好推不出另一個好

per-item AP 由「item $j$ 在**它的正例 query 裡**的分數 vs **同 query 其他 item**」決定；item 內 AUC 比的是「$j$ 在正例 query vs 負例 query」的分數——$j$ 的正例列與負例列分屬不同 query，**永遠不會出現在同一個排序裡**，指標從頭到尾沒做過這種比較。兩者只在「其他 item 的分數對 $j$ 的正例/負例 query 大致同分佈」時近似掛鉤。反例見 Ch 2：常數 2.5 的 item AUC＝0.5、AP 近滿分；常數 0.5 的 item AUC 同為 0.5、AP 爛。

### 推導三：為什麼不能對 rank 指標直接做 SHAP，四條替代各自的成立條件

SHAP 的可加分解對象是**單一列**的分數或 loss；per-item mAP 對分數是階梯函數（幾乎處處梯度為零）、跨列不可加、依賴整個 query 的聯合分數——「把 SHAP 值加總成指標的分解」在數學上不成立。四條替代：

| 路線 | 成立條件 | 成本 | 判斷 |
|---|---|---|---|
| 歸因給可加代理損失（query 內成對 logistic） | 代理與指標的界要緊【文獻：Wang et al., 2018 的 LambdaLoss 框架】 | 低 | 可用的列層工具 |
| 成對帳本（$\lvert\Delta\text{AP}\rvert$ 會計） | 指標可寫成成對交換敏感度的和（AP/NDCG 皆可） | 低（候選少） | **首選**：零代理誤差 |
| 對集合做 Shapley | 特徵函數要自定義；精確算 $2^M$ | 高 | 用 $O(M)$ 次替換實驗（診斷項目 7a）回答同一問題 |
| 訓練資料歸因（influence 類） | 對可微訓練 loss 成立；到 rank 指標隔一層 | 高【文獻：Sharchilev et al., 2018；Brophy et al., 2023】 | 保留給點狀問題（例：冷門那幾筆正例是否綁架模型） |

## Ch 8. 天花板與誠實條款

1. **資料層天花板**：最冷 item 只有 ~100 筆正例是資料的事實。加權放大的是這 100 筆（含它們的雜訊），變不出不存在的訊號【手冊3 Ch11；手冊2 Ch6 的方差論證】。診斷框架能告訴你「這個 item 已經到天花板」，不能突破它——突破在資料層（更長觀察窗、更好的正類定義、外部訊號）。
2. **框架的真空段**：「特徵缺失型 → 補什麼特徵」沒有方法能閉合（Ch 5 槓桿 5）。
3. **統計功效**：所有 per-item 讀數在冷門端都是高雜訊的；沒有信賴區間的比較與迭代，前幾輪「改善」大概率是噪音【推論·信心高】。
4. **互斥目標要選邊**：保留 macro 就別同時追求完美校準（Ch 4 判讀 2(b)）；兩個都要是自相矛盾。
5. **本文中所有標【推論】的主張**——尤其「校準排序 ≠ macro 最優的 AP 延伸」「收縮版指標」「分箱機制」「HPO 先驗方向」——沒有直接文獻或手冊推導背書，採用前應先在自己的資料上驗證。
