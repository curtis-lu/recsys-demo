## 「ad-hoc → 可靠排程」能不能照走？

**整體判斷：七成可操作，三成需要補前提或問同事。**

**可以照走的部分（清楚的）：**
- 把所有 `INSERT INTO` 改成 `INSERT OVERWRITE PARTITION (snapshot_date='...')`：
  範例直接給、footgun 也清楚解釋，我照抄就能動手。
- `set -euo pipefail` + `_SUCCESS` sentinel：可直接複製，即使我只用 cron。
- `ANALYZE TABLE … COMPUTE STATISTICS` 排在產表後：清楚、可操作。
- `ALTER TABLE … DROP PARTITION`：懂了，但要另外確認 HDFS 清檔方式。

**需要補前提才能走的部分：**
- 「我要用 dbt 嗎？還是只用 Airflow？公司有沒有 dbt？」→ 需要問平台團隊。
- Airflow DAG 檔要放在哪、怎麼部署：本章沒有覆蓋（這不是本章定位，但對我是缺口）。
- `ExternalTaskSensor` Python 範例：看得懂邏輯，但「這段程式碼住在哪個檔案怎麼執行」
  我不知道，需要找 Airflow 入門文件。
- `logical_date` 的具體用法：需要查 Airflow 文件確認怎麼在 DAG 裡引用。

**無法確認的部分（需要環境資訊）：**
- 我的表是 Hive 表還是 datasource 表：需要用 `SHOW CREATE TABLE` 確認。
- 公司 Airflow 版本：2.x 還是 3.x，影響 `sla`、`backfill` 指令怎麼用。
- 公司有沒有設 `mapreduce.fileoutputcommitter.marksuccessfuljobs`（`_SUCCESS` 前提）。
