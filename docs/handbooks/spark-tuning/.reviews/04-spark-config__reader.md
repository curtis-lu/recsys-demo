## §4.5 一個 executor 的記憶體裡裝了什麼：execution／storage／overhead（行 114–149）── 特別留意點

這是「特別留意」的核心。我以無 DE 背景的人逐塊讀，講真話：**整體比我預期好懂，但有幾個術語和圖的細節會讓我絆倒，而且這節資訊密度明顯是全章最高的。**

- **卡關 9（術語沒先定義，重要）**：行 122–123 圖裡與行 132 內文「overhead（**堆外**記憶體）」、行 124「**heap**（8g）」——「堆外」「heap」第一次出現。內文行 132 說 overhead「給 JVM 自己、網路傳輸、shuffle 等非 heap 用途」，但**「heap」本身始終沒被解釋**：heap 是什麼？為什麼記憶體要分「heap 內」和「堆外（off-heap）」？對學過 Java 的人這是常識，對我（只寫 SQL）heap＝堆＝一個沒來由的詞。我會問「heap 是什麼？為什麼有些東西要放 heap 外面？堆外又是什麼意思、跟堆有什麼關係？」——這是無 DE 背景讀者**一定**會卡的點，而且 heap 是整節的地基（M、R、execution、storage 全在 heap 內，overhead 在 heap 外）。**強烈建議**首次出現用一句人話定義 heap（例如「heap＝JVM 拿來放你資料和運算物件的主記憶體區；heap 外（堆外／overhead）另留一塊給 JVM 自己和網路、shuffle 緩衝」）。否則整節我是踩在一個沒定義的詞上往下讀。
- **卡關 10（術語沒先定義，中度）**：行 133「（這套叫 **unified memory**，統一記憶體管理）」——「unified memory」第一次出現，括號附了中文「統一記憶體管理」算給了人話，**勉強達標**。但「統一」統一了什麼，要讀到行 139「execution 與 storage 會動態互相借用」才懂——也就是「統一＝execution 和 storage 不再各佔死、共用一個池、可互借」。建議把「統一」的意思（兩塊共用、可互借，相對於舊版固定切死）提到首次出現處點一句，否則「unified」對我只是個沒內容的標籤。

## §4.6 給 executor 配多少：core、記憶體、台數的實際設定（行 153–183）── 特別留意點

- **卡關 15（算術細節，中度）── 唯一卡的一步**：行 167 第 3 步「扣掉 overhead 得出 heap：overhead ≈ heap × 0.10，所以 **heap ≈ 20 ÷ 1.1 ≈ 18 GB**」——這裡從「20 GB（含 overhead）」反推 heap，用了 `÷ 1.1`。我會卡：**為什麼是 ÷ 1.1？** 我自己的直覺會是「20 GB 扣掉 10% = 20 × 0.9 = 18 GB」，剛好也是 18，所以我會以為是這樣算，但其實作者用的是 `heap + heap×0.1 = 20 → heap×1.1 = 20 → heap = 20/1.1 ≈ 18.18`。兩種算法答案接近（18 vs 18.18）但**邏輯不同**，而且只有 ÷1.1 才對（因為 overhead 是 heap 的 10%、不是總額的 10%）。對想複算的我，行 167 沒解釋「為什麼是除 1.1 不是乘 0.9」，我會用錯方法卻剛好得到接近的數、然後在別的數字上算錯。建議補半句：「總額 = heap + overhead = heap + heap×0.1 = heap×1.1，所以 heap = 20 ÷ 1.1」。這正是 §3「中間值要能加總回最終值」要的。

## 全章彙整

### 主旨一致性（§11）
**一致、不過度宣稱。** 從開場「AQE-first」到 §4.9 收尾，主軸（先 SQL 改寫＋喂統計 → 確認 AQE → 才動資源；資源層只能啟動時給、要連多租戶想）前後咬合。結論誠實，沒寫「調一下就好」的漂亮話，反而在 §4.9 和精確度說明 #3 主動承認「資源真不足時仍要照 §4.5–§4.7 調」。風險梯度（SQL 層低風險可回退 → 資源層高風險）這條線貫穿全章、§4.8 又走一遍，骨架清楚。

### 範疇／失衡（§11，特別留意點）
- **§4.5 記憶體段：份量合理、不必拆，但生詞拖重了它。** 這節是全章資訊密度最高、生詞最密的一節（heap／堆外／overhead／unified memory／M／R／execution／storage／動態互借／spill 救法）。內容該講、不算邊緣主題佔版面；但 heap、堆外、unified、「剩 40%」那塊都沒落地，讓它讀起來比實際更重。**補幾句人話即可，不需削減篇幅。**
- **§4.7 營運段：深度大致夠、不算太淺**，dynamic allocation 的 what／why／CDP 差異／排程三建議都有。失分在術語（external shuffle service、shuffle tracking、SLA、backlog）甩了沒落地，不是深度不夠。
- 沒有發現邊緣主題佔過多版面的問題。
