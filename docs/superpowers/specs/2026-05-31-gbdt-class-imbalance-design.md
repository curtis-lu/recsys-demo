## 2. 核心框架（貫穿全文）

先破除一個常見但不精確的印象：**「不平衡 ≠ 機率一定錯」**。

- logistic loss 是 proper scoring rule；$F_0 = \log\frac{\bar y}{1-\bar y}$ 已把 base rate 吸收進起點。在資料充足、模型有彈性時，機率估計**不因不平衡而系統性偏差**。
- 不平衡真正咬人的是**幾個具體位置**：起點 $F_0$ 的高度、梯度拔河、Gain 的「稀有區飢餓」、葉輸出收縮、決策門檻 0.5 的錯位、評估指標的錯覺。第一部逐一點名，第二部逐一開藥。

這個框架讓全文不流於「不平衡很糟、快用 SMOTE」的口號，而是精準定位 + 對症下藥。

### 第一部 — 診斷：不平衡在數學裡的足跡（逐章追蹤）

**Ch 1. 起點 $F_0$：prior 就是 base rate**（呼應主手冊 Ch3）
- $F_0 = \log\frac{\bar y}{1-\bar y}$；$\bar y$ 極小 → $F_0$ 極負 → 每人起點 $p_0 = \bar y$。
- 要把一個稀有正類推過 $p=0.5$（即 $F \ge 0$），所有樹必須合計把 $F$ 抬升 $|F_0|$。在學習率 $\nu$ × 有限樹數下可能爬不完。
- **關鍵澄清**：「模型把所有人都預測成負類」常常是**門檻假象**（$p$ 沒過 0.5），不是機率算錯 —— 對 $\bar y=1\%$ 的資料，一個真正例拿到 $p=0.4$ 其實是 base rate 的 40 倍、是高度有資訊的輸出。
- 數值：$\bar y = 1\% \Rightarrow F_0 \approx -4.595$。

**Ch 2. 梯度與海森的階級失衡**（呼應主手冊 Ch4、Ch5.3）
- $g_i = p_i - y_i$、$h_i = p_i(1-p_i)$。
- 起點處（$p_i = \bar y$ 很小）：負類 $g = \bar y$（小正值）、$h = \bar y(1-\bar y)$；正類 $g = \bar y - 1$（接近 $-1$ 的大負值）、$h = \bar y(1-\bar y)$。
- $G = \sum g_i$ 是一場拔河：多數負類各出小正梯度、少數正類各出大負梯度。全域上初始 $\sum g_i = 0$（正是主手冊 Ch3 的 $F_0$ 一階條件），但**局部區域**裡稀有正類的訊號容易被多數類稀釋。
- 點出 $h$ 對所有樣本在起點都 $= \bar y(1-\bar y)$（與 $y$ 無關），$\bar y$ 小時整體 $H$ 也小 —— 預告 Ch 3 分母效應。

**Ch 3. Gain 與「稀有區飢餓」**（呼應主手冊 Ch5.3）★ 核心機制
- 用一個手算範例貫穿（也供 Ch 11 重用）：某節點 20 樣本（18 負 + 2 正），當前所有人 $p = 0.01$。
  - 負類：$g = 0.01,\ h = 0.0099$；正類：$g = -0.99,\ h = 0.0099$。
  - 節點：$G = 18(0.01) + 2(-0.99) = -1.80$，$H = 20(0.0099) = 0.198$。
  - 一個乾淨切點把 2 正類分到左葉：$G_L = -1.98,\ H_L = 0.0198$；$G_R = 0.18,\ H_R = 0.1782$。
- 計算 Gain：
  - $\lambda = 0$：$\text{Gain} = \tfrac12[\frac{(-1.98)^2}{0.0198} + \frac{0.18^2}{0.1782} - \frac{(-1.80)^2}{0.198}] = \tfrac12[198.0 + 0.182 - 16.36] \approx 90.9$
  - $\lambda = 1$：$\text{Gain} = \tfrac12[\frac{3.9204}{1.0198} + \frac{0.0324}{1.1782} - \frac{3.24}{1.198}] = \tfrac12[3.844 + 0.0275 - 2.705] \approx 0.583$
- **重點**：乾淨的「孤立 2 正類」切點，Gain 從 $90.9$ 被 $\lambda=1$ 壓到 $0.583$（約 156 倍縮減），因為 $H_L = 0.0198$ 極小、$\lambda$ 完全主導分母。再加上 `min_child_weight` / `min_sum_hessian_in_leaf`（葉內 $H$ 下限）與 `min_data_in_leaf`，常直接**禁止**這種小葉切點。
- **誠實框架**：這不是 bug，是 bias-variance 的正則化（對只有 2 筆證據的葉子保持懷疑是對的）；但在極端不平衡下，它**系統性地讓少數類更難被切出**。這正是 Ch 11 權重要修的東西（同範例會在 Ch 11 重算）。

**Ch 4. 葉輸出 $\gamma = -G/(H+\lambda)$ 的收縮**（呼應主手冊 Ch6）
- 公式本身**無偏**（Newton step）。問題在小樣本葉的 $H$ 小，$\lambda$ 把 $\gamma$ 拉小 —— 稀有正類叢集得到的修正幅度被壓縮，於是「爬升 $|F_0|$」更慢。
- 用 Ch 3 的左葉數字示範 $\gamma$ 在 $\lambda=0$ vs $\lambda=1$ 的差。

**Ch 5. 決策門檻 0.5 的錯位**（呼應主手冊 Ch1.4）
- $p \ge 0.5 \iff F \ge 0 \iff$ odds $\ge 1{:}1$。但資料的 base odds 是 $\bar y/(1-\bar y) \ll 1$。用 0.5 等於默認「正負類等可能、等成本」，與不平衡情境矛盾。
- 一般化：對機率設門檻 $\tau$ 等價於對 $F$ 設門檻 $\operatorname{logit}(\tau) = \log\frac{\tau}{1-\tau}$。預告 Ch 9 的成本式門檻。

**Ch 6. ★ 不平衡程度光譜（$1\% \to 0.1\% \to 0.01\%$）**（新增，診斷部綜合收尾）
- 把 Ch 1–5 的每個效應「攤平成 $\bar y$ 的函數」，用一張對照表掃過三個不平衡程度：

| | $\bar y = 1\%$ | $\bar y = 0.1\%$ | $\bar y = 0.01\%$ |
|---|---|---|---|
| $F_0 = \log\frac{\bar y}{1-\bar y}$ | $-4.595$ | $-6.907$ | $-9.210$ |
| 推過 $p=0.5$ 要爬升的 $F$ | $\approx 4.6$ | $\approx 6.9$ | $\approx 9.2$ |
| 100 萬筆中的正類數 | $10{,}000$ | $1{,}000$ | $100$ |
| $\text{scale\_pos\_weight} \approx n_{neg}/n_{pos}$ | $\approx 99$ | $\approx 999$ | $\approx 9{,}999$ |
| 主要後果 | 門檻偏移、要多爬 | 稀有區開始飢餓 | 加權方差爆炸、極少正類噪音主導 |

- 文字解讀：$|F_0|$ 隨 $\bar y$ 對數成長（爬升負擔線性增加）；正類絕對數量驟減（每個正類的統計重量被放大成噪音）；要用權重補償時 $w$ 衝到上萬，極少數正類各乘上萬 → 目標函數被一把手可數的樣本主宰，方差爆炸、易過擬合。

**Ch 7. 評估與早停的陷阱（簡述）**（刻意精簡，依需求壓成一節短講）
- accuracy 被多數類灌水（全predict負類在 $\bar y=1\%$ 就有 99% accuracy）。
- ROC-AUC 對 base rate **不敏感**（可漂亮卻無助於操作點）。
- 早停看 val log-loss 會被多數類主導，可能在學到稀有結構前就停。
- 點到為止，把「該用什麼指標」留給第二部開藥時順帶提。

**Ch 8. 小結對照表**
- 一張表收束第一部：｜不平衡咬在哪｜對應數學量｜症狀｜對應處方（指向第二部章節）｜。

### 第二部 — 處方：如何有效處理

**Ch 9. 先問對問題：要可信機率值 vs. 要操作點上的好決策**
- (a) 需要**機率數值本身可信**（餵進期望損益/成本計算，絕對值有意義）。
- (b) 只需要**在某操作點做對二元決策**（選門檻 $\tau$，對 $p \ge \tau$ 行動）—— 只有排序 + 切在哪重要，$p$ 的絕對校準無所謂。
- 這個分類決定後面哪些招對你有用，也吸收掉「校準」顧慮：落在 (b) 的人，權重/重採樣把 $p$ 絕對值弄偏無害（本來就會在驗證集重挑門檻）；(a) 才需要那段機率警語。

**Ch 10. 調門檻（最便宜、不扭曲模型）**（接回 Ch 5）
- 成本式最佳門檻：在最小化期望成本下，$\tau^* = \frac{C_{FP}}{C_{FP} + C_{FN}}$（給簡短推導：比較「預測正」期望成本 $(1-p)C_{FP}$ 與「預測負」期望成本 $p\,C_{FN}$）。
- 沒有明確成本時：在驗證集上掃 $\tau$ 最大化 $F_\beta$（$\beta>1$ 偏重 recall）/ 達到目標 precision 或 recall。
- 強調：這招**不改模型、不碰訓練**，常常是不平衡最該先試的解。

**Ch 11. ★ 類別權重 / `scale_pos_weight`（深入）**（接回 Ch 2/3）
- 機制：把正類的 $g_i, h_i$ 各乘權重 $w$ → 節點/葉的 $G, H$ 改變 → 孤立正類的切點 Gain 變大、葉 $\gamma$ 變大。
- **重用 Ch 3 範例**（同 20 樣本、$w = n_{neg}/n_{pos} = 9$）：
  - 正類加權後：$g = 9(-0.99) = -8.91$，$h = 9(0.0099) = 0.0891$。
  - 節點 $G = 0.18 - 17.82 = -17.64$，$H = 0.1782 + 0.1782 = 0.3564$。
  - 孤立 2 正類左葉：$G_L = -17.82,\ H_L = 0.1782$；$G_R = 0.18,\ H_R = 0.1782$。
  - $\text{Gain}(\lambda=1) = \tfrac12[\frac{317.55}{1.1782} + \frac{0.0324}{1.1782} - \frac{311.17}{1.3564}] = \tfrac12[269.5 + 0.028 - 229.4] \approx 20.1$。
  - 對比 Ch 3 的 $\lambda=1$ 未加權 Gain $= 0.583$ → 加權後 $\approx 20.1$，該切點重新被青睞。展示權重如何救回「稀有區飢餓」。
- $w$ 的選法：$w = n_{neg}/n_{pos}$（平衡兩類總梯度質量）為常見起點，可再調。
- LightGBM 三個入口的差異：`scale_pos_weight`（純量，乘正類 grad/hess）、`is_unbalance`（自動設 $\approx n_{neg}/n_{pos}$，與 `scale_pos_weight` 互斥不要同時開）、`sample_weight`（逐列權重，最一般）。
- 與 $\lambda$ / `min_sum_hessian_in_leaf` 的交互：加權同時放大葉 $H$，使原本卡住純正類葉的 Hessian 下限較易滿足（按 $w$ 比例）。
- **機率警語（一段）**：加權改動了被最小化的 loss → 輸出 $p$ 偏高、不再是校準機率。若落在 Ch 9 的 (b) 無妨；若是 (a)，可用 prior-shift 還原：$p_{\text{true}} = \dfrac{p_w}{p_w + (1-p_w)\,w}$（完整校準超出本文範圍）。

**Ch 12. ★ 重採樣（深入）**（接回 Ch 1/2/3）
- **欠採樣（undersample 多數類）**：改變經驗 $\bar y$ → $F_0$ 平移、訓練變快、類別變平衡；代價是丟資料 → 變異數上升、可能丟失多數類結構。還原：訓練在 $\bar y_s$、真實 $\bar y_t$，校正 $F_{\text{true}} = F_{\text{sampled}} + \log\frac{\bar y_t/(1-\bar y_t)}{\bar y_s/(1-\bar y_s)}$（欠採到平衡時 $\bar y_s = 0.5$，平移量恰為真實 $F_0$）。
- **過採樣（oversample 少數類，複製）**：
  - **證明「複製 = 整數權重」**：把樣本 $i$ 複製 $k$ 份，任何含 $i$ 的節點對 $G$ 貢獻 $k\,g_i$、對 $H$ 貢獻 $k\,h_i$，與把 $i$ 加權 $k$ 在 Gain/$\gamma$ 計算上完全等價。
  - **與權重的微妙差異**：複製讓 `min_data_in_leaf` 把 $k$ 份算成 $k$ 筆（湊樣本數門檻），而加權仍是 1 筆；兩者對 `min_sum_hessian_in_leaf`（看 $k\,h_i$）則相同。
  - 過擬合風險：完全相同的列永遠落同一葉 → 樹可記憶這些點。
- **SMOTE / 合成**：在特徵空間插值造新正類；對 tree/tabular 的侷限（樹靠閾值切分，兩個真實點之間的插值點未必合理；類別特徵不適用；計算成本）。
- **三法對照表**：｜手段｜對梯度總和的效果｜對 base rate / $F_0$｜變異數 / 過擬合｜計算成本｜何時用｜。

**Ch 13. 自訂 loss（精簡）**（接回 Ch 2–4）
- focal loss / weighted log-loss 如何改 $g_i, h_i$ → 改 Gain/$\gamma$：focal 用 $(1-p_t)^\gamma$ 調降「容易的多數類」樣本的梯度，把學習力集中到難樣本。
- 實作注意：GBDT 需自備一階/二階導數；focal 的二階導數**可能變號**，需處理（LightGBM 要求合理的 $h$）。超參數（focal 的 $\gamma$）與機率校準影響。
- 篇幅精簡，作為「進階選項」而非主力。

**Ch 14. 選招決策流程圖**
- 以「要可信機率值 vs 要操作點決策」×「成本是否已知」兩軸，給一張「先試什麼、再試什麼」的建議流程（文字版決策樹即可）。

**Ch 15. 一個常被忽略的事實**
- 若只需操作點決策（Ch 9 的 (b)）+ 調門檻（Ch 10）+ 用對指標（Ch 7），原生 GBDT 在不平衡下**常常不需要**重採樣/加權，機率還保持校準。許多「處理不平衡」其實是拿校準換一個你本可直接用門檻解掉的問題。呼應全文核心框架收尾。

## 6. 關鍵設計決策與理由

- **機率校準折衷**：完全跳過校準會讓「深入講權重/重採樣」數學上不誠實（讀者誤以為加權後 $p$ 仍可照舊解讀）。折衷 = 砍掉獨立校準章節與 Platt/isotonic 教學，只在 Ch 11/12 留一段警語 + 一行 prior-shift，並用 Ch 9 的「機率值 vs 操作點」框架把多數讀者的校準顧慮直接吸收掉（只要操作點決策就不在意絕對值）。
- **單一二分類語言**：不用「top-K / 排序」這種多標的詞；改用「可信機率值 vs. 操作點上的好決策」。
