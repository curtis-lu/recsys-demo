# 繁體中文語言慣例

> 本檔專指繁體中文寫作的語言層面約定。語言無關的散文原則（段落節奏、抽象落地、結論誠實等）見 `writing-style.md`，不在此重複。

---

## 1. 繁體中文

- **全文一律繁體中文**，任何地方不得出現簡體字。
- 常見混淆字對照（左為簡體、右為繁體，只用右邊）：

  | 簡體（禁） | 繁體（用） |
  |-----------|-----------|
  | 处理       | 處理       |
  | 节点       | 節點       |
  | 数据       | 資料       |
  | 执行       | 執行       |
  | 优化       | 優化       |
  | 来源       | 來源       |
  | 过滤       | 過濾       |
  | 缓存       | 快取       |
  | 队列       | 佇列       |
  | 实例       | 實例       |

- 定稿前執行全文 grep 掃描（`一-鿿` 範圍的簡體高頻字），不靠目測。

## 2. 專有名詞翻譯政策

**原則**：有公認慣用中文譯名才譯；沒有慣用譯法就直接用原文，不硬翻、不自創譯名。

### 判斷流程

1. 業界是否有公認繁體中文譯名？（可查台灣教科書、官方文件、ITHOME 等）
2. 有 → 用譯名。
3. 沒有 → 直接用原文。首次出現可附短說明：`predicate pushdown（謂詞下推，將過濾條件前推至資料來源層）`，之後連帶說明都省略，直接用原文。

### 正例（直接用原文）

| 術語 | 不硬翻的理由 |
|------|------------|
| `shuffle` | 台灣資料工程社群一律用 shuffle，無通用譯名 |
| `executor` | Spark 架構詞，譯成「執行器」易與其他元件混淆 |
| `predicate pushdown` | 無公認單詞譯名；首見附括號說明後直用原文 |
| `driver` | Spark 元件名，業界行文直用 driver |
| `schema` | 資料庫領域術語，中文社群慣用原文 |
| `checkpoint` | ML/Spark 兩領域均通用，無穩定譯名 |

### 反例（硬翻反而更難懂）

| 硬翻（禁） | 應改為 |
|----------|--------|
| 隨機排列（shuffle 的逐字翻譯） | shuffle |
| 執行子（executor 的生造詞） | executor |
| 謂詞下推器（predicate pushdown 機械直譯） | predicate pushdown（謂詞下推）── 首見附說明，之後直用原文 |

> 判斷標準：把譯名給一個熟悉該技術的同事看，若他需要停下來解碼，就不該譯。

## 3. 全形標點

### 基本規則

| 情境 | 用法 |
|------|------|
| 中文行文中的逗號、句號、冒號、分號 | 全形：`，`　`。`　`：`　`；` |
| 中文行文中的括號 | 全形：`（）` |
| 中英混排：英文/數字本身內部 | 半形（`()`、`,`、`.`） |
| 技術指令、程式碼片段 | 一律半形，包在 `` ` `` 或 ` ``` ` 區塊內 |

### 中英混排空白慣例

- **中文字與英文單字/數字之間加一個半形空格**，讓視覺分界清晰。
- 若英文緊跟全形標點，則不加空格（全形標點本身已有視覺空隙）。

```
正例：Spark 的 shuffle 操作會產生大量 I/O。
正例：每個 executor 負責 4 個核心。
正例：詳見第 3 章的說明。
反例：Spark的shuffle操作會產生大量I/O。（無空格，視覺擁擠）
反例：每個 executor 負責 4 個核心 。（全形句號前多了空格）
```

## 4. 稽核標點集（與腳本同源）

純標點稽核腳本 `references/scripts/punctuation_audit.py` 檢查的禁用字元集，定義在 `references/scripts/punctuation_set.txt`。

**以下為 `references/scripts/punctuation_set.txt` 的逐字引用：**

```
— ，：。（）；…
```

此為 `references/scripts/punctuation_set.txt` 的同步引用；純標點稽核腳本 `punctuation_audit.py` 讀的就是那個檔。要改標點集**改那個檔**，本段同步更新。

### 各符號說明

| 符號 | 名稱 | 在本手冊中的角色 |
|------|------|----------------|
| `——` | 破折號（兩個 em dash） | 僅限兩種用途（見 §5） |
| `，` | 全形逗號 | 中文行文標準逗號 |
| `：` | 全形冒號 | 列舉、引言、定義前 |
| `。` | 全形句號 | 中文句子結尾 |
| `（）` | 全形括號 | 中文行內補充說明 |
| `；` | 全形分號 | 並列複句之間 |
| `…` | 省略號（單個） | 省略內容（用兩個 `……` 表示完整省略號） |

## 5. 破折號 `——` 紀律

破折號只在以下兩種情況使用；其餘一律以逗號或句號替代。

### 允許的兩種用途

| 用途 | 說明 | 範例 |
|------|------|------|
| **術語定義** | 在術語後緊接其定義，等同「即」 | `predicate pushdown——將過濾條件前推至資料來源層` |
| **平行標籤** | 在條列項目中標注對稱的標籤或分類 | `正例——用原文；反例——硬翻生造詞` |

### 不該用破折號的情況（改用逗號或句號）

| 誤用情境 | 錯誤示範 | 正確改法 |
|----------|---------|---------|
| 補充說明插入句 | `Spark——一個分散式運算框架——支援多種語言` | `Spark（一個分散式運算框架）支援多種語言` |
| 轉折或遞進 | `效能很好——但記憶體耗用高` | `效能很好，但記憶體耗用高。` |
| 列舉展開 | `三個關鍵點——分區、快取、廣播` | `三個關鍵點：分區、快取、廣播。` |
| 總結收尾 | `以上都做到——才算完成` | `以上都做到，才算完成。` |

> 自查：定稿前 grep `——`，逐一確認每個用途是否屬於「術語定義」或「平行標籤」，否則替換。
