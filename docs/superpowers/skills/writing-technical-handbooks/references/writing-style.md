# Writing Style：語言無關的散文原則

> 字句層的「怎麼下筆」。繁體中文用字規範見 `language-conventions.md`；章的結構骨架見 `checklists/chapter-template.md`。本檔只談「一句話該怎麼寫」，與語言、主題無關。

一句話精神：**寫給一個聰明、但不熟這個領域的人。一步都不要跳；每個抽象主張都要落到具體；結論要對得起內文。**

## 1. 簡潔，但要「夠讓人動手」的描述性

精簡是美德，但別把讀者**真正要動手所需的描述**一起砍掉。判準不是「字夠不夠少」，而是「讀者照這句，能不能做出下一步」。

- ❌ 過度精簡：「設好參數就會變快。」（哪個參數？在哪設？設成什麼？讀者卡住。）
- ✅ 夠描述性：「在設定檔把 `batch_size` 從 1000 調到 100；數字越小、單批記憶體越省，但批數變多、總時間略增。」

精簡的對象是**贅字與重複**，不是**讀者需要的具體資訊**。

## 2. 宣稱與新名詞，先給脈絡再用

- **任何宣稱先交代「為什麼／在什麼條件下成立」**，不要丟一句沒有前提或出處的斷言。
  - ❌「這樣最快。」 ✅「在資料已照 key 排序時這樣最快；沒排序的話反而多一次 shuffle。」
- **新名詞首見當場用白話 gloss 一句，再開始用。** 讀者第一次承重的地方就要接住他。
  - ❌ 通篇用 `idempotent` 不解釋。 ✅「冪等（idempotent，同一步驟重跑幾次、結果都一樣）」。

哪些字要 gloss、哪些讀者已會，由 persona 決定（見 `persona-elicitation.md`）。

## 3. 抽象主張要有具體數字落地，並可複算

形容詞讓讀者無從體會程度。「很慢」「省很多」「爆掉」都要給一個可手算的小例子。

- ❌「這個寫法會放大很多。」
- ✅「每筆掃 22 個候選、1000 萬筆就是 2.2 億次比較；換成索引查找約降到 1/100。」

所有數值範例都要**實際算過、中間值能加總回最終值**；關鍵步驟可標 `✓`。

## 4. 用一個貫穿範例（capstone）前後呼應

同一組數字／設定，在「問題」處與「解法」處重複使用，讓讀者親眼看到因果——比兩個各自獨立的例子，說服力高得多。換範例物件時（續作、或結構不同）靠「角色身分」延續、不硬塞舊數字（見 `checklists/reviewer-checklist.md`）。

## 5. 結論誠實、與內文一致

- 不寫「其實沒那麼難／其實不用做」這類為了顯得有洞見的漂亮話，除非內文證據真的撐得住。
- 結論要對得起前面的診斷：內文花三節證明一件事很棘手，結論不能收成「調一下就好」。
- **寧可承認「沒有萬靈丹／有天花板／要看情況」**，也不要為了乾淨俐落而過度簡化。
- 自查：把結論單獨拿出來，問「這跟前面某節的證據矛盾嗎？」

## 6. 不要把寫作鷹架洩漏進成品

讀者讀到的，不該有任何只對作者有意義的東西：

- 標題不放給自己的備註（❌「第 7 章 評估（簡述）」）。
- 不用代號代稱概念（❌ 通篇「情況 (a)／(b)」），改用描述性命名（「判讀問題／學習問題」）。
- **不對讀者講你的寫作決定（後設旁白）**：❌「這節刻意寫短」「先埋一條主軸」「順手驗證一下」「這裡先記住」。把實質正常寫進行文即可。
- 後設旁白會反覆溜出 → 當成**全文 grep 稽核**（掃 `刻意`、`順手`、`先記住`、`埋.*主軸`、`收斂`、`這一節／這一章` 等），別只改被點到的單句。
- 也別解釋「範例怎麼造」：❌「我把 B 擺成和 A 鏡像」→ 改寫成讀者直接觀察得到的事實：「B 的處境和 A 相同，只差…」。
- 不留 `TODO`、`暫名`、規劃階段的佔位字句。

## 7. 符號與術語紀律

- 新符號／術語在首次出現處定義（或收進開頭對照表）；公式／流程用到的每個量，讀者都要能在前文找到定義。
- **絕不用同一符號代表兩件事**；真的撞名就改記號並註明原出處慣例。
- 借自他領域的行話，翻成本手冊一致的領域詞，全書統一。具名範例角色（Alice／Bob）也算術語：首見當場引介，不要前一章就 forward-ref 一個還沒定義的名字。

## 8. 通用原理先於具體案例，兩者分節

先講普適原理，再套到手上的問題；兩層用明確的小節或段落隔開，讓讀者知道現在在讀哪一層。先建立「這是什麼、為什麼」，再示範「在這個案例裡長怎樣」。
