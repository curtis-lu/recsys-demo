# 定稿前自審清單

逐條打勾。有一條沒過就先修再定稿，不得帶著已知問題發布。

---

## 貫穿範例一致性

- [ ] 貫穿範例的**參考點**全書只有一份：每次重算都從同一組初始值出發，或明確標出「這裡換到新起點，原因是 X」。
- [ ] 所有中間步驟顯示出來，且能加總回最終值（可手算複核）。
- [ ] 若新章節的主題需要結構不同的範例物件，明確說明「沿用角色身分，重建數字」，不硬塞舊值進新結構。

---

## 必然 vs 巧合

- [ ] 每個出現在範例的關係，都標明了「這是**必然**（由定義/等式強制）」還是「這是**這組數字才這樣**（巧合）」。
- [ ] 讀者不需要靠猜測來判斷某個等式是恆等式還是碰巧成立。

---

## 簡化斷言的規模壓力測試

- [ ] 凡是「一步到位 / 一刀解決 / 計算很便宜」的斷言，都明確說了它在 N 很大或類別很多時**是否仍成立**。
- [ ] 若 N 增大後方法的成本非線性增加，已誠實說明（不能只在 N=2 的小例子上說「便宜」）。

---

## 方向性宣稱先驗算正負號

- [ ] 每個「做 X → Y 會改善」的宣稱，都有一個數字（哪怕只有一個小例子）確認方向是對的。
- [ ] 直覺和算式衝突時，以算式為準，並在文中說明「直覺在這裡容易誤導，實際算出來是……」

---

## 新機制引入後回頭校舊章

- [ ] 每引入一個新機制，都問過「這個機制有沒有讓前面某章的敘述失準或需要補但書」。
- [ ] 若有，已在舊章補了一句「在引入 Y 後，這裡的描述需要加上前提：Z」，或在新章的開頭明確說明。

---

## 旋鈕給起手值與調法

- [ ] 每個可調旋鈕都給了：（1）起手值建議（附依據或慣例來源）、（2）往哪個方向調、（3）用什麼指標判斷調的方向對不對。
- [ ] 沒有只丟一個符號或參數名稱就結束，讀者不知道從哪裡開始。

---

## 引用查證

- [ ] 所有具名引用（作者姓名、年份、論文標題、工具版本）都已對照原始來源核實，不靠記憶填寫。
- [ ] 沒有把甲論文的共同作者安到乙論文上。
- [ ] URL 或版本號已標記，讀者能找到原始出處。

---

## 軟建議 vs 硬限制

- [ ] 「建議 X」和「規定 X / 上限是 X」沒有混用：文件若原文是「建議」，就不能寫成「限制」或「上限」。
- [ ] 每個硬限制都有來源（文件規格、硬體限制、語言語義），不是「感覺上應該這樣」。

---

## 鷹架與後設旁白零殘留

對全文執行以下掃描，每個命中的詞逐句判斷是否在「講寫作決定」而非「講內容」：

- [ ] 標題中無「（簡述）」「（待補）」「(a)/(b)」「TODO」等作者自用記號。
- [ ] 內文中無以下關鍵詞作為後設旁白使用：`劃出去`、`埋.*主軸`、`兌現`、`順手`、`先記住`、`動身`、`刻意(寫|擺|挑)`、`這一章/這一節想`、`我把.*擺成`。
- [ ] 沒有「解釋範例怎麼建構」的句子（例：「我把這組數字挑成可以整除」）——改寫成讀者直接觀察到的事實。
- [ ] 無殘留 `TODO`、暫名、規劃階段的佔位字句。
- [ ] 自問每一段：「這句在講範例呈現了什麼，還是在講我怎麼造它？」後者一律刪或改寫。

---

## 對比範例結論無歧義

- [ ] 每個「A 和 B 脫鉤」的對比範例，讓 A 和 B 指向**相反方向**（一高一低），不是一起高或一起低。
- [ ] 讀完對比後，讀者不會問「所以哪個比較好？我沒看出差別」。

---

## 數學精確度

- [ ] 「線性 / 指數 / 單調」等形容詞都已標明是「整條成立」還是「某段／某極限成立」。
- [ ] 引用公式的所有隱藏參數（正則係數、截斷值、預設常數）都已明文寫出，讀者能照著複算。
- [ ] 沒有一符二義：同一個符號在全文只代表一件事；若借用外領域符號且和本手冊已有符號衝突，已改記號並加注說明。

---

## 範疇平衡

- [ ] 邊緣主題沒有佔比核心主題更多的篇幅；若有，已壓縮並加「X 是獨立主題，本文不展開」一句劃出。
- [ ] 核心概念沒有因為「讀者應該知道」而跳過，尤其是「首次承重」的概念。
