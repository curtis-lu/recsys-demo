# 數學自學手冊寫作指引

寫《二元分類 GBDT 數學推導手冊》《類別不平衡的數學影響與處理》這類**自學向數學說明文件**時的指引。抽象自主手冊的既有風格，加上多輪修訂回饋。新文件、續寫、或請 subagent 審稿時，都對著這份走。

> 一句話精神：**寫給一個聰明但不熟這個領域的人，一步都不要跳，每個抽象主張都要有具體數字落地，結論要對得起內文。**

---

## 1. 先定讀者與前提，並明確劃定範疇

- **開頭就講清楚預設讀者會什麼、不會什麼。** 例：會微積分/線代/logistic regression，但沒系統學過 boosting。讀者已知的不重述，未知的不跳過。
- **把假設寫成明確「前提」。** 不要讓讀者讀到一半才發現文件其實只談某個情境。例：「本文假設輸出主要用來排序、不是拿機率絕對值去運算。」
- **切題外的主題一句話劃出去，不要讓它佔篇幅。** 句型：「X 是重要但獨立的主題，本文不展開。」邊緣主題佔太多版面，是常見的失衡 —— 把篇幅留給核心。

## 2. 通用原理先於具體案例，兩者分節

先講普適原理，再套到手上的問題；兩者用明確的小節或段落隔開，讓讀者知道現在在讀哪一層。（例：先講 bias-variance 是什麼，再說它在加權上怎麼具體發作。）

## 3. 每個抽象主張都要有具體數字落地，並驗算

- **不要只用形容詞。** 「被主宰」「放大很多」「方差爆炸」「最便宜」這種話，讀者無從體會程度。給一個可手算的小數字例子。
  - 反例：「極少數正類被放大一萬倍灌進梯度。」
  - 正例：「100 個正類各乘 10000；其中 1 筆標錯，它對梯度的貢獻 = 10000×g，相當於 1 萬個正常樣本一起施力。」
- **所有數值範例都要實際算過、可複算。** 寫完用程式或計算器複核一遍；顯示的中間值要能加總回最終值。關鍵步驟用 `✓` 標驗證。

## 4. 用一個貫穿的範例前後呼應

同一組數字在「問題」與「解法」兩處重複使用，讓讀者親眼看到因果。（例：同一個 20 樣本節點，先在「稀有區飢餓」算出 Gain 被壓到 0.583，再在「加權」用同組數字算回 20.1。）比起兩個各自獨立的例子，貫穿範例的說服力高得多。

## 5. 結論必須與內文一致且誠實

- **不寫反直覺的「其實沒那麼難 / 其實不用做」式漂亮話**，除非內文證據真的撐得住。為了顯得有洞見而過度宣稱，會與內文打架。
- **結論要對得起前面的診斷。** 若內文花了好幾章證明某問題很棘手，結論不能輕描淡寫成「調一下就好」。
- **寧可承認「沒有萬靈丹 / 有天花板 / 要看情況」，也不要為了乾淨俐落而過度簡化。** 誠實的「先分清是哪種情況，分別怎麼辦」勝過漂亮但偏頗的單一結論。
- 自我檢查：把結論單獨拿出來，問「這跟第 X 章的證據矛盾嗎？」

## 6. 不要把寫作鷹架洩漏進成品

讀者讀到的，不該有任何只對作者有意義的記號：

- **標題不要放給自己的備註。** 反例：「Ch 7. 評估與早停的陷阱（簡述）」——「（簡述）」是寫給自己的提醒，不是給讀者的。
- **不要用代號代稱概念。** 反例：通篇用「Ch 9 (a)」「Ch 9 (b)」「mode 1」指代兩種情況，逼讀者翻回去查代號。**用描述性命名**：「判讀問題 / 學習問題」。
- **不要對讀者講你的寫作決定（後設旁白）。** 反例：「這一章刻意寫短，因為…」「一個必要的但書（別過度宣稱）」「本章的態度是 hedge」——這些是你對自己的提醒。把實質（這效應次要、這裡有個但書）正常寫進行文即可，別把「我決定怎麼寫」說給讀者聽。
- 不要殘留 `TODO`、`暫名`、規劃階段的佔位字句。

## 7. 流程與步驟要具體可操作

決策流程不要用密碼式縮寫塞進程式區塊（讀者看到 `τ* = C_FP/(C_FP+C_FN)` 夾在 ASCII 樹裡會卡）。改用「**第幾步 → 看什麼 → 得到什麼結果 → 再決定什麼**」的具體散文或清楚的編號清單，每一步都讓讀者知道怎麼執行、怎麼判斷下一步。

## 8. 符號紀律

- 沿用既有記號；新符號在首次出現處定義，或收進開頭的符號對照表。
- **絕不重用同一符號代表兩件事。** （例：$\gamma$ 已是葉輸出，focal 的聚焦指數就改記 $\gamma_{\mathrm f}$，並註明原論文用 $\gamma$。）
- 公式裡用到的每個量，讀者都要能在前文找到定義。

## 9. 跨章引用要正確

「見 Ch N」「呼應主手冊 Ch X」必須真的指對章節與小節。改動結構後，回頭檢查所有引用沒指錯（量是定義在 Ch 4 還是 Ch 5.3，要分清）。

## 10. 體例（對齊主手冊）

- **繁體中文**，不出現簡體字。
- 數學：行內 `$...$`、展示式 `$$...$$`（單獨成行、前後空行）。
- 關鍵結論用 `$$\boxed{...}$$`。
- **動機先於公式**：先說「為什麼需要這個量」，再給式子，再解讀。
- 一節一個概念；某節過長是「它做太多事」的訊號，拆開。
- 導覽：每章結尾「上一章 / 下一章」；目錄用 anchor 連結；離線 HTML 版加右下角浮動回頂鈕。
- 跨檔交叉連結雙向擺好（主手冊 ↔ 延伸手冊）。

## 11. 收尾：reader-subagent 通讀挑錯

定稿後，派一個**扮演目標讀者**的 subagent 從頭到尾通讀，逐段標卡關處。除了傳統的「卡關處」，務必要它主動檢查：

- **主旨一致性**：結論 / 開場主張是否與內文證據矛盾？有沒有過度宣稱？
- **抽象未落地**：哪些主張只有形容詞、缺具體數字或例子？
- **鷹架洩漏**：標題 / 內文有沒有作者自用的記號（如「（簡述）」「(a)/(b)」「TODO」）？
- **範疇失衡**：有沒有邊緣主題佔太多、或核心講太淺？
- **數值與符號**：實際驗算所有手算範例；檢查符號是否先定義、有沒有一符二義；跨章引用是否指對。

依回報分「真缺陷（補）/ 可加強（斟酌）/ 誤讀（不改或微調）」處理。

## 12. 多輪修訂才學到的進階校驗

這些是手冊被 subagent / 讀者反覆抓到、值得在定稿前主動自查的點：

- **貫穿範例的「參考點」要一致。** 若範例宣告「所有量都在某個起點（如 $p = \bar y$）評估」，後面每次重算都要待在**同一個參考點**；真要換（例如重採樣後 base rate 變了），必須明講「這裡換到新起點」，否則想驗算的讀者會卡死、且與前面的約定矛盾。
- **標明哪些關係是「必然」、哪些是「這組數字才這樣」。** 例：兩塊梯度和等值反號（$G_A = -G_B$）是 $\sum g_i = 0$ 逼出來的恆等式，但它的「值」是範例數字決定的。不講清楚，讀者會把巧合當定律、或把必然當巧合。
- **簡化斷言要對「規模」壓力測試。** 「一刀就解決」對 2 個項目成立，對 $M$ 個就要 $\sim M-1$ 刀——下任何「一步到位 / 很便宜」的結論前，先問「$N$ 很大、類別很多時還成立嗎」：成立就講清楚為什麼仍便宜（維度低、可攤分），不成立就修正。
- **「做 X → Y 會改善」的方向性宣稱，先算一個數確認正負號。** 直覺常騙人（例：欠採多數類負樣本，直覺以為少數類更突出、Gain 更大，實算反而是多數類的 Gain 下降）。算式的符號才是裁判，不要靠語感。
- **引入新機制時，回頭檢查它有沒有推翻前面的章節。** 例：後面才講的 leaf-wise 生長，會讓前面「弱勢類被強勢類的切點服務」的框架需要改寫成「那是分群切開之前的事」。新機制進場 ＝ 回頭巡一遍舊章是否還自洽。
- **多個處方針對同一問題時，要談它們的交互（疊加 / 衝突 / 冗餘），別各講各的。** 例：對弱勢加權 與 對強勢欠採 部分冗餘，疊加會過頭——明說擇一主力、聯合調參、盯過擬合訊號。
- **每個可調旋鈕都給「起手值 ＋ 怎麼調」，不要只丟一個符號。** 例：權重起手 $\propto 1/P_j$、但要阻尼（開根號或設上限）、最後在對的驗證指標（macro / per-group）上調。
- **查證引用與「軟建議 vs 硬限制」。** 作者姓名、年份、出處要對（別把 A 論文的共同作者安到 B 論文上）；把文件的「建議」寫成「上限 / 規定」是過度宣稱（例：套件對高基數類別是「建議改當數值」，不是「$\le 1000$ 的硬限制」）。
- **續作的貫穿範例：物件形狀變了就重建，靠「身分」續接而非「數字」。** 當續作的主題自然需要一個與前作**結構不同**的範例物件（手冊3 的範例是「per-item 聚合」，手冊4 的 learning-to-rank 需要「每位客戶一張排序 list」），別硬把舊數字塞進新結構。做法：建一個乾淨好手算的新範例，但讓「角色／身分」延續（沿用同一個熱門 $H$、冷門 $C$ 的身分與冷熱精神），讀者接得上，又不會為遷就舊值算出醜數字。
- **要證明「某量不依賴某變因」，用「控制變因的精確相等」，別用「兩個真實案例剛好接近」。** 手冊4 原想用「冷門買家 Alice 的梯度 ≈ 熱門買家 Bob 的梯度」（$0.279\approx0.274$）證明 per-query 梯度不看 item 冷熱，但這個「剛好接近」既要加「巧合」但書、又讓三位審稿讀者一起撞牆（Bob 其實處境不對稱）。改成「把兩人配置擺成鏡像、只差冷熱身分」→ 算出**精確相等**（$0.279=0.279$），既乾淨又把論點釘死。要 demo「$X$ 與 $Y$ 無關」，就刻意固定其他一切、只動 $Y$，秀 $X$ 不變。
- **引用的數字要連同它依賴的隱藏參數一起給，否則讀者復現不出來。** 手冊4 初稿給了葉值 $\gamma=0.591$ 卻沒說它用了 L2 正則 $\lambda=1$，三個審稿者都卡在「照 $\gamma=-G/H$ 算不出這個數」。凡是依賴某個沒寫出來的預設值（正則係數、截斷 $K$、$\sigma_0$…）的結果，就等於不可複算——把每個它依賴的量當場標出。
