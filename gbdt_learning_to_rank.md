# 從二元分類到排序學習：learning-to-rank vs pointwise GBDT，與多 item 冷熱門

這份手冊接續《[二元分類 GBDT 數學推導手冊](gbdt_binary_classification.md)》（下稱**手冊1**）、《[二元分類 GBDT：類別不平衡的數學影響與處理](gbdt_class_imbalance.md)》（**手冊2**）、《[多 item 共享模型下的冷熱門不平衡](gbdt_multiitem_imbalance.md)》（**手冊3**）。前三冊從頭到尾都站在同一塊地基上：**pointwise 二元分類**——每一列 $(\text{客戶}, \text{item})$ 是一道獨立的二元題，loss 是 `binary_logloss`。本文要問的是：

> **如果把這塊地基換掉，會怎麼樣？** 把訓練目標從 pointwise 的 `binary_logloss` 換成**按客戶分組的排序損失**（learning-to-rank，lambdarank）——因為目標不同，樹怎麼切、葉子輸出什麼，都跟著變。這對冷熱門 item 又意味著什麼？另一條路是不換目標、只**換結構**：每個 item 各訓一個模型。本文把這兩條路講清楚。

手冊3 在「單一共享 pointwise 模型」框架內證明了冷門 item 的病根：先驗便宜（$O(M)$ 刀 item split 修掉），但冷門的個人化切點搶不到 leaf-wise 的**葉預算**，且當冷門 item 的**絕對正類筆數**低到一定程度（保單 100 人 / 1000 萬列），單一共享模型有它的**天花板**。手冊3 結尾留下兩條可能突破的路，都是「換掉某個東西」——這正是本文的兩部。

**這份手冊的範疇（刻意收斂）：**

- **第一部（換目標函數）**：pointwise `binary_logloss` vs pairwise / listwise（RankNet → lambdarank）的**梯度差異**；以及「同一套 GBDT 建樹機器、餵不同的 $(g,h)$ → 切分與葉輸出皆變」的**生成策略差異**（明確對照手冊1）；最後分析 by-customer 排序目標對冷熱門做了什麼。
- **第二部（換模型結構）**：per-item 獨立模型——它消除了什麼、又付出什麼代價。

**明確不展開（一句話劃出去）：** 完整的**機率校準方法**（Platt scaling / isotonic regression）是重要但獨立的主題，本文只談「LTR 輸出為何不是校準機率」的**機制與後果**，方法留待後續；lambdarank 的**完整工程設定教學**（group 構造、`label_gain`、truncation 等旋鈕）只在與數學相扣處輕點；listwise 家族的其他成員（ListNet、softmax 交叉熵等）只點名定位、不逐一展開。

**前提：** LTR 一律以 **query = 客戶**分組（排該客戶名下的 items，對應 top-K 推薦）；「以 item 為 query、排客戶」只在把貢獻**歸因到某個 item / 做 per-item 評估**時才計算，不是訓練目標。沿手冊1–3 立場，輸出主要用來**排序、挑高分客戶**。預設讀者已讀過手冊1–3。

## Ch 1. 為什麼 pointwise 不是排序問題的「原生」損失

手冊1–3 的 `binary_logloss` 對**每一列**獨立發問：「這位客戶這個 item 的購買機率 $p_i$，你估得準不準？」它要的是每一列的機率都對。

但回頭看推薦的用途：行銷 PM 拿到的是「對這位客戶，22 個產品按分數高低排序，先推前幾名」。也就是說，真正被使用、被評估的，是**同一位客戶名下、各 item 的相對次序**——不是任何單一機率的絕對值。

這就埋下一個錯位：**「把每列機率估準」和「把同一客戶名下的 items 排對」並不是同一件事。**

舉個具體的例子。某客戶實際買了 item A、沒買 B。考慮兩個模型：

- 模型甲：$p_A = 0.30,\ p_B = 0.20$。
- 模型乙：$p_A = 0.12,\ p_B = 0.45$。

若這位客戶的真實購買率接近 $0.2$ 上下，模型甲的兩個機率都離真值不遠、`binary_logloss` 漂亮；模型乙把 $p_B$ 估到 $0.45$、logloss 很差。但從**排序**看：模型甲把 A 排在 B 前面（$0.30 > 0.20$，正確），模型乙卻把 B 排在 A 前面（$0.45 > 0.12$，把該推的 A 排到後面去了）。logloss 漂亮的甲排對了——這次。但反過來也成立：你完全可以建構一個 logloss 更差、排序卻更對的模型。**logloss 的高低和排序的對錯沒有被綁在一起**：兩個模型可以有相同的 logloss 卻天差地別的排序，也可以有相同的排序卻不同的 logloss。

pointwise 損失沒有「同一位客戶名下要比較」這個概念——它把這位客戶的 A 列和那位客戶的某列一視同仁地丟進同一個總和裡。要讓**訓練目標**對齊**排序用途**，就得換一個「眼裡有排序」的損失。這就是 learning-to-rank。

在動身之前，先埋一條會貫穿整個第一部的主軸。手冊1 推導的整套建樹流程——選切點、算葉值——從頭到尾**只用到每列的一階導 $g_i$ 與二階導 $h_i$**（手冊1 Ch 4–6）。所以「換損失」這件事，具體落到 GBDT 上，就是**換餵進去的 $(g_i, h_i)$**。第一部會一路追這條因果：換損失 → 換梯度 → 同一套公式長出不同的樹。

## Ch 2. 三種取向：pointwise / pairwise / listwise

learning-to-rank 不是單一演算法，而是一個取向光譜，差別在於**損失「一次看多大範圍」**。

- **Pointwise**：一次看**一列**。把每列的標籤當回歸或分類目標去擬合——這正是手冊1–3 的 `binary_logloss`。它根本不知道「同一個 query 裡還有別的 item」。
- **Pairwise**：一次看**同一 query 內的一對** $(i, j)$。不去管 $i$ 的絕對分數該是多少，只要求「該排前面的那個，分數要比另一個高」。代表作是 **RankNet**（Burges et al., 2005）。
- **Listwise**：一次看**整張 list**。直接對整張排序的品質指標（如 NDCG、mAP）下手。

這三者一路把「視野」放大：從單列、到成對、到整張清單。視野越大，越貼近「排序」這件事的本質，但損失也越不平滑、越難直接微分（NDCG 是把分數排序後查名次算出來的，對分數而言是階梯狀、幾乎處處導數為零）。

**lambdarank 坐在 pairwise 與 listwise 之間**，而且是個聰明的折衷：它借用 pairwise（RankNet）那個平滑、好微分的梯度形狀，再乘上一個來自 listwise 指標（NDCG）的權重，讓「修正哪一對最划算」由整張清單的指標說了算。這個折衷正是 Ch 3（pairwise 梯度）與 Ch 4（乘上 $|\Delta\mathrm{NDCG}|$）要推的。

> 其他 listwise 成員（如 ListNet、softmax 交叉熵）用不同方式逼近清單損失，是各自獨立的題目，本文不展開。我們聚焦 lambdarank，因為它就是 LightGBM `lambdarank` objective 的數學內核，也最能與手冊1 的 GBDT 機制直接對照。

## Ch 3. RankNet：成對比較的梯度

先動機，再公式。我們要的不是「$i$ 的分數該是多少」，而是「$i$ 該不該排在 $j$ 前面」。怎麼把「該排前面」寫成一個可微的損失？

RankNet 的辦法：把分數差 $s_i - s_j$ 透過 sigmoid 變成一個機率，表示「模型認為 $i$ 排在 $j$ 前」的把握：

$$P_{ij} = \sigma\big(\sigma_0 (s_i - s_j)\big) = \frac{1}{1 + e^{-\sigma_0 (s_i - s_j)}}$$

這裡 $\sigma(\cdot)$ 還是手冊1 那個 sigmoid，$\sigma_0$ 是一個控制陡峭程度的常數（原論文記作 $\sigma$，本文為了不和 sigmoid 撞名改記 $\sigma_0$，玩具例取 $\sigma_0 = 1$）。對一個「已知 $i$ 確實該排在 $j$ 前」的對（$\mathrm{rel}_i > \mathrm{rel}_j$），目標機率 $\bar P_{ij} = 1$，用交叉熵當成本：

$$C_{ij} = -\bar P_{ij}\log P_{ij} - (1-\bar P_{ij})\log(1 - P_{ij}) \;\xrightarrow{\ \bar P_{ij}=1\ }\; \log\!\big(1 + e^{-\sigma_0 (s_i - s_j)}\big)$$

這個成本的意思很直白：當 $s_i \gg s_j$（已經排對、且差距大），$e^{-\sigma_0(s_i-s_j)} \to 0$、成本 $\to 0$；當 $s_i < s_j$（排反了），成本隨差距線性增大。

對分數 $s_i$ 微分（鏈鎖律，跟手冊1 附錄 A 對 sigmoid 微分同一套）：

$$\frac{\partial C_{ij}}{\partial s_i} = -\frac{\sigma_0}{1 + e^{\sigma_0 (s_i - s_j)}} \;\equiv\; \lambda_{ij}, \qquad \frac{\partial C_{ij}}{\partial s_j} = -\lambda_{ij}$$

這個 $\lambda_{ij}$ 就是 RankNet 帶給文件 $i$ 的梯度貢獻（一對一個）。值得停下來和手冊1 對比：

$$\underbrace{g_i = p_i - y_i}_{\text{pointwise：單列 vs 標籤}} \qquad\text{變成}\qquad \underbrace{\lambda_{ij} = -\frac{\sigma_0}{1 + e^{\sigma_0(s_i - s_j)}}}_{\text{pairwise：一對 vs 分數差}}$$

梯度的「作用單位」從「一列對一個標籤」變成「一對、只看兩者的分數差」。一筆文件 $i$ 的總梯度，是把所有牽涉到 $i$ 的對加起來：$\lambda_i = \sum_{j} \lambda_{ij}$（依 $i$ 是該對裡較相關還較不相關的一方定正負號）。

下一章補上最後一塊：RankNet 只問「對不對」，不問「在哪個名次出錯」。把第 1、2 名弄反，和把第 50、51 名弄反，代價不該一樣。

## Ch 4. 從 RankNet 到 lambdarank：用 $|\Delta\mathrm{NDCG}|$ 加權

排序的價值集中在頂端：客戶只看推薦清單的前幾名。把該排第 1 的 item 從第 2 名救到第 1 名，比把它從第 51 名救到第 50 名重要得多。RankNet 的 $\lambda_{ij}$ 對這兩種情況一視同仁——少了「位置」這個維度。

補進位置維度的指標是 **NDCG**（Järvelin & Kekäläinen, 2002）。先定義折扣累積增益，名次越後、折扣越重：

$$\mathrm{DCG}@K = \sum_{r=1}^{K} \frac{2^{\mathrm{rel}_r} - 1}{\log_2(r+1)} \;\xrightarrow{\ \mathrm{rel}\in\{0,1\}\ }\; \sum_{\substack{\text{買了的 item}\\\text{排在前 }K}} \frac{1}{\log_2(\text{rank}+1)}$$

把它除以「理想排序」（把所有相關 item 都擺到最前面）的 $\mathrm{IDCG}$，就得到落在 $[0,1]$ 的 $\mathrm{NDCG} = \mathrm{DCG} / \mathrm{IDCG}$。位置折扣 $1/\log_2(\text{rank}+1)$ 是關鍵：第 1 名拿 $1/\log_2 2 = 1$、第 2 名只剩 $1/\log_2 3 \approx 0.63$、第 3 名 $1/\log_2 4 = 0.5$——往後快速遞減。

lambdarank 的核心一步：把 RankNet 對每一對的梯度，**乘上「交換這對的排名位置會造成的 NDCG 變化量」**：

$$\boxed{\;\lambda_{ij} = -\,\frac{\sigma_0}{1 + e^{\sigma_0 (s_i - s_j)}}\;\big|\Delta\mathrm{NDCG}_{ij}\big|\;}$$

其中

$$\big|\Delta\mathrm{NDCG}_{ij}\big| = \frac{1}{\mathrm{IDCG}}\,\big|2^{\mathrm{rel}_i} - 2^{\mathrm{rel}_j}\big|\;\Big|\frac{1}{\log_2(1+\mathrm{pos}_i)} - \frac{1}{\log_2(1+\mathrm{pos}_j)}\Big|$$

直覺：$\big|\Delta\mathrm{NDCG}_{ij}\big|$ 是「這一對如果排序錯了，整張清單的 NDCG 會少多少」。錯在頂端（兩個名次的折扣差很大）→ $|\Delta\mathrm{NDCG}|$ 大 → 這一對的梯度被放大、優先修；錯在尾端 → $|\Delta\mathrm{NDCG}|$ 小 → 梯度被壓低、晚點再說。RankNet 平滑好微分的梯度形狀保留了，但「修哪一對最划算」改由整張清單的指標決定——這就是 pairwise × listwise 的折衷。

**二階導**：對 $s_i$ 再微分一次，得到的 hessian 正比於 $\sigma_0^2\,\rho_{ij}(1-\rho_{ij})\,|\Delta\mathrm{NDCG}_{ij}|$（其中 $\rho_{ij} = 1/(1+e^{\sigma_0(s_i-s_j)})$）。它是**正的**（GBDT 的 Newton step 需要），但**不再是手冊1 的 $p(1-p)$**——它是排序損失的二階近似，沒有「機率信心」那層意義。記住這點，Ch 5 會用到。

把整章收成一張對照表——這是理解第一部的核心：

| 面向 | `binary_logloss`（pointwise） | lambdarank（pairwise + listwise） |
|---|---|---|
| 梯度的作用單位 | 單列 $(\text{客戶}, \text{item})$ | 同一 query 內的「一對」$(i, j)$ |
| 一階導 | $g_i = p_i - y_i$ | $\lambda_i = \sum_j \lambda_{ij}$ |
| 由什麼驅動 | 標籤 $y$ 與預測機率 $p$ 的差 | 同 query 內相對分數順序 $\times$ 指標位移 $\lvert\Delta\mathrm{NDCG}\rvert$ |
| 在乎什麼 | 每列機率對不對（校準） | 同 query 內的相對次序對不對 |
| 位置敏感 | 否 | 是（$\lvert\Delta\mathrm{NDCG}\rvert$ 含 $\log_2$ 位置折扣） |
| 二階導 $h_i$ | $p_i(1-p_i)$，機率信心 | $\propto \sigma_0^2\rho(1-\rho)\lvert\Delta\mathrm{NDCG}\rvert$，無機率語意 |

兩欄的差別全在 $(g, h)$ 這一行。下一章就看：把這組不同的 $(g, h)$ 餵進手冊1 同一套建樹機器，會長出什麼不一樣的樹。

## Ch 5. 同一套建樹機器、不同的 $(g,h)$：切分與葉輸出怎麼變

這是 Ch 1 埋的主軸要兌現的地方。先把那條因果鏈講白：

> LightGBM 對**所有** objective 共用同一段建樹與算葉值的程式碼。手冊1 推導的 Gain 公式（**Ch 5.3**）與葉輸出公式（**Ch 6.3–6.4**）**原封不動**，唯一變的，是餵進去的 $(g_i, h_i)$。

複習這兩條公式（一字未改）：

$$\text{Gain} = \frac12\Big[\frac{G_L^2}{H_L+\lambda} + \frac{G_R^2}{H_R+\lambda} - \frac{G^2}{H+\lambda}\Big], \qquad \gamma = -\frac{G}{H+\lambda}$$

其中 $G = \sum g_i$、$H = \sum h_i$。pointwise 餵的是 $g = p - y,\ h = p(1-p)$；lambdarank 餵的是 $g = \lambda_i,\ h = $ 上一章那個排序 hessian。同樣的公式，不同的輸入，於是——

### 切分（樹結構）怎麼變

Gain 衡量「一刀分得好不好」，而「好不好」完全由 $(g, h)$ 定義。

- **binary 下**，一刀的 Gain 高，代表它把**正類的負梯度**從**負類的正梯度**分得乾淨（$g = p - y$：正類 $g < 0$、負類 $g > 0$）。樹在學「誰會買」。
- **lambdarank 下**，一刀的 Gain 高，代表它把**同一 query 內排錯序的成對**修正得多（$\lambda$ 驅動）。樹在學「怎麼把每位客戶名下的清單排對」。

這是兩種不同的「分得好」。同一筆資料，兩個目標會在**不同的特徵、不同的切點**上找到各自的最大 Gain，於是**長出形狀不同的樹**。

拿後面 Ch 7 會完整手算的 Alice 來看一眼（她買冷門 item C，模型現況把熱門 H 排在她清單頂端）。她那三列 $\{H, C, X\}$ 在兩個目標下拿到的梯度截然不同：

$$\text{pointwise } g = [\,g_H = 0.622,\; g_C = -0.5,\; g_X = 0.378\,]$$
$$\text{lambdarank } \lambda = [\,\lambda_H = +0.230,\; \lambda_C = -0.279,\; \lambda_X = +0.049\,]$$

pointwise 的 $g$ 只反映「每列的 $p$ 離 $0/1$ 多遠」（$H$ 是負類、$p$ 偏高所以 $g$ 大正；$C$ 是正類所以 $g$ 負）。lambdarank 的 $\lambda$ 反映的是「為了把 Alice 的清單排對，每個 item 該往上推還往下壓」：$C$ 要大力往上（$-0.279$）、$H$ 要往下讓位（$+0.230$）。兩個向量指向不同方向，依它們算出來的最佳切點自然可能不同。

### 葉輸出（葉節點）怎麼變

葉值 $\gamma = -G/(H+\lambda)$ 的公式沒變，但它的**意義**變了。

- **binary**：$\gamma$ 是一個 **log-odds 增量**。它加進 $F$，再經 $\sigma$ 就還原成一個校準過的機率（手冊1 Ch 1.3、Ch 6）。
- **lambdarank**：$G, H$ 是 $\lambda$ 梯度與排序 hessian 的和，$\gamma$ 是一個**排序分數的增量**，沒有任何機率錨點。

具體看一片**跨客戶**的葉子（假設 Alice 的 C 列和 Bob 的 H 列因為共用某個客戶特徵落進同一葉）。同一條 $\gamma = -G/(H+\lambda)$，兩個目標給出不同的值：

$$\gamma_{\text{binary}} = 0.591 \;(\text{log-odds 增量}) \qquad \gamma_{\text{lambdarank}} = 0.425 \;(\text{排序分數增量})$$

不只數值不同，**意義不同**：前者你可以丟進 sigmoid 讀出機率，後者不行。

### 為什麼 lambdarank 的輸出不是機率（嚴格理由）

這不是「實務上不準」這種模糊說法，而是有嚴格根據的。回看 Ch 3：排序損失 $C_{ij}$ 只透過**分數差** $s_i - s_j$ 進入。於是對任何一個 query，把它名下**所有** item 的分數同時加上一個常數 $c$，所有差 $s_i - s_j$ 都不變 → 損失一點都不變。換句話說，「整個 query 的分數該停在哪個絕對水準」這件事，**損失函數根本沒有意見**。

這對應到梯度上一個漂亮的恆等式：在一個 query 內，$\sum_i \lambda_i = 0$（每一對貢獻 $\lambda_{ij}$ 給 $i$、$-\lambda_{ij}$ 給 $j$，加總相消）。它的戲劇化後果是：**如果一片葉子剛好裝下某位客戶的整張 list**，那這片葉子的 $G = \sum \lambda_i = 0$，於是 $\gamma = -0/(H+\lambda) = 0$——樹對這片葉子不做任何更新。模型對「絕對水準」毫無學習訊號，它只在乎相對次序。

> 這個 $\sum_i \lambda_i = 0$ 是**必然**（源於損失只依賴分數差的恆等式）；而 Alice 那組 $\lambda$ 加起來剛好是 $0.230 - 0.279 + 0.049 = 0$，是這個必然的一次具體現身。

所以「LTR 分數不是機率」不是缺陷、是定義使然。Ch 8 會講它的實務後果（要機率得另外校準）；這裡先記住機制。

### 連帶：正則化旋鈕的語意也變了

既然 lambdarank 的 $h$ 不再是 $p(1-p)$，那麼依賴 $h$ 的旋鈕——尤其 `min_sum_hessian_in_leaf`——擋的東西就不一樣了：它不再是「葉內機率信心的總量」，而是「葉內排序梯度二階量的總量」。同一個閾值，在兩個目標下擋掉的葉子不同。這點在調參時容易被忽略。

## Ch 6. by-customer 排序對冷熱門做了什麼：兩軸分析

現在把目標換成 lambdarank、query = 客戶，回到手冊3 的問題：冷門 item 會不會好過一點？要回答得乾淨，先把手冊3 對冷門的「稀釋」拆成**兩條軸**：

- **軸 (a)：全域共池稀釋。** pointwise 下，冷門 C 的那 ~100 個正類，和全域 ~1000 萬列負類擠在**同一個 loss 總和**裡被數量淹沒；而且冷門個人化切點的 Gain，是拿「占總列 logloss 多少」來衡量的，那 100 列只是零頭，還沒算 `min_data_in_leaf` 直接禁掉只有兩三列的小葉（手冊3 Ch 4、Ch 7）。
- **軸 (b)：出現頻率。** 冷門 C 只在 ~100 位客戶的清單裡是「買了的相關 item」（熱門 H 在 ~40000 位）。

換成 by-customer 的 lambdarank，這兩軸的命運不同。

### 軸 (a) 被移除——用數字看

關鍵問題：冷門 item 全域稀有，會不會讓「為冷門買家修正排序」這件事的梯度也跟著被稀釋掉？**實算說不會。**

對照兩位客戶（完整設定見 Ch 7）：Alice 買了冷門 C、Bob 買了熱門 H，兩人現況分數相同。各自為「自己買的那個 item」貢獻的梯度大小是：

$$|\lambda_C|_{\text{Alice}} = 0.279 \qquad\approx\qquad |\lambda_H|_{\text{Bob}} = 0.274$$

**幾乎一樣大。** 重點在於：C 的全域稀有，**完全沒有縮小** Alice 這一筆的力道——因為 $\lambda$ 只在 Alice 自己的清單內計算，再按她自己的 $\mathrm{IDCG}$ 正規化，這位客戶的清單裡有沒有別人、C 在別處多罕見，都進不到這個數。

換句話說，lambdarank 把 Gain 的「計價貨幣」換掉了：從 pointwise 的「**占總列 logloss 的比例**」（冷門是零頭），換成「**各客戶 NDCG 改善之和**」（每一位被服務到的冷門買家都是一個**完整單位**）。per-query 正規化讓每位客戶等權，於是「熱門靠更多正類列灌爆全域梯度和」這個 pointwise 機制被中和了。這就是軸 (a) 被移除的意思。

> $0.279$ 與 $0.274$ 的微小不等是兩人現況名次/margin 的巧合，**本質是等大**，不是說它們會精確相等。

### 軸 (b) 仍在

但 lambdarank 變不出本來沒有的東西。要長出「把 C 排高給 C-買家」的切點，累積的 $\lambda$ 質量是對 **~100 位 C-買家**求和——仍遠少於熱門那 ~40000。樹照樣按全域最大 Gain 逐葉生長，冷門的個人化切點累積到的總梯度仍小，**還是會被低估，只是不再災難性**：從 pointwise 那種「被 1000 萬列尺度壓成零頭」，變成「只少了個 ~400 倍的客戶數差距」。另外，`min_data_in_leaf` 是**列數**的硬底、與 objective 無關，lambdarank 並沒有把它拿掉——冷到正類剩個位數時，小葉一樣會被禁切。

### 誠實的結論

把 by-customer LTR 對冷熱門的效果講準：

$$\boxed{\text{LTR 打掉「共池稀釋」軸 (a)，但「絕對稀疏 / 出現頻率」軸 (b)——也就是手冊3 的天花板——沒被變不見。}}$$

per-query 正規化讓每位冷門買家都是完整一票，這是真的進步；但只有 100 個人買的 item，換成 LTR 也生不出不存在的偏好訊號。能把冷門從「災難性低估」拉到「按客戶數比例的低估」，已經是換目標函數能拿到的東西——但別期待它根治。

對照手冊3 Ch 4：手冊3 把冷門的瓶頸定位成「搶不到 leaf-wise 葉預算」，那是在 pointwise 的 Gain 貨幣下說的。換成 lambdarank，貨幣變了、軸 (a) 那部分的壓迫鬆開了，但葉預算競爭的底層邏輯（按梯度質量排隊）還在，只是現在質量按 (b) 的客戶數計——這就是同一個瓶頸在新目標下的樣貌。

## Ch 7. 貫穿範例：一位冷門買家的 NDCG 與 $\lambda$ 手算

前面幾章的 $\lambda$、$|\Delta\mathrm{NDCG}|$、兩軸結論，全濃縮在這個能完全手算的小例子裡。

**設定。** 客戶 Alice，候選清單三個 item：熱門 $H$、冷門 $C$、中性 filler $X$。她實際買了 $C$（$\mathrm{rel}_C = 1$，$\mathrm{rel}_H = \mathrm{rel}_X = 0$）。模型現況分數帶著 popularity bias，把熱門 $H$ 頂在最前面：

$$s_H = 0.5,\quad s_C = 0.0,\quad s_X = -0.5 \;\Rightarrow\; \text{現況排序 } H@1,\; C@2,\; X@3$$

取 $\sigma_0 = 1$。位置折扣：

$$\tfrac{1}{\log_2 2} = 1.0,\qquad \tfrac{1}{\log_2 3} = 0.63093,\qquad \tfrac{1}{\log_2 4} = 0.5$$

### NDCG

Alice 唯一買的 $C$ 排在第 2 名：

$$\mathrm{DCG} = \frac{1}{\log_2(1+2)} = 0.63093, \qquad \mathrm{IDCG} = \frac{1}{\log_2(1+1)} = 1.0 \;(\text{把 }C\text{ 擺到第 1 名})$$
$$\mathrm{NDCG} = 0.63093 / 1.0 = 0.63093 \;\checkmark$$

模型把該排第 1 的 $C$ 壓到第 2，NDCG 從滿分 $1$ 掉到 $0.631$。

### 每一對的 $|\Delta\mathrm{NDCG}|$ 與 $\lambda$

$C$（相關）對上兩個不相關的 item，各成一對。交換它們的名次，NDCG 會變多少：

$$|\Delta\mathrm{NDCG}_{C,H}| = \tfrac{1}{1}\,|2^1 - 2^0|\,\big|\tfrac{1}{\log_2 3} - \tfrac{1}{\log_2 2}\big| = |0.63093 - 1.0| = 0.36907$$
$$|\Delta\mathrm{NDCG}_{C,X}| = \tfrac{1}{1}\,|2^1 - 2^0|\,\big|\tfrac{1}{\log_2 3} - \tfrac{1}{\log_2 4}\big| = |0.63093 - 0.5| = 0.13093$$

把 $C$ 和頂端的 $H$ 對調（救到第 1 名）能多賺 $0.369$ 的 NDCG，遠比和尾端 $X$ 對調的 $0.131$ 划算——位置維度在這裡發揮作用。代進 lambdarank 梯度（$\sigma_0 = 1$）：

$$\lambda_{C,H} = -\frac{1}{1 + e^{\,s_C - s_H}}\,(0.36907) = -\frac{1}{1+e^{-0.5}}\,(0.36907) = -0.62246 \times 0.36907 = -0.22973$$
$$\lambda_{C,X} = -\frac{1}{1 + e^{\,s_C - s_X}}\,(0.13093) = -\frac{1}{1+e^{0.5}}\,(0.13093) = -0.37754 \times 0.13093 = -0.04943$$

$C$ 拿到的總梯度：

$$\lambda_C = \lambda_{C,H} + \lambda_{C,X} = -0.22973 - 0.04943 = -0.27916 \;\checkmark$$

負號表示「把 $s_C$ 往上推」。其中和頂端 $H$ 那一對貢獻了 $0.230$、佔了絕大部分——把冷門從頂端的熱門手裡救上來，正是這位客戶最該修的事。

### 對照 pointwise 會給什麼

同樣這三列，若用 `binary_logloss`（$p = \sigma(s)$）：

$$g_H = \sigma(0.5) - 0 = 0.62246,\quad g_C = \sigma(0) - 1 = -0.5,\quad g_X = \sigma(-0.5) - 0 = 0.37754$$

pointwise 只說「$C$ 是正類、$p$ 該往 1 推」（$g_C = -0.5$），但它對「$C$ 該排在 $H$ 前面」這個**清單內的相對關係**一無所知——$g_C$ 和 $g_H$ 是各自獨立算出來的，不知道彼此同屬 Alice。lambdarank 的 $\lambda_C$ 則把「$C$ 要越過 $H$」的位置資訊編了進去。這就是 Ch 1 那個「目標 vs 用途錯位」在梯度層面的具體長相。

### query 內梯度和為零

順手驗證 Ch 5 的恆等式。$H$ 是 $(C,H)$ 對裡較不相關的一方，得 $\lambda_H = -\lambda_{C,H} = +0.22973$；$X$ 同理得 $\lambda_X = +0.04943$：

$$\lambda_H + \lambda_C + \lambda_X = 0.22973 - 0.27916 + 0.04943 = 0 \;\checkmark$$

這是必然（損失只依賴分數差），不是巧合——只有「值」是這組數字決定的。

### 放大到真實量級

Alice 只是一個人。真實裡，買冷門 C 的客戶約 100 位、買熱門 H 的約 40000 位。每一位冷門買家貢獻的梯度，都和 Alice 一樣是「完整一票」（軸 a 已移除，Ch 6）；但票數就是 $100 : 40000$，約 $1 : 400$（軸 b）。lambdarank 讓每張票等值，卻變不出更多張票——這就是它能做到與做不到的界線。

## Ch 8. mAP 變體與「輸出不再是機率」的後果

### 換指標 = 換 $|\Delta|$ 加權

lambdarank 的框架有個好處：把 $|\Delta\mathrm{NDCG}|$ 換成別的指標位移，整套梯度推導原封不動。要直接優化 **mAP（mean average precision）**，就把每一對的權重換成「交換這對會造成的 average precision 變化」$|\Delta\mathrm{AP}_{ij}|$（Yue et al., 2007 用結構化 SVM 做過同一件事的另一種版本）：

$$\lambda_{ij} = -\frac{\sigma_0}{1 + e^{\sigma_0(s_i - s_j)}}\;\big|\Delta\mathrm{AP}_{ij}\big|$$

NDCG 與 mAP 的取捨：NDCG 有 $\log_2$ 位置折扣、且支援多級相關（$2^{\mathrm{rel}}-1$）；mAP 是純二元相關、以 precision 為導向，對「前段命中密度」更敏感。要哪個，看你的業務指標長什麼樣——但對 GBDT 而言，它們只是 $|\Delta|$ 這個權重換了個算法，建樹機器完全不用動。

### 輸出不再是機率

承 Ch 5 的零空間論證：排序損失不約束分數的絕對水準，所以 lambdarank 訓出來的 $s = F(x)$ **不是校準過的機率**。這帶來兩個實務後果：

- **同一位客戶名下排序**：完全不受影響。加上任何常數、做任何單調變換，client 的清單次序不變（保序）。挑 top-K 推薦，這就夠了。
- **跨 item / 跨客戶比分數，或需要機率絕對值**（例如「分數 > 0.8 才推播」「估算期望申購數」）：這時分數的絕對水準有意義，而 lambdarank 沒給你這個保證。需要的話，得在排序模型之外**另做一層校準**，把分數映回機率。

完整的校準方法（Platt scaling、isotonic regression，以及它們在多 item 下「跨 item 可比」的額外難處）是一個獨立題目，沿手冊2 的立場本文不展開。這裡只要記住因果：**換成排序目標，就是拿「機率的絕對值」換「排序的對齊」**——你選了排序，就要接受輸出不是機率這個代價，並在需要機率時補一道校準。

## Ch 9. 換模型結構：per-item 獨立模型

第一部全程待在「一個共享模型」裡，靠換目標函數調整冷門的待遇。第二部換一個完全不同的方向：**不換目標，換結構**——乾脆讓每個 item 各訓一個模型（22 個產品就是 22 個獨立的 pointwise 二元模型，各自只吃自己那個 item 的列）。

這一刀下去，手冊3 的核心病根**因結構直接消失**。回憶手冊3 Ch 4：冷門 item 學不好的最深原因，是它的個人化切點在 leaf-wise 的**全域葉預算競爭**裡贏不過熱門 item。但「全域」指的是「同一棵樹裡所有 item 的切點一起排隊」——一旦每個 item 有自己的樹，**就沒有跨 item 的隊可排了**。冷門 item 的模型獨佔自己全部的 `num_leaves`，每一片葉子、每一刀切點都只為它自己服務，不必和熱門搶。

連先驗那筆小帳也省了。手冊3 Ch 2 的全域起點 $F_0$ 對冷門太高，要靠 item-isolation 切點修；per-item 模型裡，這個模型的起點直接就是 $F_0 = F_{0,C} = \log\frac{\bar y_C}{1-\bar y_C}$——**先驗自動正確**，連修都不用修。

把兩部擺在一起對照，就看清它們是正交的兩招：

- **第一部（LTR）**：留在同一個模型裡，**改梯度的計價貨幣**，讓冷門買家的每一票等值（軸 a 鬆開）——但冷門的票數仍少（軸 b 在）。
- **第二部（per-item）**：**拆開模型**，讓冷門根本不必和熱門競爭葉預算——競爭問題從定義上消失。

聽起來 per-item 一勞永逸。但天下沒有白吃的午餐，下一章算它的帳。

## Ch 10. per-item 的三筆代價

拆開模型消除了競爭，但也拆掉了「共享」本來帶來的好處，並且把冷門 item 推回一個更孤立的處境。三筆代價：

### 代價一：失去跨 item 的正遷移

這是手冊3 Ch 6 的故事翻過來看。手冊3 講過共享葉子的**負遷移**：冷門列被裝進一片以熱門為多數的葉子，被套上熱門買家的輪廓——冷熱買家若不相似，冷門被帶偏。

但同一個共享機制也有**正面**的一半：當冷熱買家**相似**時，冷門 item 能**借用**熱門 item 學到的特徵結構。共享模型在 1000 萬列上學到的「高資產、特定年齡層的人傾向申購投資類產品」這種泛化規律，冷門的投資型保單可以免費沾光——即使它自己只有 100 個正類、撐不起這條規律，它也能搭上熱門產品撐起來的那班車。這在多任務學習裡叫**正遷移（positive transfer）**。

per-item 模型把葉子拆開，負遷移消失了——但**正遷移也一起消失**。冷門 item 的模型從此只能用自己那 100 個正類去發現規律，再也借不到熱門的力。負遷移與正遷移是同一個共享機制的一體兩面，拆開模型就是兩面一起放棄。值不值得，取決於你的冷熱 item 買家到底相不相似：相似度高 → 正遷移本來幫很大 → 拆開損失大；相似度低 → 本來是負遷移在害冷門 → 拆開反而解脫。

### 代價二：每個模型回到手冊2 的極端稀疏，且無處借力

冷門 item 的獨立模型，訓練集就是它自己的列：~100 個正類 + 大量負類。這正是**手冊2** 從頭到尾在講的單一目標極端不平衡——起點 $F_0$ 太低、稀有正類要爬長路、$\lambda$ 與 `min_data_in_leaf` 把小葉打折禁切。手冊2 的整套病症，原封不動套在這個小模型上。

而且比手冊2 更慘一點：在共享模型裡，冷門至少還有「跟熱門借力」的可能（代價一的正遷移）；拆成 per-item 後，這條後路也斷了。**孤立的 100 個正類，就是它能用的全部。** 換目標函數變不出訊號（Ch 6 的天花板），換結構同樣變不出——只是把「競爭問題」換成了「孤立問題」。

### 代價三：$M$ 倍的訓練與維運成本

一個模型變成 $M$ 個（這裡 $M = 22$）：$M$ 套超參數要調、$M$ 個模型版本要管、$M$ 條監控要顧、上線要同時維護 $M$ 個 artifact。共享模型「一次訓練、服務所有 item」的工程簡潔被放棄了。對冷門 item 偏少、熱門佔絕大多數流量的場景，這筆固定成本未必划算。

## Ch 11. 合論：objective × structure 的 2×2

回到開頭那張 2×2。兩條軸——**目標函數**（pointwise / listwise）與**模型結構**（單一共享 / per-item）——是正交的，四個格子各有適用情境：

| | **pointwise** | **listwise（LTR）** |
|---|---|---|
| **單一共享** | 手冊1–3 的起點。工程最簡、輸出是校準機率；但冷門吃葉預算競爭（手冊3）。 | 第一部。輸出對齊排序用途、冷門買家的票等值（軸 a 鬆開）；代價是輸出不再是機率。 |
| **per-item** | 第二部。消除跨 item 競爭、先驗自動正確；代價是失去正遷移、每模型極端稀疏、$M$ 倍成本。 | **可組合**：每個 item 各訓一個 LTR 模型。競爭消失 ＋ 排序對齊兼得，但前兩格的代價也疊加（無機率 ＋ $M$ 倍成本 ＋ 失正遷移）。 |

選哪一格，是拿你最在意的東西去換你最能容忍的代價：要校準機率就別輕易離開左欄；要排序對齊就往右走、準備補校準；冷熱買家不相似、且冷門 item 重要到值得單獨經營，才考慮下排的 per-item。沒有一格是免費的。

### 誠實的天花板

把四格走一遍會發現一個共同的底線：**這些招數都在重新安排「誰跟誰競爭、用什麼梯度、誰借誰的力」，但沒有一招能無中生有地造出訊號。** 冷門 item 的根本困難——只有 100 個人買過——是**資料層**的事實。換目標函數（Ch 6 的軸 b 仍在）、換結構（Ch 10 的孤立問題），都繞不過它。

這呼應手冊3 的收尾，也把它講得更滿：當冷門 item 的絕對正類筆數低到一個程度，真正可能突破的不在「再換一個模型形態」，而在**資料層**（更長的觀察窗、更好的正類定義、引入外部訊號）或**更上層的結構**（把冷門 item 歸類、用 item 屬性做 content-based 補強、cold-start 專門機制）。誠實地知道天花板在哪，比硬把某一格的指標調漂亮更重要（cold-start／popularity bias 的固有難處，Meehan & Pauwels, 2025）。

### 後續

本文把「換目標」與「換結構」講完，但留了一個一路擱置的主題：**機率校準**。一旦走進右欄（LTR），輸出不再是機率（Ch 8），而推薦上線時往往又需要可比、可解讀的分數——怎麼把排序分數校回機率、跨 item 的分數又怎麼擺到可比的尺度，是一個獨立且夠分量的題目，留待後續手冊。
